# 说明书"具体实施方式"清楚性与逻辑一致性规范（所有案件类型通用）

## 适用场景

- 撰写说明书"具体实施方式"部分的过程中，每写完一段就应该顺手对照本 skill 自查，而不是全部写完才回头检查。
- 对已撰写完成的说明书"具体实施方式"部分做整体质检时使用。
- 检查的原则和 `权利要求质检/patent-claims-quality-review` 体系下的 `patent-claims-clarity-and-logic` 完全对应，只是检查对象从"权利要求"换成了"具体实施方式"的叙述性描述——权利要求要求精炼的法律文本表达，说明书则是详细、完整的技术叙述，但"清楚、逻辑自洽"这个底线要求是相通的。

## 规则一：孤岛特征——仅出现一次、前后都未呼应的特征

**什么是孤岛特征**：具体实施方式中提到的某个部件、参数或步骤，只出现了一次——前面没有交代它从何而来，后面也没有任何描述用到它、依赖它、或说明它和其他部件的连接/配合关系，像一座"孤岛"游离在整个实施方式的技术叙述之外。

**不要轻易判定为孤岛的例外情况**：
- **公知术语**：本领域技术人员一看就懂、不需要进一步说明其作用或连接关系的常规部件（比如"螺栓""电机""控制器"），即使只出现一次也不算孤岛。
- **暗含于后续操作或说明书其他位置的特征**：如果该特征虽然只出现一次，但它的作用能从紧邻的上下文动作中直接看出（比如"设有转轴12"之后紧跟"驱动组件带动转轴12转动"，转轴12的作用已经体现），或者在附图说明、权利要求书里对它有更完整的呼应，也不算孤岛。

**判断方法**：对具体实施方式中每一个带附图标记或专门命名的部件/参数提问——"它后面还有没有被继续提到（被连接、被驱动、被进一步描述结构或作用）？如果没有，它是不是公知术语，或者作用能不能从紧邻上下文直接看出？"三问都是"否"，才判定为孤岛特征。

**发现孤岛特征后怎么处理**：不要直接删除了事——先判断是撰写疏漏（这个部件本该有后续描述但漏写了）还是确实不必要，向用户说明具体情况，不要擅自删除。

示例：
```
不清楚：说明书中提到"壳体内还设有指示灯15"，之后再没有任何一处提到指示灯15的作用、触发条件或连接关系。
应补充：指示灯15与哪个部件电连接？在什么条件下点亮？起什么提示作用？
```

## 规则二：突然出现且不是客观清楚物理量的特征

- **客观清楚的物理量**：长度、角度、温度、转速等本领域公认、有明确度量方式的物理量，不需要额外解释。
- **人为定义的词**：撰写者自己根据本发明的具体结构定义出来的概念（比如"调节比""覆盖系数""响应等级"），如果说明书中突然使用、却没有给出明确的定义或计算方式（比如没有说明这个"比"具体是哪两个量的比值），就构成了公开不充分或含义不清楚。

**判断方法**：遇到说明书里的每一个"量""比例""系数""等级"类表述，先问"这是本领域公认的标准物理量，还是本发明特有的人为定义概念？"如果是后者，检查是否在首次出现处就给出了明确的定义或计算方式。

示例：
```
不清楚："当调节比大于1.5时，喷雾效果更好。"（"调节比"未定义，是哪两个量的比值？）
清楚："所述调节杆的伸出长度与所述壳体高度的比值（以下称调节比）大于1.5时，喷雾效果更好。"
```

## 规则三：技术概念模糊或含义不确定的用语

避免使用没有客观标准、因人而异的模糊表述（"大致""左右""明显""较好地""合理的"等），除非说明书本身给出了可以量化或明确判断的标准。

**判断方法**：逐句扫描，遇到程度副词、相对性形容词，追问"这个词的边界在哪里？本领域技术人员能否根据这个词准确复现或判断该实施方式？"如果答案模糊，需要替换成更精确的表述或补充量化标准。

示例：
```
不清楚："连接件的长度设置得比较长，以便于安装。"
清楚："连接件的长度大于壳体高度的一半，以便于安装。"
```

## 规则四：具体实施方式不能有歧义或反例

- **歧义**：同一段描述能被解读出两种或以上不同的技术含义（比如某个修饰语到底修饰前面哪个部件不清楚）。
- **反例**：能构造出一个具体场景，字面上符合说明书的描述，但明显不能实现该实施方式声称的技术效果，或者该描述的机构按字面理解根本无法正常工作——这说明具体实施方式的描述存在逻辑漏洞或表达不严谨。

**判断方法（反例测试）**：读完一段具体实施方式描述后，尝试构造一个"按字面理解却明显行不通"的场景。如果能构造出来，说明这段描述需要补充限定条件或修正逻辑漏洞，把这种不合理的字面理解排除掉。

示例：
```
描述："转轴可移动地设置在壳体内，从而实现高度调节。"
反例：如果没有限定移动方向和移动方式，字面上"可移动"甚至可以理解为转轴能在壳体内部朝任意方向自由移动（包括脱离原有配合关系），这显然不是发明真正想表达的"沿竖直方向滑动以实现高度调节"——需要补充具体的移动方向/导向结构描述。
```

## 规则五：描述需要清楚、简洁

- 避免不必要的重复表述、冗长的修饰语堆砌；一段话尽量只说清一个核心意思。
- 避免同一个技术点在不同段落反复用不同句式强调，导致读者需要来回对照才能确认是不是同一件事。
- 每个技术特征的描述应当只有一种自然的理解方式，读完应该能在脑海中直接构建出对应的结构/流程，而不需要反复琢磨句子结构。

**判断方法**：通读整段描述，如果需要读两遍以上才能确定某个短语具体指代的是谁、限定的是什么，就说明表达不够清楚简洁，需要重新组织语言或拆分成多个短句。

## 规则六：不能违背物理/数学常识，或前后逻辑矛盾

- **违背常识**：描述的技术方案在物理上不可能实现，或违反基本数学关系（比如声称某封闭腔体的容积随某个线性尺寸的减小而增大，量纲关系明显有误）。
- **前后逻辑矛盾**：说明书不同段落之间、或说明书与权利要求书之间，存在互相矛盾的描述（比如背景技术部分说现有技术"固定连接"存在某种缺陷，具体实施方式却把改进后的方案也描述成完全等价的固定连接方式，没有体现出改进点；或者附图标记指代的部件，在不同段落里被赋予了矛盾的连接关系）。

**判断方法**：
1. 对涉及物理量、数量关系的描述，检查量纲和数量级是否合理，是否符合基础物理/数学常识。
2. 逐段核对同一个部件/参数在不同位置的描述是否前后一致，是否存在互相排斥、无法同时成立的情况；顺带核对是否与权利要求书的相应表述一致。

## 检查清单

- [ ] 具体实施方式中每一个带标记/专门命名的部件或参数，是否都能在后文找到呼应，或者属于公知术语/暗含于后续操作？有没有游离的孤岛特征？
- [ ] 是否存在人为定义、非客观物理量的术语，且没有给出明确定义或计算方式？
- [ ] 是否存在"大致""适当""合理"这类没有客观边界的模糊表述？
- [ ] 能否构造出字面符合描述、但明显行不通或达不到声称效果的反例场景？描述是否存在可能被解读出多种含义的歧义表达？
- [ ] 整段描述读起来是否清楚、简洁，是否存在冗余重复的表述？
- [ ] 涉及物理量/数量关系的描述，是否符合基本物理/数学常识？说明书不同段落之间、说明书与权利要求书之间是否存在互相矛盾的描述？

## 应用范围

本规则适用于任意技术方案类型（结构类/方法类/算法类）说明书"具体实施方式"部分的撰写与质检。与 `权利要求质检/patent-claims-quality-review`（含其引用的 `patent-claims-clarity-and-logic`）配合使用：建议先质检权利要求书，明确独权和关键从权的技术特征与保护范围，再用本 skill 质检说明书是否清楚、完整、逻辑自洽地支持这些权利要求；也可以和 `通用/说明书撰写` 目录下的 `patent-technical-field-sentence-pattern`、`patent-background-technology-drafting` 配合，覆盖说明书的不同部分。
