# 创造性答复意见陈述书撰写规范

本 skill 用于**分步**撰写（或按用户反馈逐步修订）针对创造性缺陷的意见陈述书。最终目标是完整可提交正文，但**禁止在同一次回复中一次性写出全文**。

## 核心结构（务必按此布局）

答复由三块组成，不得把独权创造性论证拆散到别处：

1. **针对审查意见原文的逐条答复**（可有可详，视审查意见结论疏密）：完整照录审查员结论 → “对此，申请人答复如下：”→ 其一/其二… 针对性拆解。这段里可以把**区别特征指向、技术问题纠正、结合动机、技术启示、相反教导**等交织进去；**但有益效果不写在针对性答复里**。
2. **独权创造性论述——固定三节**（参照岩芯模版，标题与顺序不可改）：
   - **2.1、**申请人认为，权利要求1与最接近的现有技术对比文件1相比，至少具有以下区别技术特征：
   - **2.2、**权利要求1所保护的技术方案是非显而易见的。
   - **2.3** 修改后的（或本申请）权利要求1的技术方案具有显著的技术进步
3. **其他权利要求** + 开场/结尾。

**有益效果归属（硬性）**：凡针对性答复中本来会写的“本申请能够……效果”“取得预料不到的技术效果”等，**一律放到 2.3 技术进步一节**集中论述，不在逐条答复的“其一/其二/其三”里做效果收束。

### 针对性答复段模板（不含有益效果）

```text
第【序】、针对审查意见指出的：【完整照录该条审查意见原文，不得压缩概括】
对此，申请人答复如下：
其一、【就该条审查结论拆解：可指向区别特征（N）、纠正问题界定、指出对比文件未公开等】
其二、【无动机 / 无启示 / 相反教导 / 过度概括 / 退一步…——用第1步为本条选定的手法】
（视需要再写其三；强结论用“退一步来讲……再退一步来讲……”设防）
【此处不要写有益效果；效果统一写入后文 2.3】
```

### 独权创造性三节模板（2.1 → 2.2 → 2.3）

**2.1 区别技术特征（编号 + 权利要求原文）+ 实际技术问题 + 与审查员概括对比**

> 2.1、申请人认为，权利要求1与最接近的现有技术对比文件1相比，至少具有以下区别技术特征：
>
> 结合修改后的权利要求1，申请人认为，修改后的权利要求1相对于对比文件1至少进一步具有以下区别技术特征：
>
> 区别特征（1）：【权利要求原文，不得自行概括压缩】
>
> 区别特征（2）：【权利要求原文】
>
> ……
>
> 【简要说明对比文件1已公开何处、未公开上述哪一项】
>
> 基于上述区别技术特征，并结合本申请的背景技术，申请人认为，修改后的权利要求1实际要解决的技术问题应当为：
> 【申请人界定的具体技术问题】
>
> 具体而言，【由区别特征如何达成该问题的简要展开】。
>
> 审查员将实际解决的技术问题概括为“【审查员原概括用语】”，该概括未能具体反映区别技术特征所产生的【……效果】。相较而言，上述技术问题更能客观体现修改后的权利要求1相对于最接近现有技术所实现的技术改进。

**2.2 非显而易见性（四层递进，可与针对性答复分工）**

> 2.2、权利要求1所保护的技术方案是非显而易见的。
>
> 首先，……（最接近对比文件单独不能给出）
> 其次，……（其他对比文件不能弥补）
> 再次，……（结合后仍不能自然得到；连续改造清单；禁止事后拼贴）
> 最后，……（适用时反驳“常规步骤/公知常识”）

凡在针对性答复中已充分展开的点，此处做**体系化归纳**并可用“如本意见前文针对审查意见第×点所述……”指引，避免逐字重复；但 **2.2 标题与四层骨架必须保留**。

**2.3 显著的技术进步（有益效果统一写在这里）**

> 2.3 修改后的权利要求1的技术方案具有显著的技术进步
>
> 【按独权特征的数据流/结构顺序复述闭环：先…再…从而…】
> 【每一个效果必须挂在区别特征（N）上，并与 2.1 的实际技术问题镜像对应】
> 【明确：该效果不是最接近现有技术的简单公开方式所能直接体现的】
>
> 因此，修改后的权利要求1相对于现有技术具有显著的技术进步。
>
> 综上所述，修改后的权利要求1具有突出的实质性特点和显著的进步，具备创造性，符合专利法第二十二条第三款的规定。

### 区别技术特征列举硬性要求

- 用**权利要求原文**列出，不得压缩成自己的概括句。
- 用**编号**列出：`区别特征（1）：……` `区别特征（2）：……`；2.1 一次性列出全部；针对性答复与 2.2/2.3 引用时写“上述区别特征（N）”。

### 技术问题界定后必须对比审查员概括

1. 先写申请人界定的实际技术问题。
2. “具体而言，……”简要展开。
3. **紧跟**：`审查员将实际解决的技术问题概括为“【审查员用语】”，该概括未能具体反映……。相较而言，上述技术问题更能客观体现……`
4. 不得只写自己的问题而不点出审查员概括何处不当。

## 情形分支：是否修改权利要求

- **情形一｜修改了权利要求**：先写「一、修改说明」——将拟补充内容**接到原独立权利要求末尾**添加（并入来源 + 完整新增原文 + 法33条 + 细则51条）；再写针对性答复（若有）与「二、修改后的权利要求1具备创造性」下的 2.1→2.2→2.3。
- **情形二｜未修改**：无修改说明，开头一句“申请人坚持原权利要求”；针对性答复可为正文主干的一部分，但其后仍须有完整的独权创造性三节 2.1→2.2→2.3（用语改为“本申请权利要求1”）。

## 答复要点：核心必达 + 视情选用

**A. 核心必达**

1. **区别技术特征**（落在 2.1，编号 + 原文）
2. **实际解决的技术问题** + **与审查员概括对比**（落在 2.1）
3. **非显而易见性**（落在 2.2；针对性答复可先打具体靶子）
4. **有益效果 / 显著的技术进步**（**只落在 2.3**，不在针对性答复段内收束）
5. **修改说明**（仅情形一）

**B. 视情况选用**（按审查意见与案情挑用，不必全覆盖）

- 结合动机、技术启示、相反技术教导、过度概括、事后分析、正常研发路径、领域可迁移、退一步/再退一步、结合障碍等——按第1步选法对照表选定，用得上才写；多在针对性答复与 2.2 中展开。

## 硬性流程规则（最高优先级）

1. **禁止一次生成答复全文**。
2. **先分析选法、再落笔**：摘出审查员结论性论断，为每条挑选 2–4 种贴合手法；**并非每种都用**；选法须先交用户确认。工具箱只收通用手法，案情特有表述不得默认选用。
3. **用 plan/待办清单驱动全过程**：列出全部步骤与子项（每条针对性答复各一子项；2.1、2.2、2.3 各一步），逐步更新状态，始终只有一个“进行中”。
4. **一步只做一个交付物**；须等用户确认再进入下一步；按用户提示推进。
5. 用户要求“一口气写完”时仍分步；修订也按步。

每步输出格式：`【第X步 / 共N步】步骤名称` + 本步交付物 + 请用户确认 + 预告下一步 + 更新 plan 状态。

---

## 适用场景

- 审查意见指出不符合专利法第22条第3款（创造性）需书面答复。
- 第N次通知书；**次数、发文日、申请号按用户材料填写**。
- 不区分结构类/方法类/算法类。

不适用：仅新颖性、超范围、不清楚、得不到说明书支持、单一性等且不夹带创造性评述。

## AI 能力边界

- 不得编造对比文件未记载内容、段落号、效果或公知常识出处。
- 不得把本申请特征事后拆开贴回对比文件当作“启示”。
- 信息不足时只论证已能支撑的部分，并列出待核引用。

## 分步流程（必须按序执行）

```text
第1步 材料核对 + 审查意见结论清单 + 逐条选定反驳手法 + 判明情形 →（建立 plan 清单）
  ↓ 用户确认
第2步 修改说明（情形一）/ 坚持原权利要求（情形二）
  ↓ 用户确认
第3步-①/②/③… 逐条「针对性答复」（完整照录 + 驳斥；可交织特征指向/问题/动机/启示；**不含有益效果**）
  ↓ 每条各自确认
第4步 撰写独权创造性「2.1 区别技术特征 + 技术问题 + 与审查员概括对比」
  ↓ 用户确认
第5步 撰写独权创造性「2.2 非显而易见性」
  ↓ 用户确认
第6步 撰写独权创造性「2.3 显著的技术进步」（**有益效果统一写在此步**）
  ↓ 用户确认
第7步 其他权利要求
  ↓ 用户确认
第8步 组装全文 + 检查清单
```

### 第1步：材料核对 + 结论清单 + 选法 + plan（必做交付物）

**必收材料**：通知书、对比文件要点、当前权利要求书、拟修改方案或坚持不改。

**交付物：**

1. 材料齐备情况表。
2. 审查意见结论清单（结合动机型 / 问题界定型 / 常规公知型 / 教导忽略型）。
3. **逐条选定反驳手法**：从工具箱（A–E、共11种通用手法）挑 2–4 种；输出对照表（审查结论 / 类型 / 选用手法 / 对应区别特征编号）。**不是每种都用**。
4. 判明情形一/二；说明针对性答复与 2.1–2.3 的篇幅分配。
5. **建立 plan 清单**，例如：

   - [ ] 第1步 分析与选法
   - [ ] 第2步 修改说明 / 坚持原权
   - [ ] 第3步-① 针对性答复·结论①（手法 …）
   - [ ] 第3步-② 针对性答复·结论②（手法 …）
   - [ ] 第4步 2.1 区别特征 + 技术问题 + 与审查员概括对比
   - [ ] 第5步 2.2 非显而易见
   - [ ] 第6步 2.3 显著技术进步（有益效果）
   - [ ] 第7步 其他权利要求
   - [ ] 第8步 组装全文

### 第2步：修改说明 / 坚持原权

**情形一**：补充特征**接到原独立权利要求末尾**；完整引述新增原文与来源从权；声明法33条、细则51条。

> 1、申请人对原独立权利要求1进行了修改：根据原权利要求【X】的附加技术特征，在原独立权利要求1中添加以下内容：
> “【接在原独权末尾的完整新增文字，通常以「其中，……」起句】”
> 从而形成新的权利要求1。
> ……符合专利法第33条……细则第51条……

**情形二**：仅一句坚持原权利要求。

### 第3步（多子项）：针对性答复（不含有益效果）

逐条交付。要求：

- “针对审查意见指出的：”后**完整照录**该条原文。
- “对此，申请人答复如下：”后用第1步选定手法拆解；可指向区别特征（N）、纠正问题界定、论证无动机/无启示等。
- **禁止在本步写有益效果收束**；相关效果素材记下来，留到第6步 2.3 使用。
- 只用已选定手法；需增减先回第1步确认。

### 第4步：2.1 区别技术特征 + 技术问题 + 与审查员概括对比

严格按上文「2.1 模板」输出；顺序为编号原文特征 → 未公开说明 → 实际技术问题 → 具体而言 → **与审查员概括对比**。标题用：

> 2.1、申请人认为，权利要求1与最接近的现有技术对比文件1相比，至少具有以下区别技术特征：

### 第5步：2.2 非显而易见性

严格按上文「2.2 模板」四层输出。标题用：

> 2.2、权利要求1所保护的技术方案是非显而易见的。

与第3步分工：第3步打审查员具体靶子；本步做相对对比文件单独/结合的体系化非显而易见论证，可用“如前所述”指引，不逐字重复。

### 第6步：2.3 显著的技术进步（有益效果统一在此）

严格按上文「2.3 模板」输出。标题用：

> 2.3 修改后的权利要求1的技术方案具有显著的技术进步

- 效果挂在区别特征（N）上，与 2.1 技术问题镜像对应。
- 写入第3步答复时**刻意留到本步**的全部有益效果要点。
- 以创造性符合法22.3的收束句结束本节。

### 第7步：其他权利要求

从权依附独权具备创造性的概括 + 择要强化重点从权。

### 第8步：组装

仅当第2–7步均确认后拼接。按情形使用下文骨架。附检查清单自检。

开场：

> 申请人已收到贵局于【发文日】发出的就专利申请号为【申请号】发明专利申请的第【N】次审查意见通知书，感谢您对本申请认真负责的审查。
> 申请人认真研究了审查员的审查意见，答复如下：

结尾：

> 综上所述，本申请符合专利法和专利法实施细则的规定，敬请审查员在考虑上述意见陈述后能加快审查程序，批准本申请为发明专利为盼。如果本次意见陈述不能说服审查员授予本申请专利权，恳请审查员再次给予答复机会。最后，衷心感谢审查员的用心审查。
> 此致
> 衷心的感谢！

## 全文骨架（组装时用）

**情形一：**

```text
尊敬的审查员：您好！
开场

一、修改说明
二、针对审查意见核心观点的逐条答复（若有；段内可交织特征/问题/动机/启示；不含有益效果）
三、修改后的权利要求1具备专利法第22条第3款的创造性
    2.1 区别技术特征（编号+原文）→ 实际技术问题 → 与审查员概括对比
    2.2 非显而易见性（首先/其次/再次/最后）
    2.3 显著的技术进步（有益效果统一在此）+ 创造性收束
四、其他权利要求具备创造性

结尾 / 此致
```

**情形二：**

```text
尊敬的审查员：您好！
开场（含坚持原权利要求）

一、针对审查意见核心观点的逐条答复（不含有益效果）
二、本申请权利要求1具备专利法第22条第3款的创造性
    2.1 ……
    2.2 ……
    2.3 显著的技术进步（有益效果统一在此）+ 创造性收束
三、其他权利要求具备创造性

结尾 / 此致
```

若审查意见几乎无具体结论、针对性答复极短，可将“一、针对……”压缩为过渡句，仍须保留完整 2.1→2.2→2.3。

## 反驳手法工具箱（第1步据此选法；第3步、第5步取用）

**备选清单而非必用清单**。只收可跨案件复用的通用手法。

> 个别案件审查意见里的特有表述（例如“对比文件没有排除某手段，故有动机……”）属**案情特例**，不是通用手法，不得收入工具箱、更不得默认选用；仅当本案确有同类表述时，可按事后分析（第1项）等通用逻辑就事论事反驳。

### A. 结合动机 / 显而易见

1. **反对事后分析（事后诸葛）**
2. **正常研发路径叙事**
