# 创造性答复意见陈述书撰写规范

本 skill 用于**分步**撰写（或按用户反馈逐步修订）针对创造性缺陷的意见陈述书。最终目标是完整可提交正文，但**禁止在同一次回复中一次性写出全文**。

## 两种论证范式与深度融合（本 skill 的核心）

实务中优秀的创造性答复来自两种范式的结合：

- **范式A｜结论驳斥式**（针对审查员的结论性观点作答）：审查意见往往给出若干**结论性论断**（如“本领域技术人员有动机结合”“这是简单迁移”“属于公知常识”“实际解决的技术问题是……”）。范式A **逐条引述**每个论断，再“其一、其二、其三”**针对性拆解反驳**，配套一套可复用的**反驳手法工具箱**（见文末）。它的长处是"打靶精准"，直接瓦解审查员的说理。
- **范式B｜创造性三段论式**（正面立论）：按“区别技术特征 → 实际解决的技术问题 → 非显而易见（四层递进）→ 显著的技术进步/有益效果”正面建立创造性。它的长处是"体系完整"，把本申请的创造性讲成一条自洽的逻辑链。

**深度融合原则：**

1. **先破后立**：用范式A的“逐条驳斥”消解审查员的具体结论，再用范式B的“三段论”正面立起创造性。二者在同一份意见书中前后衔接。
2. **分工不重复**：针对审查员**具体论断**的反驳放进驳斥节（第二节）；本申请创造性的**体系化正面陈述**放进创造性节（第三节）。同一论点不在两处逐字重复——若非显而易见的某一层已在驳斥节充分展开，则在三段论对应处凝练归纳并指向前文（“如前所述……”），避免注水。
3. **按审查意见形态调配篇幅**：审查意见结论性论断越密集、说理越具体（典型如驳回或多篇对比文件结合），驳斥节（第二节）越重；审查意见较概括时，则以三段论（第三节）为主、驳斥节从简。无论哪种，两节都要有，只是篇幅此消彼长。

## 硬性流程规则（最高优先级）

1. **禁止一次生成答复全文**：不得在材料刚齐或用户只说“写意见陈述书”时，一次性写出开场、各节与结尾。
2. **先分析选法、再落笔**：拿到审查意见后，**第一件事是分析**——把审查员的结论性论断逐条摘出，为每条**挑选**将要使用的反驳手法（详见第1步）。**并非工具箱里每种手法都要用**，只选真正贴合本案该条结论的 2–4 种；未选用的手法不写进答复。选法结论须先交用户确认。
3. **用 plan/待办清单驱动全过程**：确定选法后，**必须用 plan（待办清单）功能**把本案答复要做的**全部步骤与子项**（含第二节每条审查结论各是一个子项、并标注为其选定的手法）一次性列出，让用户一眼看清"总共几步、每步做什么、当前写到哪一步"。此后**每完成一步就更新清单状态**（进行中 / 已完成），始终只有一个"进行中"。这份清单是用户跟踪进度的主要依据，不得省略。
4. **一步只做一个交付物**：每轮对话只完成清单中的**当前一步**，输出该步交付物后**停下**，请用户确认、修改或补充。
5. **须等用户确认再进入下一步**：未获用户对本步交付物的确认（或明确修改指示并完成修订）前，不得擅自撰写后续章节正文，也不得把清单里后续项标记为已完成。
6. **须按用户提示推进**：每一步都以用户当轮的提示/反馈为准来撰写或修订；用户没有发话推进时不自行跳步。
7. **用户要求“一口气写完”时仍分步**：应说明本 skill 强制分步，从清单中未完成的最早一步开始；仅在用户走完各步确认后，才在最后一步组装全文。
8. **修订也按步**：用户只改某节时，只重写该节，不连带重写无关章节，更不重出全文（除非用户已进入组装步）。

每步输出格式：

- 开头标明：`【第X步 / 共N步】步骤名称`（N 为本案 plan 清单的实际条数，随第二节结论条数而变）
- 正文为该步交付物
- 结尾固定追问：请用户确认本步内容（可直接改字），确认后再进入下一步；并**一句话预告**下一步
- 同时更新 plan 清单状态（本步标为已完成、下一步标为进行中）

## 适用场景

- 审查意见指出权利要求不符合专利法第22条第3款（创造性），需要书面答复。
- 适用于第N次审查意见通知书（一通、二通或更多次）；**通知书次数、发文日、申请号一律按用户材料填写**，不得沿用其他案件或模版中的次数表述。
- 不区分结构类/方法类/算法类；创造性论证逻辑通用。

不适用（另用对应专题 skill；尚无 skill 时如实说明并按常规法理答复）：

- 仅涉及新颖性、修改超范围、不清楚、得不到说明书支持、单一性等问题，且不夹带创造性评述。
- 通知书同时评述创造性与其他问题时：创造性部分按本 skill 分步；其他问题单独分节，不塞进创造性节。

## AI 能力边界

- **不得编造**对比文件未记载的技术内容、段落号、效果或“本领域公知常识”的具体出处。
- 不得在已知本申请方案后，把本申请的具体限定**事后拆开再贴回**对比文件当作“启示”——这本身就是要反驳审查员的手法（见工具箱“反对事后分析”），更不能自己犯。
- 创造性评估依赖用户提供的对比文件信息；信息不足时只论证已能支撑的部分，并列出待用户补充核验的引用。

## 分步流程（必须按序执行）

步骤总数 N 不是固定的：第3步会按第二节实际要驳斥的审查结论条数拆成若干子项，一并计入 plan 清单。下面是骨架步骤：

```text
第1步 材料核对 + 审查意见结论清单 + 逐条选定反驳手法 + 答复架构确认 →（据此建立 plan 清单）
  ↓ 用户确认（选法 + 清单）
第2步 撰写「一、修改说明」（或坚持不改的说明）
  ↓ 用户确认
第3步 撰写「二、针对审查意见核心观点的逐条答复」（范式A：引述+逐点驳斥）
      —— 按审查结论逐条撰写，每条为 plan 中一个子项，只用第1步为其选定的手法
  ↓ 每条各自确认
第4步 撰写「三 2.1 区别技术特征 + 实际解决的技术问题」（范式B）
  ↓ 用户确认
第5步 撰写「三 2.2 非显而易见性」（四层递进；与第3步分工、不重复）
  ↓ 用户确认
第6步 撰写「三 2.3 有益效果 / 显著的技术进步」+ 创造性节收束
  ↓ 用户确认
第7步 撰写「四、其他权利要求」
  ↓ 用户确认
第8步 组装开场 + 各节 + 结尾 → 完整意见陈述书 + 检查清单
```

> 若本案审查意见几乎不含具体结论性论断（少见），可与用户商定将第3步压缩为一句“审查意见的核心分歧”过渡，把论证重心放到第4–6步；反之若为纯结论驳斥打法，第5步四层可大幅精简并指向第3步。**两节都保留、篇幅可调**，这一调配在第1步与用户确认。

### 第1步：材料核对 + 审查意见结论清单 + 逐条选定反驳手法 + 建立 plan 清单（必做交付物）

写任何章节正文前，先只输出本步交付物（分析与清单），不写意见书正文。**本步是"先分析后撰写"的分析环节，最关键。**

**必收材料**（缺一则本步只追问，不进入第2步）：

1. 审查意见通知书全文，或至少含对比文件编号、评述逻辑、针对哪些权利要求
2. 对比文件要点（援引的段落/图号及公开内容）；仅有摘要时后续论证标注“据用户提供摘录，段落号待核”
3. 当前权利要求书（通知书所针对的版本）
4. 拟修改方案：并入哪些特征、来自哪条原从权/说明书何处；或明确坚持不改

**本步交付物：**

1. **材料齐备情况表**（已有 / 缺失）；材料不足时只列需补充的问题，结束本轮。
2. **审查意见结论清单**：把审查员的**结论性论断**逐条摘出并**分类**。常见类型：
   - 结合动机型（“本领域技术人员有动机……”“简单叠加/迁移”）
   - 问题界定型（“实际解决的技术问题是……”——常过度上位）
   - 常规/公知型（“属于常规选择/公知常识/常规调整”）
   - 教导忽略型（无视相反教导、缺失功能、结合障碍）
3. **逐条选定反驳手法（核心）**：对清单中**每一条**审查结论，从文末工具箱（A–E、共12种）中**挑选** 2–4 种最贴合本案的手法，用"手法编号 + 一句话为何适用本案"说明。**不是每种手法都用**；不贴合的坚决不选，避免为凑数硬套。用一张对照表输出，例如：

   | 审查结论 | 类型 | 选用手法 | 选用理由（结合本案） |
   |---------|------|---------|--------------------|
   | 结论① … | 结合动机型 | 1 不排除≠动机；4 相反教导；10 退一步 | … |
   | 结论② … | 问题界定型 | 6 纠正过度上位 | … |

4. **答复架构建议**：驳斥节（第二节）与三段论节（第三节）的篇幅分配；是否修改独权、并入哪条从权、创造性主线一句话。
5. **建立 plan 待办清单**：据上述结论条数与选法，**调用 plan/待办清单功能**列出本案完整步骤，让用户看清全貌与进度。清单形如：

   - [ ] 第1步 分析与选法（本步，完成后标记）
   - [ ] 第2步 一、修改说明
   - [ ] 第3步-①驳斥结论①（手法：1/4/10）
   - [ ] 第3步-②驳斥结论②（手法：6）
   - [ ] 第3步-③驳斥结论③（手法：7/8/E）
   - [ ] 第4步 三 2.1 区别特征+技术问题
   - [ ] 第5步 三 2.2 非显而易见
   - [ ] 第6步 三 2.3 有益效果/显著进步
   - [ ] 第7步 四、其他权利要求
   - [ ] 第8步 组装全文 + 检查清单

   此后每步开始时把该项置为"进行中"、完成并经用户确认后置为"已完成"，始终只有一个进行中项。

> 本步须等用户对"选法对照表 + plan 清单"确认或调整后，再进入第2步。用户可增删审查结论、改换某条的选用手法；据其反馈更新清单。

### 第2步：一、修改说明（必做交付物）

**仅输出第一节正文。**

#### 有修改时

1. 写明修改动作（如：将原权利要求X的附加技术特征并入原独立权利要求1）
2. 完整引用所添加或替换的文字（引号给出）
3. 声明未超范围（专利法第33条）
4. 声明针对审查意见修改（细则第51条）

句式参考：

> 1、申请人对原独立权利要求1进行了修改：根据原权利要求【X】的附加技术特征，在原独立权利要求1中添加以下内容：
> “【完整新增文字】”
> 从而形成新的权利要求1。
>
> 上述修改未超出原权利要求书和说明书所记载的范围，符合专利法第33条的规定。
>
> 申请人的上述修改，是针对审查意见通知书中关于权利要求【…】不符合创造性所作出，因此，上述修改也符合专利法实施细则第51条的相关规定。

#### 无修改时

仅输出简短“坚持原权利要求”说明，不虚构已修改内容。

### 第3步：二、针对审查意见核心观点的逐条答复（范式A，逐条为独立子项）

本步不是一次写完整节，而是**按第1步清单中的审查结论逐条撰写**，每条对应 plan 中一个子项（第3步-①、第3步-②……），**逐条交付、逐条确认**；写某条时把该子项置为进行中。这是与旧版最大的不同，务必落实“针对结论性观点针对性作答”。

**每条结论的撰写结构：**

```text
第一、针对审查意见指出的：【原文引述或忠实转述该条审查结论】
对此，申请人答复如下：
其一、【本条在第1步选定的手法①，紧扣该结论】
其二、【选定的手法②】
其三、【选定的手法③】
（视需要"进一步地/退一步来讲/再退一步来讲"层层递进）
```

要求：

- **只用第1步为本条选定的手法**：严格按第1步"选法对照表"里为该条挑定的 2–4 种手法展开，**不临时增用未选手法、也不遗漏已选手法**；若撰写中发现需增减手法，先回到第1步更新对照表并请用户确认，再动笔。
- **先引述再驳斥**：先用“针对审查意见指出的：”客观引述审查员观点（可压缩但不得歪曲），再“对此，申请人答复如下：”展开，使答复"有的放矢"。
- **善用递进反驳**：对“结合显而易见”类强结论，采用“退一步来讲……再退一步来讲……”层层设防（即使公开该思想→从思想到方案仍有鸿沟；即使结合→冲突无规则/存在结合障碍）。
- **不要在此处编造**对比文件未记载内容；引用对比文件段落须为用户材料所有。
- 全部结论逐条写完并确认后，第3步整体标记完成，进入第4步。

### 第4步：三 2.1 区别技术特征 + 实际解决的技术问题（范式B，必做交付物）

**仅输出创造性节的 2.1**（可含“三、（新的）权利要求1具备……创造性”总标题 + “2.1、……”小标题）。

固定顺序：

1. **列出区别技术特征**（区别特征（1）（2）…）：相对最接近现有技术（通常对比文件1），（新的/修改后）独权至少还具有哪些特征；用语贴近权利要求，可压缩但不得删掉构成完整方案所必需的步骤/数据关系链条。可注明该区别特征系“论证得出的区别 + 审查员认同的区别 + 并入的原从权特征”之来源构成。
2. **对照最接近现有技术说明“未公开何处”**。
3. **归纳实际解决的技术问题**：由区别特征客观导出，具体、可检验。
4. **纠正审查员过度上位的问题界定**（若有）：指出上位界定会使“任何相关改进都无创造性”，反证其不当，给出更贴切的具体问题表述。该点若已在第3步“问题界定型”结论中展开，此处凝练呼应即可。

区别特征、技术问题、后文效果须同一逻辑链。

### 第5步：三 2.2 非显而易见性（必做交付物）

**仅输出 2.2。** 按**首先→其次→再次→最后**四层：**单独不足 → 其他不能补 → 结合无启示 → 反驳常规步骤定性**；对比文件多于两篇时扩展“其次”。

- 第一层：最接近对比文件单独不能给出独权方案（技术目的、公开层次、未公开的完整链条、未试图解决对应问题）。
- 第二层：其他对比文件不能弥补（区分上位思想 vs 具体方案；拆开“列举可选项”与“实施例实际采用手段”；映射/结构来源不同不能移植）。
- 第三层：结合后仍不能自然得到（连续改造清单；整体判断；禁止事后拼贴）。
- 第四层：反驳“常规步骤/公知常识”（适用时）：并入内容是完整过程/结构配合而非单一上位标签，要求按修改后独权整体重评。

**与第3步的分工（重要）**：第3步已针对审查员**具体结论**逐条驳斥；第5步是本申请非显而易见性的**体系化正面归纳**。凡第3步已充分展开的点，此处不逐字重复，用“如本意见第二节所述……”指引，再补足体系性总结。若审查意见为纯结论驳斥打法且第3步已很充分，本步可大幅精简为一段收束。

### 第6步：三 2.3 有益效果 / 显著的技术进步 + 创造性节收束（必做交付物）

**仅输出 2.3 + 收束句。**

- 按独权特征的数据流/结构顺序复述闭环（先…再…从而…）。
- 效果必须挂在独权特征上，并与 2.1 的实际技术问题镜像对应；可分“区别特征（1）带来…”“区别特征（2）带来…”对应展开。
- 强调该效果非最接近现有技术的简单公开方式所能直接体现，必要时点明“取得预料不到的技术效果”。

收束句式：

> 综上，本申请（新的/修改后的）权利要求1所保护的技术方案，具有突出的实质性特点和显著的进步，具备创造性，符合专利法第22条第3款的规定。

（有无修改统一用语，与全文一致。）

### 第7步：四、其他权利要求（必做交付物）

**仅输出第四节正文。**

1. 概括：从属权利要求直接或间接引用独立权利要求，独权具备创造性的，从权亦具备创造性。
2. 择要：对通知书重点攻击或确有附加区别的从权，用“值得注意的是：”单独展开。
3. 无额外争点时概括一段即可，勿注水。

### 第8步：组装完整意见陈述书（必做交付物）

**仅当第2–7步均已经用户确认（或确认后的修订稿）之后**执行。按序拼接：

## 反驳手法工具箱（第1步据此选法，第3步、第5步取用）

下列手法从实务范例提炼，**按审查结论类型索引**，是一份**备选清单而非必用清单**——第1步据本案审查结论从中**挑选**贴合的少数几种，未选中的不出现在答复里。每条给出适用情形与论证要点；正式撰写须替换为本案真实对比文件与权利要求用语，不得照抄示例措辞。

### A. 针对“结合动机 / 显而易见”类结论

1. **不排除 ≠ 有动机**：对比文件“没有排除某手段”不等于“教导 / 有动机得到本申请方案”，二者不同层次；把前者直接推为后者，是以本申请为指引的事后分析。
2. **反对事后分析（事后诸葛）**：审查员在已知本申请后，将完整方案拆成局部动作再分别评价、或以本申请思路重新解释现有技术；应回到申请日前、以现有技术自身的客观教导判断本领域技术人员是否会想到。
3. **正常研发路径叙事**：研发是“先发现问题、再设计方案”。审查员把问题发现描述得过于上位，再反向说结合显而易见，颠倒了顺序；应还原发明人真实的问题—方案发现过程。
4. **根本差异 → 相反教导 / 互斥方案（teaching away）**：从检测/工作**目标**、**原理**、**方案与条件**三层指出与对比文件的根本不同，甚至互斥（采用对方的必要步骤会破坏本发明赖以成立的特征），因此本领域技术人员没有动机由对比文件出发得到本申请。
5. **自洽闭环、缺失动机与核心功能**：对比文件已是自洽闭环、能独立完成其既定目标，本领域技术人员没有动机再引入额外环节（会增成本、降效率，违背其发明初衷）；且对比文件缺失本申请的核心功能（如触发/推送/判定机制）。

### B. 针对“实际解决的技术问题”界定类结论

6. **纠正技术问题过度上位**：审查员把实际技术问题界定得过宽，掩盖本申请具体问题的特殊性；若如此上位，任何涉及该方向的改进都将无创造性——以此反证界定不当，并给出更贴切、更具体的技术问题表述。

### C. 针对“简单迁移 / 常规调整”类结论

7. **反对过度概括对比文件**：审查员抽掉对比文件的关键限定要素（如具体设备类型、覆盖范围、计算逻辑、输出形式），只留一个上位“核心思想”，再以本申请思路重新解释——属对公开内容的过度概括，应把被抽掉的限定逐一还原。
8. **相反方向 / 无逆向动机**：对比文件教导朝某方向（更贵、更全、更精），本申请恰朝相反方向（更省、更简、更高效）；本领域技术人员从对比文件获得的教导不会产生走向本申请的逆向动机。

### D. 针对“公知常识 / 跨领域可迁移”类结论

9. **领域可迁移性反驳**：处理对象/场景不同（如规整独立单元 vs 受环境干扰的连续图谱），且要解决的**技术问题不同**；本申请的专属改进（自适应迭代、双条件判定、连续区域聚合等）并非通用方法的简单变型，对比文件未公开该专属改进。

### E. 递进式（arguendo）层层退让反驳

10. **退一步：从思想到方案的鸿沟**：即使对比文件公开某上位思想，从思想到本申请具体方案仍存在一系列无教导的技术难题（选哪些对象、如何简化、如何保证可靠性），这些在对比文件中均无明示或暗示。
11. **再退一步：结合后冲突无规则**：即使强行结合两手段，当二者结论冲突时（如手段甲判“有病害”而手段乙判“正常”）应以何为准？对比文件均未提供冲突判定规则。
12. **结合障碍：违背对比文件初衷**：结合会显著增加成本、降低效率，直接违背对比文件所追求的减负增效初衷，故客观上存在结合障碍。

> 使用建议：C/D 类结论常配 7/8/9 正面拆解，再用 E 类（10→11→12）层层设防；A 类结论常配 1/2/3 破除动机，再用 4/5 指出相反教导与缺失动机；B 类结论用 6 单独成段并与 2.1 的技术问题呼应。

## 微型示例（虚构，仅示范式与语气；勿当作组装全文）

**范式A（第3步·逐条驳斥，示 A 类结论）：**

> 第一、针对审查意见指出的：本领域技术人员有动机在对比文件1基础上引入另一检测手段进行复核，属简单叠加。
> 对此，申请人答复如下：
> 其一、对比文件1“未排除其他手段”并不等于“有动机得到本申请方案”，将前者直接推为后者，属以本申请为指引的事后分析。
> 其二、对比文件1是自洽闭环的单级方案，已能独立完成其既定目标，本领域技术人员没有动机再引入额外环节；且其检测目标、原理与本申请存在根本差异，采用其必要步骤反而会破坏本申请赖以成立的条件，二者实为互斥方案。
> 其三、退一步，即便产生结合念头，从该上位思想到本申请具体方案仍存在选区、简化、可靠性保证等一系列无教导的技术难题。

**范式B（第4步·区别+问题，示纠正上位）：**

> 审查意见将实际解决的技术问题界定为“如何提高检测准确性”，界定过度上位，掩盖了本申请具体问题的特殊性——若如此上位，任何涉及检测改进的方案都将无创造性。本申请实际解决的技术问题应为：如何在减少……工作量的同时，提高对……的判定可靠性。
