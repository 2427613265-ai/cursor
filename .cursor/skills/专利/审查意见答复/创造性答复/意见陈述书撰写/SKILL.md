# 创造性答复意见陈述书撰写规范

本 skill 用于**分步**撰写（或按用户反馈逐步修订）针对创造性缺陷的意见陈述书。最终目标是完整可提交正文，但**禁止在同一次回复中一次性写出全文**。

## 核心：两种范式融合到同一段论证（务必理解）

实务中有两种论证素材：

- **结论驳斥（针对审查意见原文作答）**：完整照录审查员的结论性论断，再针对性拆解。
- **创造性要点（正面立论）**：区别技术特征、实际技术问题、结合动机、技术启示、非显而易见、有益效果。

**本 skill 要求的融合，不是"先写一节逐条驳斥、再写一节三段论"这种分块拼接**（那只是把两份模版并排放在同一文档里，融合度不够）。而是**在回应每一条审查意见的同一段论证里，把上述创造性要点就地交织进去**，形成一条"照录审查观点 → 指出其忽略/未公开的区别技术特征 → 由该特征界定或纠正实际技术问题 → 论证无结合动机/无技术启示/相反教导（即非显而易见）→ 就地点出该特征带来的有益效果"的连贯论证链。

也就是说：**区别特征、技术问题、非显而易见、效果不再各占一个独立小节，而是分散融进对每一条审查意见的答复段中**，谁的靶子就在谁那里连立论带反驳一次讲透。

### 融合论证段模板（每一条审查意见对应一段，这是全篇的主体）

```text
第【序】、针对审查意见指出的：【完整照录该条审查意见原文，不得压缩概括】
对此，申请人答复如下：
其一、（区别特征 + 技术问题）审查意见忽略了，（新的/修改后）权利要求1相对对比文件还限定了【区别技术特征】，该特征在本申请中【作用】，据此本申请实际解决的技术问题是【具体问题】，而非审查意见所概括的【过宽问题】——若如此上位，任何相关改进都将无创造性。
其二、（无动机/无启示/相反教导＝非显而易见）就审查员所称的结合动机/技术启示而言，对比文件实际教导的是【……】，与本申请【根本差异/相反方向/自洽闭环】，本领域技术人员没有动机、也得不到得到本申请的技术启示；退一步，即使结合，【从思想到方案的鸿沟/冲突无规则/结合障碍】。
其三、（有益效果收束）正因上述区别技术特征，本申请能够【有益效果，与其一的技术问题镜像对应】，此非对比文件单独或简单结合所能达到。
```

> 上面“其一/其二/其三”只是承载不同要素的锚点，可按本案审查逻辑合并或增删，但**同一段里必须同时出现区别特征、技术问题、非显而易见论证、效果**，让立论与驳斥交织，而不是各自成节。其二中选用哪些反驳手法，取自第1步的"选法对照表"（并非每种都用）。

### 全篇主体与辅助部分的关系

- **主体**：上述"逐条融合论证段"构成创造性论述的主体（对应审查意见有几条核心论断就有几段）。
- **可选立论开篇（简明）**：主体之前可先用一小段总述"相对最接近现有技术的区别技术特征来源（审查员认同的 + 论证得出的 + 并入的）+ 由此界定的实际技术问题"，作为各融合段共同的立论基础；但**创造性的展开论证仍在各融合段内完成，开篇只作简明铺垫，不展开成三段论**。
- **收束**：全部融合段之后，用一段小结整体归纳"本申请方案整体非显而易见、具有显著进步、具备创造性"。
- **修改说明（情形一）**：作为独立首节，见下文情形分支。

## 情形分支：是否修改权利要求

撰写前先判明本案属哪种情形（第1步与用户确认）。两种情形**都采用上述"逐条融合论证"主体**，区别只在是否有修改说明、以及立论开篇的详略：

- **情形一｜本次修改了权利要求（参照岩芯模版）**：先写「一、修改说明」（并入来源 + 完整新增文字 + 法33条 + 细则51条）；创造性论述的立论开篇可稍充实（把并入特征作为区别特征讲清），主体仍为逐条融合论证。
- **情形二｜本次未修改权利要求（参照路面模版）**：无「修改说明」，仅在开头一句“申请人坚持原权利要求”；直接进入逐条融合论证主体，立论开篇从简。

## 答复要点：核心必达 + 视情选用（不是每项都要覆盖）

**A. 核心必达（每一份创造性答复整体上都应覆盖，且应分散融进各融合段）：**

1. **区别技术特征**（相对最接近现有技术，本申请/修改后独权还有哪些特征）
2. **实际解决的技术问题**（由区别特征客观导出；审查员过度上位时予以纠正）
3. **非显而易见性**（结论必达，其展开层次可深可浅）
4. **有益效果 / 显著的技术进步**（挂在独权特征上，与技术问题镜像对应）
5. **修改说明**（仅**情形一**必达：并入来源、新增文字、法33条、细则51条）

**B. 视情况选用（按审查意见与案情挑用，不必全覆盖、不硬套）：**

- **结合动机**（无动机 / 结合障碍）——审查意见以“有动机结合/改造”评述时才用
- **技术启示**（对比文件单独或结合未给出得到本申请的启示）——视审查逻辑选用
- **相反技术教导 / teaching away**——**仅当本案确有相反教导**时才用
- 工具箱其余手法（过度概括、事后分析、正常研发路径、领域可迁移性反驳、退一步/再退一步等）——按第1步“选法对照表”选定，用得上才写

> 判断原则：B 档是"打靶工具"，靶子（审查员的具体论断）在哪就用哪几种，无对应靶子硬写反而答非所问、稀释说服力。

## 硬性流程规则（最高优先级）

1. **禁止一次生成答复全文**：不得在材料刚齐或用户只说“写意见陈述书”时，一次性写出开场、各段与结尾。
2. **先分析选法、再落笔**：拿到审查意见后，**第一件事是分析**——把审查员的结论性论断逐条摘出，为每条**挑选**将要使用的反驳手法。**并非工具箱里每种手法都要用**，只选真正贴合本案该条结论的 2–4 种。选法结论须先交用户确认。
3. **用 plan/待办清单驱动全过程**：确定选法后，**必须用 plan（待办清单）功能**把本案答复的**全部步骤与子项**（每一条审查意见的融合论证段各是一个子项、并标注为其选定的手法）一次性列出，让用户看清"总共几步、每步做什么、当前写到哪一步"。此后**每完成一步就更新清单状态**，始终只有一个"进行中"。这份清单是用户跟踪进度的主要依据，不得省略。
4. **一步只做一个交付物**：每轮对话只完成清单中的**当前一步**（通常是一个融合论证段），输出后**停下**，请用户确认、修改或补充。
5. **须等用户确认再进入下一步**：未获用户对本步的确认（或明确修改指示并完成修订）前，不得擅自撰写后续段落，也不得把清单后续项标记为已完成。
6. **须按用户提示推进**：每一步都以用户当轮反馈为准撰写或修订；用户没有发话推进时不自行跳步。
7. **用户要求“一口气写完”时仍分步**：说明本 skill 强制分步，从清单未完成的最早一步开始；仅在用户走完各步确认后，才在最后一步组装全文。
8. **修订也按步**：用户只改某段时只重写该段，不连带重写无关段落，更不重出全文（除非已进入组装步）。

每步输出格式：

- 开头标明：`【第X步 / 共N步】步骤名称`（N 为本案 plan 清单实际条数，随审查意见条数而变）
- 正文为该步交付物
- 结尾请用户确认（可直接改字），确认后再进入下一步，并**一句话预告**下一步
- 同时更新 plan 清单状态

## 适用场景

- 审查意见指出权利要求不符合专利法第22条第3款（创造性），需要书面答复。
- 适用于第N次审查意见通知书；**通知书次数、发文日、申请号一律按用户材料填写**。
- 不区分结构类/方法类/算法类。

不适用（另用对应专题 skill；尚无 skill 时如实说明并按常规法理答复）：

- 仅涉及新颖性、修改超范围、不清楚、得不到说明书支持、单一性等问题，且不夹带创造性评述。
- 通知书同时评述创造性与其他问题时：创造性部分按本 skill；其他问题单独分节。

## AI 能力边界

- **不得编造**对比文件未记载的技术内容、段落号、效果或“本领域公知常识”的具体出处。
- 不得在已知本申请方案后，把本申请的具体限定**事后拆开再贴回**对比文件当作“启示”（这既是要反驳审查员的手法，也是自己不能犯的错）。
- 创造性评估依赖用户提供的对比文件信息；信息不足时只论证已能支撑的部分，并列出待核验引用。

## 分步流程（必须按序执行）

步骤总数 N 不固定：逐条融合论证会按审查意见的核心论断条数拆成若干子项，一并计入 plan 清单。骨架步骤：

```text
第1步 材料核对 + 审查意见结论清单 + 逐条选定反驳手法 + 判明情形 →（据此建立 plan 清单）
  ↓ 用户确认（选法 + 清单）
第2步 撰写「修改说明」（情形一）/「坚持原权利要求」一句（情形二）
  ↓ 用户确认
第3步 撰写创造性论述的「立论开篇」（简明：区别特征来源 + 实际技术问题）
  ↓ 用户确认
第4步-①/②/③… 逐条撰写「融合论证段」（每条审查意见一段，段内交织区别特征/问题/非显而易见/效果）
  ↓ 每段各自确认
第5步 撰写创造性小结（整体非显而易见 + 显著进步）
  ↓ 用户确认
第6步 撰写「其他权利要求」
  ↓ 用户确认
第7步 组装开场 + 各部分 + 结尾 → 完整意见陈述书 + 检查清单
```

### 第1步：材料核对 + 审查意见结论清单 + 逐条选定反驳手法 + 建立 plan 清单（必做交付物）

写任何正文前，先只输出本步交付物（分析与清单）。**这是"先分析后撰写"的分析环节，最关键。**

**必收材料**（缺一则本步只追问）：

1. 审查意见通知书全文，或至少含对比文件编号、评述逻辑、针对哪些权利要求
2. 对比文件要点（援引段落/图号及公开内容）；仅有摘要时后续标注“据用户摘录，段落号待核”
3. 当前权利要求书（通知书所针对版本）
4. 拟修改方案：并入哪些特征、来自何处；或明确坚持不改

**本步交付物：**

1. **材料齐备情况表**（已有 / 缺失）；材料不足时只列需补充的问题，结束本轮。
2. **审查意见结论清单**：把审查员结论性论断逐条摘出并分类（结合动机型 / 问题界定型 / 常规公知型 / 教导忽略型）。
3. **逐条选定反驳手法（核心）**：对**每一条**审查结论，从工具箱（A–E、共12种）中挑 2–4 种最贴合的手法，"手法编号 + 一句话为何适用本案"。**不是每种都用**。并标注该条融合段里要就地讲哪个区别技术特征、对应哪个技术问题与效果。用对照表输出：

   | 审查结论 | 类型 | 选用手法 | 该段就地讲的区别特征 / 问题 / 效果 |
   |---------|------|---------|--------------------------------|
   | 结论① … | 结合动机型 | 1 不排除≠动机；4 相反教导；10 退一步 | 区别特征X / 问题P / 效果E |
   | 结论② … | 问题界定型 | 6 纠正过度上位 | … |

4. **判明情形**（情形一/二）并说明立论开篇详略、是否并入哪条从权、创造性主线一句话。
5. **建立 plan 待办清单**：调用 plan/待办清单功能列出完整步骤，让用户看清全貌与进度。清单形如：

   - [ ] 第1步 分析与选法（本步）
   - [ ] 第2步 修改说明 / 坚持原权
   - [ ] 第3步 立论开篇（区别特征 + 技术问题）
   - [ ] 第4步-① 融合论证段·驳结论①（手法 1/4/10，讲区别特征X）
   - [ ] 第4步-② 融合论证段·驳结论②（手法 6）
   - [ ] 第4步-③ 融合论证段·驳结论③（手法 7/8/E）
   - [ ] 第5步 创造性小结
   - [ ] 第6步 其他权利要求
   - [ ] 第7步 组装全文 + 检查清单

   此后每步置"进行中"/"已完成"，始终只有一个进行中项。

> 本步须等用户对"选法对照表 + plan 清单"确认或调整后再进入第2步。

### 第2步：修改说明 / 坚持原权（必做交付物）

**情形一（有修改）**，仅输出修改说明正文：

> 1、申请人对原独立权利要求1进行了修改：根据原权利要求【X】的附加技术特征，在原独立权利要求1中添加以下内容：
> “【完整新增文字】”
> 从而形成新的权利要求1。
> 上述修改未超出原权利要求书和说明书所记载的范围，符合专利法第33条的规定。
> 申请人的上述修改，是针对审查意见通知书中关于权利要求【…】不符合创造性所作出，因此，上述修改也符合专利法实施细则第51条的相关规定。

**情形二（无修改）**：仅输出一句“申请人经研究后坚持原权利要求，理由陈述如下”，不虚构修改。

### 第3步：立论开篇（简明，必做交付物）

**仅输出创造性论述的开篇一小段**：概述相对最接近现有技术的**区别技术特征**（可标来源：审查员认同的 + 论证得出的 + 并入的）与由此界定的**实际解决的技术问题**（若审查员过度上位，在此点出并给出更贴切表述）。

- **保持简明**：这是各融合段的共同基础，不在此展开非显而易见四层，也不在此详述效果——那些留到各融合段内交织完成。
- 顺序：**先区别特征、再技术问题**，不可颠倒。

### 第4步（拆为多子项）：逐条融合论证段（全篇主体，必做交付物）

**按审查意见的核心论断逐条撰写，每条一个融合段、一个 plan 子项，逐段交付确认。** 每段严格用前文“融合论证段模板”：

1. `第【序】、针对审查意见指出的：` 后**完整照录该条审查意见原文全文**（公开认定、作用认定、启示认定、结合动机等整段论述链），**不得压缩、概括、改写或只摘结论句**；仅可去除通知书版式套语、按条拆分成段，语义与论证链须与原文一致。
2. `对此，申请人答复如下：` 后，**在同一段内交织**（这是本 skill 的关键）：
   - **区别技术特征**：指出该审查结论忽略/未公开的、本申请或修改后独权所限定的区别特征；
   - **实际技术问题**：由该区别特征就地界定/纠正问题；
   - **非显而易见**：用第1步为本条选定的手法，论证无结合动机、无技术启示、或相反教导；强结论用“退一步来讲……再退一步来讲……”层层设防；
   - **有益效果**：收束时点出该区别特征带来的效果，与本段技术问题镜像对应。
3. **只用第1步为本条选定的手法**，不临时增删未经确认的手法；需增减先回第1步更新对照表并请用户确认。
4. **不编造**对比文件未记载内容；引用段落须为用户材料所有。

全部融合段写完并确认后，第4步整体完成，进入第5步。

## 反驳手法工具箱（第1步据此选法，第4步各融合段取用）

下列手法**按审查结论类型索引**，是**备选清单而非必用清单**——第1步据本案挑选贴合的少数几种，未选中的不出现在答复里。正式撰写须替换为本案真实对比文件与权利要求用语。

### A. 针对“结合动机 / 显而易见”类结论

1. **不排除 ≠ 有动机**：对比文件“没有排除某手段”不等于“教导 / 有动机得到本申请方案”；把前者直接推为后者，是以本申请为指引的事后分析。
2. **反对事后分析（事后诸葛）**：审查员在已知本申请后，将完整方案拆成局部动作再分别评价、或以本申请思路重新解释现有技术；应回到申请日前、以现有技术自身教导判断。
3. **正常研发路径叙事**：研发是“先发现问题、再设计方案”；审查员把问题发现描述得过于上位，再反向说结合显而易见，颠倒了顺序。
4. **根本差异 → 相反教导 / 互斥方案（teaching away）**：从目标、原理、方案与条件三层指出与对比文件的根本不同，甚至互斥（采用对方必要步骤会破坏本发明），故无动机由对比文件出发得到本申请。
5. **自洽闭环、缺失动机与核心功能**：对比文件已自洽闭环、能独立完成其目标，没有动机再引入额外环节（增成本降效率、违背其初衷）；且缺失本申请核心功能。

### B. 针对“实际解决的技术问题”界定类结论

6. **纠正技术问题过度上位**：审查员界定过宽、掩盖本申请具体问题特殊性；若如此上位任何相关改进都将无创造性——反证其不当，并给出更贴切的具体问题。

### C. 针对“简单迁移 / 常规调整”类结论

7. **反对过度概括对比文件**：审查员抽掉对比文件关键限定要素，只留上位“核心思想”再以本申请思路重解释；应把被抽掉的限定逐一还原。
8. **相反方向 / 无逆向动机**：对比文件教导朝某方向（更贵、更全、更精），本申请恰朝相反方向（更省、更简、更高效）；不会产生走向本申请的逆向动机。

### D. 针对“公知常识 / 跨领域可迁移”类结论

9. **领域可迁移性反驳**：处理对象/场景不同、要解决的技术问题不同；本申请的专属改进并非通用方法的简单变型，对比文件未公开该专属改进。

### E. 递进式（arguendo）层层退让反驳

10. **退一步：从思想到方案的鸿沟**：即使公开某上位思想，从思想到本申请具体方案仍有一系列无教导的技术难题。
11. **再退一步：结合后冲突无规则**：即使强行结合，两手段结论冲突时以何为准？对比文件均未提供冲突判定规则。
12. **结合障碍：违背对比文件初衷**：结合会增成本、降效率，违背对比文件追求的减负增效初衷，故存在结合障碍。

> 使用建议：C/D 类常配 7/8/9 再用 E（10→11→12）设防；A 类常配 1/2/3 破动机再用 4/5 指相反教导；B 类用 6 并与技术问题呼应。这些手法都在**融合段的“其二（非显而易见）”里就地展开**，与该段的区别特征、效果连成一体。

## 微型示例：一个真正融合的论证段（虚构，勿当作组装全文）

> 第一、针对审查意见指出的：对比文件2公开了基于温湿度计算体感/舒适评价值并据此调节光源色温，且上述技术特征在对比文件2与本申请中所起的作用相同，都是通过检测环境温湿度来量化人体体感温度，并基于此对灯光色温进行自动调节以提高舒适度；因此对比文件2给出了将上述基于温湿度体感评价来调节灯光色温的技术手段应用于对比文件1以解决其技术问题的启示；在对比文件1已公开基于照度闭环调节亮度的基础上，本领域技术人员在面对如何进一步提升室内环境舒适度这一技术问题时，有动机结合对比文件2的教导，将色温调节功能与对比文件1的照度调节功能相结合。
>
> 对此，申请人答复如下：
>
> 其一、（区别特征 + 技术问题）审查意见的上述认定忽略了修改后权利要求1的关键区别技术特征：本申请并非简单地"检测温湿度→调色温"，而是限定了【……以照度与色温联合闭环、按預設优先级协调二者的具体调节逻辑……（区别技术特征）】。正因该特征，本申请实际解决的技术问题是【如何在照度调节与色温调节相互耦合时避免二者冲突、实现稳定协调的联合调光（具体问题）】，而非审查意见概括的“进一步提升室内环境舒适度”这一过宽问题——若将问题如此上位，则任何调光调色改进都将当然无创造性，显属不当。
>
> 其二、（无动机 / 无启示 / 相反教导＝非显而易见）就审查员所称“作用相同、故有启示与动机结合”而言，对比文件2实际教导的是【单一地由体感值直接映射色温】，其并未涉及照度与色温的耦合协调，本领域技术人员从中得不到得到本申请上述联合闭环逻辑的技术启示；相反，对比文件1的照度闭环与对比文件2的色温映射分别独立运作，二者简单叠加恰会在耦合处产生冲突，构成结合障碍。退一步讲，即使本领域技术人员产生结合念头，从“分别调节”这一上位思想到本申请“按优先级协调的联合闭环”仍存在无教导的鸿沟；再退一步，即便勉强结合，当照度目标与色温目标相互牵制时以何者为准，对比文件1、2均未给出任何规则。
>
> 其三、（有益效果收束）正因上述区别技术特征，本申请能够在同时调节照度与色温时避免二者相互干扰、输出稳定协调的照明，这一效果非对比文件1、2各自公开的独立调节所能达到，也非将二者简单叠加所能当然获得。

注意：上例把**区别特征、技术问题、结合动机、技术启示、相反教导、非显而易见、有益效果**全部融进了对“同一条审查意见”的答复段中，而不是分成“驳斥节”和“三段论节”。正式案件须替换为真实对比文件与权利要求用语。
