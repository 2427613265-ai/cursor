# 创造性答复意见陈述书撰写规范

本 skill 用于**分步**撰写（或按用户反馈逐步修订）针对创造性缺陷的意见陈述书。最终目标仍是完整可提交正文，但**禁止在同一次回复中一次性写出全文**。

## 硬性流程规则（最高优先级）

1. **禁止一次生成答复全文**：不得在材料刚齐或用户只说“写意见陈述书”时，就把开场、第一至三节、结尾一次性全部写出。
2. **一步只做一个交付物**：每一轮对话只完成下面“分步流程”中的**当前一步**，输出该步标注明确的交付物，然后**停下**，请用户确认、修改或补充。
3. **须等用户确认再进入下一步**：未收到用户对本步交付物的确认（或明确的修改指示并完成修订）之前，不得擅自撰写后续章节正文。
4. **用户要求“一口气写完”时仍按分步**：若用户要求一次出全文，应简要说明本 skill 强制分步推进，并从未完成的最早一步开始；只有用户走完各步确认后，才在**最后一步**组装全文。
5. **修订也按步**：用户只改某一节时，只重写该节；不要连带重写无关章节，更不要重出全文，除非用户明确要求进入组装步。

每步输出格式要求：

- 开头标明：`【第X步 / 共7步】步骤名称`
- 正文为该步交付物
- 结尾固定追问：请用户确认本步内容（可直接改字），确认后再进入第X+1步；并**一句话预告**下一步将做什么

---

## 适用场景

- 审查意见指出权利要求不符合专利法第22条第3款（创造性），需要书面答复。
- 适用于第N次审查意见通知书（一通、二通或更多次）；**通知书次数、发文日、申请号一律按用户材料填写**，不得沿用其他案件或模版中的次数表述。
- 不区分结构类/方法类/算法类；创造性论证逻辑通用。

不适用（应另用对应专题 skill，尚无 skill 时如实说明并按常规法理答复，勿硬套本骨架的全部节次）：

- 仅涉及新颖性、修改超范围、不清楚、得不到说明书支持、单一性等形式或其它实质问题，且不夹带创造性评述。
- 若通知书同时评述创造性与其他问题：创造性部分按本 skill 分步推进；其他问题在创造性各步之外单独分节，不强行塞进第二节。

## AI 能力边界

- **不得编造**对比文件未记载的技术内容、段落号、效果或“本领域公知常识”的具体出处。
- 不得在已知本申请方案后，把本申请的具体限定**事后拆开再贴回**对比文件，当作本领域技术人员申请日前已有的“启示”。
- 创造性评估依赖用户提供的对比文件信息；信息不足时只论证已能支撑的部分，并列出待用户补充核验的引用。

## 分步流程（必须按序执行）

```text
第1步 材料核对与答复策略确认
  ↓ 用户确认
第2步 撰写「一、修改说明」（或坚持不改的说明）
  ↓ 用户确认
第3步 撰写「2.1 区别技术特征 + 实际解决的技术问题」
  ↓ 用户确认
第4步 撰写「2.2 非显而易见性」（四层递进，本步内写完该节）
  ↓ 用户确认
第5步 撰写「2.3 显著的技术进步」+ 第二节收束句
  ↓ 用户确认
第6步 撰写「三、其他权利要求」
  ↓ 用户确认
第7步 组装开场 + 已确认各节 + 结尾 → 完整意见陈述书 + 检查清单
```

### 第1步：材料核对与答复策略确认（必做交付物）

开始任何章节正文前，先核对并**仅输出**本步交付物（清单 + 策略结论），不写意见书正文。

**必收材料**（缺一则本步只追问，不进入第2步）：

1. 审查意见通知书全文，或至少含对比文件编号、评述逻辑、针对哪些权利要求的摘要
2. 对比文件要点（审查员援引的段落/图号及公开内容）；若仅有摘要，后续论证须标注“据用户提供摘录，段落号待核”
3. 当前权利要求书（通知书所针对的版本）
4. 拟修改方案：并入哪些特征、来自哪条原从权/说明书何处；或明确坚持不改

**本步交付物：**

- 材料齐备情况表（已有 / 缺失）
- 建议的答复策略：是否修改独权、并入哪条从权的哪些特征、创造性抗辩主线（一句话）
- 若材料不足：只列需用户补充的问题，结束本轮

### 第2步：一、修改说明（必做交付物）

**仅输出第一节正文**（可含该节小标题），不含开场、不含第二节及以后。

#### 有修改时

1. 写明修改动作（例如：将原权利要求X的附加技术特征并入原独立权利要求1）
2. 完整引用所添加或替换的文字（引号给出）
3. 声明未超范围（专利法第33条）
4. 声明针对审查意见修改（细则第51条）

句式参考：

> 1、申请人对原独立权利要求1进行了修改：根据原权利要求【X】的附加技术特征，在原独立权利要求1中添加以下内容：
> “【完整新增文字】”
> 从而形成新的权利要求1。
>
> 上述修改未超出原权利要求书和说明书所记载的范围，符合专利法第33条的规定。
>
> 申请人的上述修改，是针对审查意见通知书中关于权利要求【…】不符合创造性所作出，因此，上述修改也符合专利法实施细则第51条的相关规定。

#### 无修改时

仅输出简短“坚持原权利要求”说明，不要虚构已修改内容。

---

### 第3步：2.1 区别技术特征与实际解决的技术问题（必做交付物）

**仅输出 2.1 节正文**（可含“二、…”总标题 + “2.1、…”小标题），不含 2.2、2.3。

固定顺序：

1. **列出区别技术特征**（区别特征（1）（2）…）：相对最接近现有技术（通常对比文件1），修改后独权至少还具有哪些特征；用语贴近权利要求，可压缩但不得删掉构成完整方案所必需的数据关系/步骤链条。
2. **对照最接近现有技术说明“未公开何处”**：承认已公开部分，指出未公开到本申请所要求的具体程度（例如仅“调用封装函数”，未公开完整处理过程）。
3. **归纳实际解决的技术问题**：由区别特征客观导出，具体、可检验；避免“如何得到更准确/更优结果”等过宽概括。
4. **必要时纠正审查员的问题概括**：若通知书概括过宽/过空，指出不足并给出申请人认为更恰当的问题表述。

区别特征、技术问题、后文效果须同一逻辑链。本步结束时请用户重点确认：区别特征列表与实际技术问题表述是否采纳。

---

### 第4步：2.2 非显而易见性（必做交付物）

**仅输出 2.2 节正文**。在标题“权利要求1所保护的技术方案是非显而易见的”下，按**首先→其次→再次→最后**四层写完；对比文件多于两篇时扩展“其次”，层次仍为：**单独不足 → 其他不能补 → 结合无启示 → 反驳常规步骤定性**。

本步虽含四层，仍算**一步一个交付物**（整节 2.2）。若用户明确要求按层拆开确认，则将本步临时拆为 4a–4d，每层确认后再写下层——以用户当场要求为准。

#### 第一层：最接近对比文件单独不能给出独权方案

- 概括其技术目的与公开层次；引用用户材料中有的段落/图
- 列出其未公开或未启示的完整限定链条
- 点明其未发现或未试图解决与区别特征对应的具体问题

#### 第二层：其他对比文件不能弥补

- 区分上位思想与具体方案
- 拆开“概述列举的可选项”与“实施例实际采用手段”
- 若映射/结构来源不同，说明不能直接移植

#### 第三层：结合后仍不能自然得到

- 分述以 D1 为起点会得到什么、参考 D2 至多得知什么
- 给出**连续改造清单**
- 强调整体判断；禁止事后拆开再拼贴为“启示”

#### 第四层：反驳“常规步骤/公知常识”（适用时）

- 说明原从权/并入内容是完整过程或结构配合，而非单一上位标签
- 要求按**修改后独权整体**重评，不得沿用对原从权的“常规步骤”定性直接否定新独权

不适用时可省略第四层，但前三层必须完整。

---

### 第5步：2.3 显著的技术进步 + 第二节收束（必做交付物）

**仅输出 2.3 正文 + 第二节收束句**，不写第三节、不开场结尾。

- 按独权特征的数据流或结构顺序复述闭环
- 效果必须挂在独权特征上，并与第3步确认的实际技术问题镜像对应
- 说明该效果非最接近现有技术的简单公开方式所能直接体现

收束句式：

> 综上所述，修改后的权利要求1具有突出的实质性特点和显著的进步，具备创造性，符合专利法第二十二条第三款的规定。

（无修改时删去“修改后的”等用语，与全文一致。）

---

### 第6步：三、其他权利要求（必做交付物）

**仅输出第三节正文。**

1. 概括：从属权利要求直接或间接引用独立权利要求，独权具备创造性的，从权亦具备创造性。
2. 择要：对通知书重点攻击或确有附加区别的从权，用“值得注意的是：”单独展开。
3. 无额外争点时概括一段即可，勿注水重复第二节。

---

### 第7步：组装完整意见陈述书（必做交付物）

**仅当第2–6步均已经用户确认（或用户确认的修订稿）之后**，才执行本步。

将下列部分按序拼接为**一份完整意见陈述书**：

1. 开场（称呼 + 收到第N次通知书 + 引出答复）
2. 第2步已确认的第一节
3. 第3–5步已确认的第二节（含 2.1、2.2、2.3 与收束）
4. 第6步已确认的第三节
5. 结尾客套 + 此致

开场句式：

> 申请人已收到贵局于【发文日】发出的就专利申请号为【申请号】发明专利申请的第【N】次审查意见通知书，感谢您对本申请认真负责的审查。
>
> 申请人认真研究了审查员的审查意见，答复如下：

结尾句式：

> 综上所述，本申请符合专利法和专利法实施细则的规定，敬请审查员在考虑上述意见陈述后能加快审查程序，批准本申请为发明专利为盼。如果本次意见陈述不能说服审查员授予本申请专利权，恳请审查员再次给予答复机会。最后，衷心感谢审查员的用心审查。
>
> 此致
>
> 衷心的感谢！

联系电话等署名：仅在用户提供时写入。

本步输出后，附上检查清单自检结果；若清单有未通过项，先列出并请用户决定是否回到对应步骤修改，不要默默忽略。

## 全文骨架（组装时用，勿在早期步骤整篇输出）

```text
尊敬的审查员：
您好！
开场

一、修改说明
二、…具备创造性
  2.1 区别技术特征 + 实际解决的技术问题
  2.2 非显而易见性（四层递进）
  2.3 显著的技术进步
  小结
三、其他权利要求具备创造性

结尾客套
此致
```

## 微型示例（虚构，仅示 2.2 语气；勿当作第7步全文）

> 对比文件1公开了通过人机交互选取目标区域四角点，并调用图形库透视变换函数完成校正。该文件将像素重映射整体封装于所述函数中，未公开“由目标矩形尺寸创建输出图像矩阵→对变换矩阵求逆→逐目标像素反求源点浮点坐标→分解整数坐标与小数偏差→按邻域加权赋值”的完整数据处理链条。
>
> 对比文件2虽在一般意义上列举了若干插值可选方式，但其实施例实际采用的是另一类插值，且坐标映射关系由成像姿态参数建立，与本申请基于源四边形与目标矩形点对点映射再求逆的路径不同。对比文件2至多给出“非整数源坐标可插值”的上位思想，不能给出本申请限定的完整像素重建数据链。
>
> 将两篇结合，本领域技术人员至多得到“选取角点并调用函数校正 + 可选插值”的拼合，仍需在无明确启示下连续放弃封装式调用、展开求逆与邻域权重赋值等多重改造，不能在不付出创造性劳动的情况下得到修改后的权利要求1。

正式案件须替换为用户材料中的真实对比文件与权利要求用语。

## 第7步检查清单

- [ ] 通知书次数、发文日、申请号、对比文件编号与用户材料一致
- [ ] 第2–6步均已经用户确认后再组装，而非跳步一次生成
- [ ] 有修改时：修改来源、新增文字、法33条与细则51条均已写明
- [ ] 2.1 区别特征、实际技术问题、2.3 效果三者对应；必要时已纠正审查员过宽概括
- [ ] 2.2 按“单独→其他不能补→结合无启示→（如适用）反驳常规步骤”递进
- [ ] 未编造对比文件段落、效果或启示；摘要性引用已标明待核
- [ ] 未把本申请特征事后拆开拼贴为对比文件的结合启示
- [ ] 2.3 效果挂在独权特征上，非空泛赞美
- [ ] 第三节有概括；对重点从权有择要强化（若存在）
- [ ] 组装稿含开场与结尾，为完整可提交正文
