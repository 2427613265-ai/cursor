# 创造性答复意见陈述书撰写规范

本 skill 用于**分步**撰写（或按用户反馈逐步修订）针对创造性缺陷的意见陈述书。最终目标是完整可提交正文，但**禁止在同一次回复中一次性写出全文**。

结构来源与分工（提取模版，勿原案照抄）：

| 来源 | 角色 |
|------|------|
| 岩芯批量校正意见陈述书 | **一通**骨架：常规三段论 2.1→2.2→2.3；审查员观点在需要处嵌入 2.2 |
| 路面深层病害 OA2 意见陈述书 | **二通**骨架：先针对审查员核心观点逐条作答，再收束权利要求创造性 |
| 不合理审查逻辑及应对 | **手法库与组合逻辑**：如何拆上位、破事后分析、建鸿沟/失败预期、讲结合障碍与整体协同 |

## 通知书次数分支（硬性；先判一通/二通）

第1步必须先判明本次是**一通**还是**二通及以后**，再选正文重心。二者不是同一套篇幅分配。

### 一通｜以常规三段论为基础（岩芯模版）

正文重心 = **创造性三段论**；审查员结论性观点在需要直接交锋时**引述后嵌入**非显而易见性论证。

```text
一、修改说明 / 坚持原权利要求
二、修改后的（或本申请）权利要求1具备创造性
    2.1 区别技术特征 → 实际技术问题 → 与审查员概括对比
    2.2 非显而易见性（内嵌：引述审查员结论性观点 + 其一/其二拆解；
         并以 2.1 问题为锚完成技术启示否定；首先/其次/再次/最后）
    2.3 显著的技术进步（镜像 2.1 技术问题）
三、第二项独权 3.1/3.2/3.3（若有）
四、其他权利要求
```

要求：

1. **骨架不可改**：2.1 / 2.2 / 2.3 标题与顺序固定。
2. **引述嵌入 2.2**：凡审查员作出“有动机 / 有启示 / 作用相同 / 常规手段 / 简单叠加”等结论性论断，在 2.2 中按“针对审查意见指出的：……对此，申请人答复如下：”拆解；不另设与三段论并列的大节抢戏。
3. **手法服务三段论**：选法后按逻辑递进写入 2.2，最终落回“针对如 2.1 所述技术问题无技术启示”。

### 二通及以后｜以针对审查员观点作答为主（路面 OA2 模版）

一通已用三段论完整论证过一次；二通正文重心 = **针对本次通知书核心观点的逐条答复**，三段论节用于收束与固化修改后方案，避免再写一篇等长的“教科书式三段论复读”。

```text
一、针对第【N】次审查意见的核心观点作答如下：
    第一、针对审查意见指出的：【引述/概括】→ 对此答复（其一/其二…递进）
    第二、……
    第三、……
二、修改后的（或本申请）权利要求1具备创造性
    2.1 区别技术特征 → 实际技术问题 → 与审查员概括对比（仍须完整）
    2.2/2.3 可相对一通压缩：非显而易见性要点已在「一」展开的，
         用“如本意见第一部分针对审查意见第×点所述……”回引，
         补足尚未覆盖的启示否定与技术进步镜像即可
三、第二项独权（若有；同样“针对性为主 + 三段论收束”）
四、其他权利要求
```

要求：

1. **第一部分是主战场**：按审查员本次核心观点逐条交锋；每条内部手法递进（见下文“手法组合”），不得只堆并列口号。
2. **仍须有 2.1**：区别特征与实际技术问题（含与审查员概括对比）不可省——这是三步法第二步，也是后文启示锚点。
3. **禁止二通再写一通同构长文**：若「一」已充分完成启示/动机/鸿沟论证，2.2 作体系化收束即可；2.3 仍单独写技术进步并镜像 2.1 问题。
4. **二通新争点优先**：审查员新的概括、新的组合方式、对新修改特征的“常规手段”认定，必须在第一部分正面拆解。

### 逻辑递进与衔接（一通、二通共用，硬性）

无论一通还是二通，单条答复与全文均须**递进**，不得平铺。下列是**可选进阶层级**（按本案成立与否取舍），不是必须写满的清单：

```text
纠错问题界定/扳倒上位（若审查员问题上位）
  → 对比文件客观教导与本申请的真实差异（作用是否真相同）
  → 针对如 2.1 所述技术问题有无技术启示 / 有无结合动机
  →（仅当事实支持时）退一步：即便存在框架思想，从思想到本案具体方案仍有鸿沟
  →（仅当事实支持时）再退一步：结合后冲突无规则 / 违背对比文件初衷 / 失败预期
  →（仅当特征间确有前提/输入/约束关系时）整体性与协同：不可暴力拆分后拼贴
```

**取舍标准**：每一层必须能回答“删去本层，上一层结论是否仍缺一块关键论证”。若某层只是换说法重复、或靠夸张才能成立，删除该层。衔接检查：删去上一层后，下一层是否失去前提？若会，说明递进成立；若各层可任意调序，说明仍是并列堆砌，必须重排或删层。

## 核心结构（独权三段论组件；一通全量展开，二通收束复用）

独权创造性论述的固定三节（标题与顺序不可改）：

1. **2.1、**申请人认为，权利要求1与最接近的现有技术对比文件1相比，至少具有以下区别技术特征：
2. **2.2、**权利要求1所保护的技术方案是非显而易见的。
3. **2.3** 修改后的（或本申请）权利要求1的技术方案具有显著的技术进步

- **一通**：上述三节为正文主体；审查员观点的逐条答复**归入 2.2**。
- **二通**：上述三节为第二部分收束；审查员观点的逐条答复**前置为第一部分主战场**，2.2 回引第一部分并补漏。
- **其他独权、从属权利要求** + 开场/结尾。

**有益效果归属（硬性）**：凡 2.2 针对性答复中本来会写的“本申请能够……效果”“取得预料不到的技术效果”等，**一律放到对应独权的技术进步一节**（第一项独权为 2.3，第二项独权为 3.3）集中论述，不在逐条答复的“其一/其二/其三”里做效果收束。

**技术问题只在区别特征节展开（硬性）**：实际技术问题的界定、展开，以及与审查员问题概括的对比，**只在 2.1（或第二项独权的 3.1）完整论述一次**。进入 2.2/3.2 后，不得再次整段复述申请人的技术问题或再次展开与审查员概括的对比；若审查结论本身就是问题界定类，仅用一句“如 2.1 所述……”指向前文，随即转入动机、启示、常规手段等非显而易见性论证。

**“不重复”≠“不衔接”（硬性）**：禁止在 2.2/3.2 **重写**技术问题整段，但**必须把 2.1/3.1 已界定的实际技术问题当作非显而易见性与技术启示论证的锚点**。凡写“有无技术启示 / 有无结合动机 / 是否常规手段”，一律回答：现有技术是否给出了将区别特征（N）用于最接近现有技术、**以解决如 2.1（或 3.1）所述实际技术问题**的教导。禁止只写“对比文件未公开某特征”就停笔，也禁止脱离该技术问题空谈“无启示”。

### 技术问题 → 技术启示衔接（硬性；三步法第三步）

创造性三步法第三步的判断对象不是抽象的“特征是否已知”，而是：在已确定区别特征与实际技术问题后，现有技术整体上是否给出将所述区别特征应用到最接近现有技术以解决**该**实际技术问题的**技术启示**。本 skill 要求：

1. **2.1/3.1 产出可被引用的问题表述**：技术问题须以**问题要素**具体到可由区别特征（N）解决的层面，避免宽泛到任何改进都能对上，也**禁止把技术方案手段写入问题句**；写完后应能用一短句回指（如“如 2.1 所述，实际要解决的是……”）。
2. **2.2/3.2 必须专设技术启示论证**：体系化归纳中，“首先/其次/再次”每一层都要落到“针对如 2.1 所述技术问题，该文件/该组合是否给出采用区别特征（N）的启示”；不得用“未公开”替代“无启示”。
3. **技术启示的标准问句**（每项独权至少完整回答一遍）：
   - 对比文件自身要解决的技术问题/技术目的是什么？与 2.1 所述实际技术问题是否相同或足够接近？
   - 对比文件客观教导的技术手段与作用是什么？是否指向采用区别特征（N）这一手段？
   - 即使公开了相似部件/步骤名称，其在该文件中的作用、前提、数据关系是否与解决 2.1 所述问题所需的作用相同？
   - 审查员主张的“作用相同 / 有动机结合 / 常规技术手段”，是否建立在其上位概括的技术问题上，而非建立在 2.1 所述具体技术问题上？
4. **衔接用语（推荐，可替换同义表述）**：
   - `针对如 2.1 所述实际技术问题，对比文件【X】……并未给出采用上述区别特征（N）的技术启示。`
   - `对比文件【X】所要解决的是……问题，与如 2.1 所述……问题并不相同；其教导停留在……，本领域技术人员据此不会为解决如 2.1 所述问题而……。`
   - `审查意见所谓“有技术启示”，是以其概括的“……”问题为前提；在如 2.1 所述更具体的技术问题之下，该前提不成立。`
5. **有益效果仍归 2.3**：2.2 论证“无启示”时，可点明“因此不能显而易见地获得本申请方案”，但具体技术效果展开仍放 2.3，并与 2.1 技术问题镜像对应。

### 两项或多项独立权利要求的处理（硬性）

若权利要求书包含两项独立权利要求（常见为“方法独权 + 系统/装置独权”），不得只完整论证第一项独权，再用一句“另一独权与其对应，故也具备创造性”带过。应当：

1. **逐项识别创造性基础**：分别确定每项独权的最接近现有技术、区别技术特征、实际技术问题及审查员对该项独权的评述。两项独权共用同一对比文件时也要分别核对其公开内容。
2. **逐项设置论证，结构二选一（据发明构思关系择一，均须每项都有区别特征/充分非显而易见性/单独深入技术进步）**：
   - **构思相异 → 分章**：第一项独权写“2.1 / 2.2 / 2.3”，第二项独权另设下一大节写“3.1 / 3.2 / 3.3”（后续从权章节顺延）。
   - **共用同一发明构思与同一对比文件 → 共享 2.1/2.2/2.3 内部并列**（范文范式）：2.1 内并列列"区别特征（1）=第一独权""区别特征（2）=第二独权"，2.2 内以"第一/第二"分别针对各独权评述作答，2.3 内并列分述各独权效果。此法省重复、更清晰，但**并列不等于合并**——每项独权的区别特征、启示否定、技术进步仍须各自写足，不得共用一段带过。
3. **系统独权不得只作形式映射**：若第二项独权是与方法独权对应的系统/装置，须逐一说明其模块/单元如何对应实施方法步骤，并核对系统独权是否还包含独有的结构、连接、数据流或协同关系；对这些独有特征单独论证。可引用前文已充分论证的共同发明构思，但必须补足系统实现层面的差异，不能只有“同理可得”。
4. **独立发明构思分别论证**：若两项独权解决不同技术问题或采用不同核心手段，不得合并论述；分别建立完整创造性逻辑链。
5. **plan 按独权拆项**：每项独权的 2.1/2.2/2.3（或 3.1/3.2/3.3）各列为独立待办，逐项交付、逐项确认。

### 审查意见引述规则（短文照录，长文概括）

写“针对审查意见指出的：”之后：

1. **篇幅适中或较短**：完整照录该条审查意见原文，不得歪曲。
2. **原文非常长**：改为**忠实概括**，保留其结论链条中的关键认定（如公开了什么、作用是否相同、给出何种启示、有何种结合动机、认定是否常规等），用一两段浓缩表述对准靶子；**不得删掉关键结论点，也不得改成对自己有利的弱化版**。
3. 概括后仍须让读者一看就知道在反驳哪一条审查逻辑；必要时可在概括句中用引号保留审查员的关键原词（如“作用相同”“有动机结合”“常规技术手段”）。
4. 判断“非常长”的实务标准：单条连续论述明显超过半页、或包含多段重复铺陈时，优先概括；一两句到一小段的结论仍宜照录。

### 2.2 内的针对性答复段模板（不含有益效果；不重写技术问题，但必须衔接到启示）

```text
第【序】、针对审查意见指出的：【短文照录原文；过长则忠实概括其结论链】
对此，申请人答复如下：
其一、【就该条审查结论拆解：可指向 2.1 已列区别特征（N）、指出对比文件未公开等；若涉问题界定，仅一句“如 2.1 所述……”指向，不重写技术问题】
其二、【技术启示/结合动机核心句：针对如 2.1 所述实际技术问题，对比文件【X】……未给出将区别特征（N）应用于对比文件1的技术启示；可接无动机 / 相反教导 / 过度概括 / 退一步…——用第1步为本条选定的手法】
（视需要再写其三；**仅当上一层仍留缺口且下一层事实成立时**，才用“退一步来讲……”设防；不成立则停，勿硬凑）
【此处不要写有益效果；效果统一写入后文 2.3】
【此处不要重写 2.1 已论述的实际技术问题整段；但“无启示”必须回扣该问题】
【禁止套话堆砌：每句须能指回对比文件具体公开或区别特征（N）；写完后删掉任何只靠修辞成立的句子】
```

### 独权创造性三节模板（2.1 → 2.2 → 2.3）

**2.1 区别技术特征（编号 + 权利要求原文）+ 实际技术问题 + 与审查员概括对比**

> 2.1、申请人认为，权利要求1与最接近的现有技术对比文件1相比，至少具有以下区别技术特征：
>
> 结合修改后的权利要求1，申请人认为，修改后的权利要求1相对于对比文件1至少进一步具有以下区别技术特征：
>
> 区别特征（1）：【权利要求原文，不得自行概括压缩】
>
> 区别特征（2）：【权利要求原文】
>
> ……
>

8. **结论必须落回权利要求整体**：最终明确，本领域技术人员在申请日前没有合理成功预期，不能在不付出创造性劳动的情况下为解决如 2.1 所述问题而得到该项独权的整体方案。

**表达增强手法子库（新增；让答复"表达清楚、重点突出"）：**

> 硬性护栏：以下手法**必须以已验真的事实论证为前提，只作论证的表达外壳，不得替代论证**。每个金句/比喻/场景后必须紧跟其事实依据（对比文件段号 / 区别特征编号）；数量服从递进，不堆砌；写完删掉任何只靠修辞才成立的句子。

1. **金句化降维**：把核心区别压成一句直觉性对比（范文："本质差异不是'用了什么传感器'，而是'传感器在谁的上面'"）。
2. **工程场景叙事对比**：分步骤讲"对比文件的场景 vs 本申请的场景"，让审查员"看见"差异，而非只读术语。
3. **比喻锚定**：用一个形象（"围墙 / 翻墙"）把整段论证钉进记忆。
4. **方法论范式对比框**：一对标签把复杂技术差异结构化（"自下而上·理论建模 vs 自上而下·黑箱频响"）。
5. **单点爆破**：锁定一个对方完全没有的关键环节（"力锤标定"）打聚光灯，逐一说明对方各步骤为何都不对应。
6. **归谬法 / "有规则 vs 无规则"反衬**：把对方逻辑推到荒谬极端；用本申请"有解"反衬结合方案"无解"。
7. **"常规设计"物理关联反驳**：用具体物理结构/受力路径的关联性（如 MAX(|F1|,|F2|) 对应油膜受力方向可变），打掉审查员的"常规设计"标签。

>

> 【每一个效果必须挂在区别特征（N）上，并与 2.1 的实际技术问题镜像对应】
> 【明确：该效果不是最接近现有技术的简单公开方式所能直接体现的】
>
> 因此，修改后的权利要求1相对于现有技术具有显著的技术进步。
>
> 综上所述，修改后的权利要求1具有突出的实质性特点和显著的进步，具备创造性，符合专利法第二十二条第三款的规定。

**深入论述的固定层次：**

1. **区别特征到运行机制**：按权利要求的数据流、控制流或结构配合顺序，说明每项区别特征实际如何工作，不得只复述权利要求原文。
2. **运行机制到直接效果**：逐项写明“区别特征（N）→ 中间机制/状态变化 → 直接技术效果”，避免从特征直接跳到笼统的“提高准确性/效率”。
3. **特征协同到整体效果**：说明各区别特征如何耦合协作，缺少任一特征会出现什么不足，整体效果为何不是各已知手段效果的简单相加。
4. **与 2.1 技术问题镜像对应**：2.1 提出的每个具体问题，都应在本节得到明确回应；不得新增独权未解决的效果，也不得把从权效果写成独权效果。
5. **相对现有技术的进步**：分别对比最接近现有技术及审查员组合方案能够达到的效果，说明本申请多实现了什么、避免了什么、改善了什么，不能只写“现有技术不能实现”。
6. **显著性/不可预期性**：结合现有技术的预期方向说明为何该效果不是常规替换的自然结果。只有原始申请文件或客观材料有支持时，才使用“预料不到”“唯一”等强表述；无证据时改用审慎措辞。
7. **证据与支持**：优先引用说明书实施例、实验数据、附图关系或已记载机理；无数据不得编造定量幅度，可作有说明书支持的定性论证。

**禁止浅表写法：**

- 仅把区别特征换一种说法复述后直接写“因此效果更好”；
- 只列“提高精度、降低成本、提升效率”三个结果而不解释技术因果；
- 两项独权共用一段效果文字、不区分方法执行机制与系统结构协同；
- 把针对性答复中尚未由独权特征支撑的效果搬入 2.3。

**2.3 三点式表达骨架（范文提炼；推荐）：**

在独权效果复述之后，另起一句"与对比文件相比，本申请至少在以下 N 个方面产生了对比文件1、2单独或简单结合所不能实现的技术效果："，然后**逐点编号**，每点四段式：

```text
第【序】、【效果命名，如"零改装测试"/"进风通道内非侵入测量"/"实验力学标定避开内部参数"】
    机制：区别特征（N）如何工作 →
    直接效果：由此得到的技术效果 →
    逐点对比：D1 如何（缺陷）、D2 如何（缺陷）、本申请如何（优势）——不笼统说"现有技术不能实现" →
    场景/商业实用价值：落到具体工程场景（如大批量出厂检测、生产线末端快速质检、"测完即用"）
```

补充要求：

1. **镜像自检（硬性）**：三点式的效果点数应 ≥ 2.1 问题要素数，且与 2.1 各问题要素一一对应（范文：零改装↔不改装内部承载件、进风通道非侵入↔不干扰进风工况、避开内部参数↔可靠确定/实机代表性）；缺一即补。
2. **场景价值 / 商业化落地**：把效果落到具体工程/商业场景，让效果从"技术形容词"变成审查员可感知的实用价值。此维度与《专利/专利撰写/通用/撰写策略/专利申请文件撰写策略》skill 的"商业前景"维度同源，可打通思路。
3. **效果金句化**：可用"从……到……的质变"等金句钉住效果量级（范文："从每测一台废一台到测完即用的质变"）。
4. **护栏**：场景价值只做**定性**、须有说明书/行业常识支持，**不得编造量化市场数据**；不得把从权才有的效果写成独权效果。
5. **两独权分别写效果**：方法独权效果不得机械复制装置独权效果；共享 2.3 时并列分述、各挂各自区别特征。

### 区别技术特征列举硬性要求

- 用**权利要求原文**列出，不得压缩成自己的概括句。
- 用**编号**列出：`区别特征（1）：……` `区别特征（2）：……`；2.1 一次性列出全部；针对性答复与 2.2/2.3 引用时写“上述区别特征（N）”。

### 技术问题以问题要素为主、方向性可留、具体手段下沉（硬性；仅在 2.1 / 3.1 的问题句）

重新界定实际技术问题（含纠正审查员上位概括）时，**问题句以问题要素为主，可保留最小方向性限定，但具体器件/步骤/参数须下沉到"具体而言"**。

1. **问题要素**：从背景技术/区别特征所暴露的不足中提炼“缺什么、坏在哪、风险是什么”（如排泄不畅、失稳后直冲行车道、暴露致强度衰减），可并列 2～3 个相互独立的问题要素。
2. **可留方向性、禁止堆砌具体手段**：问题句可保留"如何在……前提下，获得/确定……"的方向性骨架（范文即用"如何在不改装内部承载件、不干扰进风工况的前提下，获得可推算轴向载荷的位移信息"）；但**不得把权利要求中的具体结构、步骤、参数、施工时序、器件选型堆进问题句**。反例：“如何通过竖向锚杆基础实现不挖地基的坡脚支挡并与墙—坡面分层排水相协同……”——把整套方案写进了问题。
3. **手段归“具体而言”**：区别特征如何针对各问题要素起作用、用了什么具体器件/试验/结构，只在紧随的“具体而言，……”中展开；问题句本身保持要素导向。
4. **与“足够具体”的关系 + 判据**：“具体”指问题要素相对审查员万能上位（提高精度/效率/可靠性等）足够可区分，**不是**把独权手段清单塞进问题句。**自检：删去问题句中的具体器件/步骤/参数名词后，是否仍能看清要解决什么缺陷；能，则合格（方向性限定可保留）；若删后只剩空泛愿望或根本读不懂，则须重写。**
5. **对抗审查员上位时同样遵守**：指出审查员概括“未能具体反映……问题要素”，对比用语也用问题要素，避免用“未能反映××结构/××步骤带来的效果”把手段又写回问题对比句（效果镜像留给 2.3）。

### 技术问题界定后必须对比审查员概括（仅在 2.1 / 3.1）

1. 先写申请人界定的实际技术问题（**问题要素为主、方向性可留、具体手段下沉**）。
2. “具体而言，……”简要展开（**此处才对应区别特征与具体手段，首句宜为发明构思命名句**）。
3. **紧跟**：`审查员将实际解决的技术问题概括为“【审查员用语】”，该概括未能具体反映【①…②…③…】。相较而言，上述技术问题更能客观体现……`（**对比清单化，逐条点名审查员漏掉的问题要素/效果**）
4. 不得只写自己的问题而不点出审查员概括何处不当。
5. **只在区别特征分析节（2.1 / 3.1）完成上述“界定 + 对比”整段**；后续非显而易见性节不得整段重复该界定。
6. **为 2.2/3.2 预留可回扣的问题锚点**：问题表述应以问题要素写清、可被短句引用；写完 2.1 后自检——若删去该技术问题，后文“无启示”是否会失去判断对象；若会，说明衔接设计正确。另自检手段是否误入问题句（见上节）。

## 情形分支：修改与否 × 一通/二通

先判**通知书次数**（上文），再判是否修改：

- **情形一｜修改了权利要求**：写「修改说明」——将拟补充内容**接到原独立权利要求末尾**（并入来源 + 完整新增原文 + 法33条 + 细则51条）。并入哪一条/哪些权利要求须遵守下文「修改策略」（**以一次答复争取授权为默认目标：核心区别特征成组并入 + 平衡保护范围不过窄 + 梯度防御靠保留从权**）。
  - 一通：修改说明 → 三段论（2.1→2.2→2.3，2.2 内嵌审查员观点）。
  - 二通：修改说明可并入开场后；**先**「针对第N次审查意见核心观点作答」，**再**创造性收束（含 2.1）。
- **情形二｜未修改**：开头“申请人坚持原权利要求”。
  - 一通：坚持原权 → 三段论全文。
  - 二通：坚持原权 → **先**核心观点逐条作答 → **再**创造性收束。

## 修改策略（硬性；一通尤其适用）

**默认目标：以一次答复争取授权**——把能立住创造性的核心区别技术特征较充分地并入独权，**不为"留后手"而故意把核心的点保留给二通**；同时**平衡"保护范围不过窄"**，并入以"够用能立住创造性"为度。梯度防御靠**保留其余有价值从权、在末节作"退一步·独立防线"论证**实现，而非靠对独权藏招。

> 这与老师范文（PA23-1239 案）实际做法一致：把从权 3 + 说明书 [0065] 的核心区别较充分并入装置独权、把从权 7/8 + 说明书多段并入方法独权以争取一次授权，同时**仍保留权 7、8 为独立从权**，在"三、其他权利要求"用"退一步"作独立防线——核心并入、非核心保留，而不是把核心留到二通。

### 核心原则

1. **并入对象=能立住创造性的核心区别特征**：从现有从权、以及**说明书已记载且符合法33条、可改写成权利要求的技术内容**中，选出直接支撑申请人界定的实际技术问题、相对最接近现有技术**区分度最高、审查员最难用"常规手段/简单叠加"一笔带过**的特征并入独权。**并入来源必须同时考虑从权与说明书，不得只搬从权。**
2. **并入充分度：以"足以一次立住创造性"为标尺**：宁可把构成"完整逻辑链条"的一组核心特征并齐（如本案将"结构一致+倾斜插装+锐角+端面延伸路径"成组并入），使 2.1 技术问题具体、2.2 启示否定站得住、2.3 效果闭环，也不要因怕缩范围而只并入孤零零一处、导致创造性说不圆。
3. **范围平衡（硬性）**：在"能一次立住创造性"的前提下，**不再叠加纯锦上添花、对创造性无实质贡献却会不必要缩小保护范围的限定**；能用高区分度、强说明书支持的少数特征立住的，就不并入更多次要限定。落笔前对每一处拟并入限定自问：删去它，本轮创造性是否仍成立？若仍成立且它明显缩小范围，则不并入、下放/保留为从权。
4. **梯度防御靠"保留从权独立抗辩"实现，而非靠对独权藏招**：核心并入独权后，其余有价值从权（细化参数、次级协同、优选方式、独有判据等）**保留其从权地位**，在"第7步/三、其他权利要求"作"退一步·独立防线"论证（见第7步）。这样即便独权被质疑仍有从权兜底——但这**不等于**故意把核心留到二通。
5. **确有多轮预期时的例外**：仅当案情复杂、争点多、预判一次难以说服时，才可有意识保留个别专克潜在下一轮争点的从权；默认不这样做。
6. **用户已指定并入方案时**：以用户方案为准，但第1步仍须标明哪些属"核心并入"、哪些属"保留作独立防线"；若用户方案会导致创造性说不圆（并入不足）或范围过窄（并入过度），须提示风险，确认后再执行。

### 第1步必做：权利要求修改分层清单（情形一时）

在选法对照表之外，另输出一张表（可与用户确认）：

| 层级 | 权利要求编号（或说明书可支持改写来源） | 为何并入/保留 | 建议处置 |
|------|--------------------------------------|------------------|----------|
| **核心并入独权（争取一次授权）** | 如原从权 X + 说明书第[..]段 | 区分度最高 / 问题锚点 / 构成完整逻辑链条 / 对抗本轮审查逻辑 | 成组接到独权末尾 |
| **保留作独立防线（不并入）** | 如原从权 Y、Z | 有价值但对独权创造性非必需；下放可避免范围过窄；末节可作退一步独立抗辩 | 保持从权、第7步单独论证 |
| **仅论证引用、不修改** | 说明书段落… | 支持效果/机理 | 不写入独权 |

自检：

- **并入是否够立住创造性**：假设只保留"核心并入"层，2.1 技术问题是否具体、2.2 启示否定是否站得住、2.3 效果是否闭环？若否，说明核心特征选少了，应上调并入。
- **范围是否被不必要缩窄**："核心并入"层里是否有删去后创造性仍成立、却明显缩小范围的限定？若有，下放为从权。
- **多独权是否都已处理**：装置独权与方法独权若都被评述，是否都做了核心并入（见第2步"多独权分别写"）。

### 与答复正文的衔接

- **2.1** 围绕**已并入（或坚持）的独权**成组列区别特征与技术问题；两独权共用同一发明构思时可在 2.1 内并列列"区别特征（1）/（2）"（见 2.1"多独权结构二选一"）。
- **2.2/第一部分** 论述整体性时可提及并入特征之间的协同；保留作独立防线的从权特征不得暗示已是独权一部分。
- **第7步/三** 把"保留作独立防线"的从权兑现为"退一步·独立防线"论证，形成"修改说明保留 → 独权并入核心 → 末节独立抗辩"的**留—用闭环**。

## 答复要点：核心必达 + 视情选用

**A. 核心必达**

1. **区别技术特征**（落在 2.1，编号 + 原文）
2. **实际解决的技术问题** + **与审查员概括对比**（落在 2.1；只在此完整论述；**问题要素为主、方向性可留、具体手段下沉；对比清单化**）
3. **非显而易见性 + 技术启示否定**（一通：落在 2.2；二通：主战场在第一部分，2.2 回引补漏）：以 2.1/3.1 所述实际技术问题为锚点；不得用“未公开”代替“无启示”；按"首先破上位→其次破方向动机→再次破鸿沟"递进；可用表达增强手法（工具箱 I）
4. **有益效果 / 显著的技术进步**（落在 2.3/3.3；镜像技术问题；不在逐条答复里做效果收束；推荐三点式+场景价值+镜像自检）
5. **引述审查员结论性观点**（一通嵌入 2.2；二通作为第一部分标题靶子）
6. **修改说明**（仅情形一）+ **权利要求修改分层**（核心区别特征成组并入争取一次授权 / 保留从权作独立防线 / 仅论证引用；平衡保护范围不过窄）

**B. 视情况选用**（按案情组合，服务于递进链条，不必全覆盖）

- 过度概括、事后分析、正常研发路径、相反教导/互斥、变好变差分类、技术鸿沟、失败预期、结合冲突、结合障碍、领域可迁移、退一步/再退一步、整体协同等——第1步选出后**排成递进序**再写。

## 硬性流程规则（最高优先级）

1. **禁止一次生成答复全文**。
2. **先判一通/二通，再分析选法**：摘出审查员结论性论断；为每条挑选 2–4 种**本案事实真正撑得住**的手法，并给出**递进顺序**（为何先写 A 再写 B）；**并非每种都用**；选法与组合方案须先交用户确认。工具箱只收通用手法，案情特有表述不得默认选用。
3. **内在合理性优先于套模板（硬性）**：骨架（一通三段论 / 二通观点作答、2.1→2.2→2.3、引述后拆解）必须遵守；**手法层数、退一步措辞、失败预期、互斥、听谁的、违背初衷等不是填空题**。落笔前对拟用的每一环自问：对比文件客观教导是否支持该论断？本领域人员申请日前会否认同？若只能靠夸大、臆测或套话才能写下去，**弃用该环**，宁可链条短而真，不可为凑“其一/其二/其三”或五层框架强行说理。第1步选法表须标注每环的**事实支点**；支点不足的手法不得进入正文。
4. **用 plan/待办清单驱动全过程**：一通按三段论步骤拆；二通按“核心观点①②③… + 2.1 + 收束”拆；逐步更新状态，始终只有一个“进行中”。
5. **一步只做一个交付物**；须等用户确认再进入下一步；按用户提示推进。
6. 用户要求“一口气写完”时仍分步；修订也按步。

若有两项独权，第4条中的“三节各一步”适用于**每一项独权**：权利要求1的 2.1/2.2/2.3 完成并确认后，再依次撰写第二项独权的 3.1/3.2/3.3；不得一轮同时交付两项独权的全部创造性正文。

每步输出格式：`【第X步 / 共N步】步骤名称` + 本步交付物 + 请用户确认 + 预告下一步 + 更新 plan 状态。

### 第1步：材料核对 + 一通/二通 + 结论清单 + 手法组合 + plan（必做交付物）

**必收材料**：通知书（含次数）、对比文件要点、当前权利要求书、拟修改方案或坚持不改；二通时尽量附一通答复要点，避免复读。

**交付物：**

1. 材料齐备情况表；**明确本次为一通或二通及以后**。
2. 审查意见结论清单（结合动机型 / 问题界定上位型 / 常规公知型 / 特征抽取拼贴型 / 教导忽略型）。
3. **变好/变差预分类（推荐）**：假设将本申请手段植入对比文件1，对比文件1会变好还是变差/互斥？据此选择主攻链条（见工具箱 F）。
4. **逐条手法组合（不是清单堆砌）**：每条挑 2–4 种手法（可更少），写明**递进顺序**、理由，以及每环的**事实支点**（对比文件哪一公开 / 本申请哪一区别特征能撑住该论断）；支点写不出的手法直接剔除。输出对照表（审查结论 / 类型 / 手法组合及顺序 / 事实支点 / 对应区别特征编号 / 落点）。**不是每种都用**；目标是组成**最有说服力且说得圆的一条链**。
5. 判明是否修改；说明**一通/二通**下的篇幅分配（一通：三段论为主；二通：观点作答为主）。
6. **独权清点（硬性）**：独立权利要求数量、编号、主题、是否方法/系统对应保护。
7. **权利要求修改分层清单（情形一时硬性）**：按上文「修改策略」列出核心并入独权的区别特征（来源=从权 ∪ 说明书，争取一次授权、成组并入）/ 保留作独立防线的从权 / 仅论证引用；核对是否够立住创造性、又未把范围缩得过窄。
8. **建立 plan 清单**（按一通或二通流程列项；二通须把“核心观点①②③”列为独立待办）。

### 第2步：修改说明 / 坚持原权

**情形一**：把第1步确认的**核心区别特征**（来源=从权 ∪ 说明书已记载内容）**接到对应独立权利要求末尾**。修改说明须**逐条编号、逆向选特征、可溯源**，遵守以下 7 条规范：

1. **逐条编号，五要素齐全**：一条修改一个条目，含【修改对象 → 并入依据（精确到从权编号 + 说明书段号）→ 新增原文照录 → 从而形成新权利要求X → 法33条 + 细则51条声明】。
2. **依据多源、精确到段**：并入来源同时考虑从属权利要求与说明书；凡以说明书为据，必须写明精确段号（如"根据原说明书第[0065]段的记载"），为法33条超范围抗辩预埋证据。**不得只搬从权而漏掉说明书里能拉开区分度的支持性记载。**
3. **多独权分别写**：装置独权与方法独权同被评述时，**各写一条独立修改说明**（各自的并入依据与新增原文），不得只实质补强一个、另一个仅调引用带过。
4. **逆向选特征**：先想清 2.1 要立的区别、2.2 要破的启示、2.3 要挂的效果，再回头决定并入哪些特征；修改说明服务于后文论证，不是孤立形式段。
5. **适应性调整一次交代干净**：删除已并入的从权、后续权利要求重新编号、原从权的从属关系适应性修改、另一独权及其从权引用关系调整——一次列全（细则形式合规硬项）。
6. **逐条双法条声明**：每条修改分别声明法33条（未超范围）+ 细则51条（针对创造性缺陷的答复性修改），不合并、不省略。
7. **防重复并入自检**：并入某从权的特征后，该从权应删除；若仅并入其部分特征、保留其余作独立防线，须明确界定"哪部分已并入、哪部分保留"，杜绝"同一特征既并入独权又作为从权保留"的编号/逻辑冲突（老师范文里"原权8 特征既并入方法独权又作新权7"即此类需警惕的隐患）。

> 1、申请人对原独立权利要求1进行了修改：根据原权利要求【X】的附加技术特征【及原说明书第[..]段的记载】，在原独立权利要求1中添加以下内容：
> “【接在原独权末尾的完整新增文字，通常以「其中，……」起句；成组并入时逐项照录】”
> 从而形成新的权利要求1。上述修改未超出原权利要求书和说明书所记载的范围，符合专利法第33条的规定；……符合专利法实施细则第51条的相关规定。
>
> 2、由于原权利要求【X】已并入新权利要求1，删除原权利要求【X】并对后续权利要求重新编号；原从权【..】从属关系作适应性修改；方法独权及其从权相应调整引用关系。
>
> 3、申请人对原独立权利要求【方法独权】进行了修改：根据原权利要求【Y、Z】的附加技术特征及说明书第[..]段的记载，补充限定：……。上述修改同样符合专利法第33条及实施细则第51条的规定。

**情形二**：仅一句坚持原权利要求。若一通选择不改（较少见，因默认导向一次授权），第1步仍可列出"必要时可并入的核心特征候选"，供后续使用。

### 一通·第3步 / 二通·第4步：2.1 区别技术特征 + 技术问题 + 与审查员概括对比

严格按上文「2.1 模板」输出；顺序为编号原文特征 → 未公开说明 → 实际技术问题 → 具体而言 → **与审查员概括对比**。标题用：

### 一通·第4步：2.2 非显而易见性（内嵌审查员观点答复 + 技术启示）

严格按上文「2.2 模板」「技术问题 → 技术启示衔接」及“充分论证的最低要求”输出。标题用：

- 按审查意见结论拆子项；“针对审查意见指出的：”后短文照录/长文概括；

- 不整段重复技术问题；禁止效果收束；
- 末步做首先/其次/再次/最后体系化归纳。

### 二通·第3步：针对第N次审查意见核心观点逐条作答（主战场）

### 二通·第5步：2.2/2.3 收束

### 第5步：2.3 显著的技术进步（有益效果统一在此）

本节必须**单独成节并深入论述**，严格按上文「2.3 模板」及“深入论述的固定层次”输出。标题用：

- 至少写清“区别特征 → 运行机制 → 直接效果 → 特征协同 → 相对现有技术的显著进步”；不得只罗列效果名称。
- 以创造性符合法22.3的收束句结束本节。

### 第6步：第二项独立权利要求（存在时，拆为三个子步骤）

若本案有第二项独权，对其分别撰写：

- **3.1 区别技术特征 + 实际技术问题 + 与审查员概括对比**（技术问题仅在此完整论述）；
- **3.2 非显而易见性**：充分程度不低于第一项独权；共同论点可准确引用 2.2，但须补充第二项独权特有的结构/模块/数据流与现有技术差异；**不整段重复 3.1 已写的技术问题**，但必须完成以 3.1 所述技术问题为锚点的技术启示否定论证；
- **3.3 显著的技术进步**：单独成节，写清第二项独权自身“特征 → 机制 → 效果”链，并与 3.1 技术问题镜像对应。方法独权的效果不能机械复制为系统独权效果。

若两项独权分别为方法与系统，3.2/3.3 至少核对：

1. 系统模块是否只是功能名称，还是限定了特定连接/数据流/协同；
2. 模块组合如何实际承载方法中的关键步骤；
3. 系统独有结构是否产生额外技术作用；
4. 对比文件是否真正公开相同模块关系，而非仅公开相似功能；
5. 即使共同发明构思已在 2.2 论证，系统化实现是否仍需无启示的结构改造。

不存在第二项独权时，在 plan 中将本步标记为“不适用”，不得虚构。

### 第7步：其他权利要求（升级为"梯度防御 / 退一步·独立防线"）

不止写"从权依附独权故具备创造性"的概括，还要把修改说明阶段**保留作独立防线**的重点从权兑现为独立抗辩，形成"修改说明保留 → 独权并入核心 → 末节独立抗辩"的**留—用闭环**。

1. **依附概括**：先写"新从属权利要求【..】直接或间接引用新权利要求【..】，在独权具备创造性的前提下亦具备创造性"。
2. **统一"退一步·独立防线"定位句**：对重点从权，用"**退一步而言，即使审查员对独立权利要求的创造性持保留意见，以下从属权利要求的附加技术特征同样构成相对于对比文件1、2及其结合的实质区别：**"起句，把从权抗辩定位成独权失守后的第二道防线。
3. **重点从权按"小独权"标准论证**：写清该从权附加特征的技术细节（如具体判据、参数关系）→ 逐点对比对比文件公开了什么、未公开什么 → 指出审查员"常规设计"标签缺乏事实依据、未证明技术启示。
4. **可复用表达手法**：结构化排比对比（"目的不同、判据不同、处理动作不同"）、范式/学科分支对比下沉（"力锤试验属实验模态分析，D2 属数值仿真修正，分属不同学科分支"）。
5. **护栏**：从权独立抗辩仍须以对比文件事实为据，不得为凑防线强行说理；每条结论落回"该从权相对对比文件1、2及其结合具备创造性"。

## 反驳手法工具箱（第1步据此选法并组合；一通写入 2.2，二通写入第一部分）

**备选清单而非必用清单**。只收可跨案件复用的通用手法。目标不是“用得越多越好”，而是**组出一条最有说服力的递进链**。

> 个别案件审查意见里的特有表述（例如“对比文件没有排除某手段，故有动机……”）属**案情特例**，不是通用手法，不得收入工具箱、更不得默认选用；仅当本案确有同类表述时，可按事后分析（第1项）等通用逻辑就事论事反驳。

### A. 结合动机 / 显而易见

1. **反对事后分析（事后诸葛）**：申请日前看对比文件，不会反向抽取本申请框架再投射回去
2. **正常研发路径叙事**：先发现具体问题再设计手段；反对“忽略问题发现难度、拆解手段再宣称容易想到”
3. **根本差异 → 相反教导 / 互斥方案（teaching away）**
4. **自洽闭环、缺失动机与核心功能**：对比文件1已能独立完成其目标时，无引入额外环节的动机

### B. 技术问题界定

5. **纠正技术问题过度上位**：扳倒“提高准确性/效率”等万能问题；回到区别特征对应的**具体问题要素**（与 2.1 呼应）；纠上位时仍禁止把独权手段写进问题句

### C. 简单迁移 / 常规调整

6. **反对过度概括对比文件**：反对抽掉前提条件与整体方案后的“上位思想 + 特征抽取”
7. **相反方向 / 无逆向动机**

### D. 公知常识 / 跨领域

8. **领域可迁移性反驳**：场景前提不同则不可当作通用手段直接迁移

### E. 递进退让

9. **退一步：从思想到方案的鸿沟**
10. **再退一步：结合后冲突无规则**（两套逻辑听谁的？）
11. **结合障碍：违背对比文件初衷**（成本/效率/复杂度与 D1 发明目的冲突）

### F. 植入后“变好 / 变差”预分类（选主攻链用）

先问：若对比文件1采用本申请手段，会变好还是变差/互斥？

| 类型 | 典型信号 | 主攻递进链（示例；须逐环验真） |
|------|----------|-------------------|
| **变差/互斥** | 改变 D1 原理、破坏其必要条件、增加无谓复杂 | 仅当互斥/无动机**事实成立**时用 3/4 → 5 纠上位 → 6 过度概括 → 针对 2.1 问题无启示 →（成立时）11 违背初衷 |
| **看似变好** | 提高稳定/排水等普遍动机说得通，具体方案对不上 | **主链宜短而真**：6 客观教导差异（作用并非审查员所说的相同）→ 针对 2.1 问题无启示 →（成立时）9 从普遍动机到本案具体方案仍有鸿沟。**不要默认写失败预期、听谁的、违背初衷**——边坡支挡排水类案件里“结合会注定失败”往往说不圆，硬写即强行说理 |

“看似变好”常用要点（按事实取舍，不必写满）：

1. 承认加强稳定/改善排水等普遍动机说得通，但普遍动机不等于教导采用**本申请这一具体手段组合**；
2. 紧扣对比文件客观公开：部件位置、连接关系、施工对象与区别特征（N）是否同一，作用是否针对如 2.1 所述问题要素；
3. 若存在真实缺口，再写从“想改进”到“得到区别特征（N）”缺哪一步教导；**仅当本领域人员确会预见失败或方案互斥时**，才写失败预期/反向教导；
4. 问题发现难度、整体协同：仅当对1未暗示如 2.1 问题要素、或特征间确有不可拆的前提关系时使用。

### G. 针对“上位思想 + 常规细节”的递进（可选用，仍须逐层验真）

审查员逻辑常为：把问题/效果上位成常规 → 再称区别特征常规或容易想到。可选用层次如下，**每一层用前先验本案是否成立**：

1. 指出对对比文件的上位化概括脱离实际教导；
2. 即便存在框架思想，从框架到本案具体方案仍有鸿沟（须能指出缺哪一环教导）；
3. 对比文件1整体教导是否真的阻碍“引入某改进”（不成立则整层删除，勿硬写“绝无动机”）；
4. 事后诸葛亮——宜点到为止，勿替代特征比对；
5. 整体性与协同（仅当特征间有真实耦合时）。

### H. 逻辑手法增补（范文提炼；仍须逐环验真）

12. **归谬法**：把审查员逻辑推到荒谬极端——若"有共同上位概念即有启示"成立，则几乎任何使用同类功能元件的方案都显而易见，有悖创造性中技术启示的认定标准。
13. **多篇对比文件"合力反转"**：证明多篇对比文件在关键问题上指向**共同路径**（如 D1、D2 都"改装被测件"），再把这股合力反转为对本申请的更强反向阻却（"它们共同构成一道围墙，本申请恰是翻墙"）。比孤立的相反教导更有力。
14. **问题发现难度专项**：指出审查员忽略了"问题本身的发现难度"——对比文件根本没意识到本案要解决的那个特定问题（如"进风通道内布传感器不阻气流"），谈何给出解决它的启示。
15. **"有规则 vs 无规则"反衬**：结合后若出现冲突（两模型结果不一致、方向不确定等），对比文件无任何判定/换算规则，而本申请恰给出明确工程准则——用本申请"有解"反衬结合方案"无解"。
16. **"拆核心 + 造新系统"组合难度**：量化得到本申请须完成几个无教导跳跃——放弃对方几个核心发明构思 + 新发明几套系统；"拆掉各自核心再造新系统"显非正向启示，而是以本申请为蓝本的事后拼凑。

### I. 表达增强手法子库（让答复"表达清楚、重点突出"；须以已验真事实为前提，只作论证外壳）

> 硬性护栏：金句 / 比喻 / 场景叙事等**不得替代论证**；每个金句/比喻后必须紧跟其事实依据（对比文件段号 / 区别特征编号）；数量服从递进、不堆砌。

17. **金句化降维**：把核心区别压成一句直觉对比（"传感器在谁的上面"）。
18. **工程场景叙事对比**：分步骤讲"对比文件的场景 vs 本申请的场景"，让审查员"看见"差异。
19. **比喻锚定**：用一个形象（"围墙 / 翻墙"）把整段论证钉进记忆。
20. **方法论范式对比框**：一对标签结构化差异（"自下而上·理论建模 vs 自上而下·黑箱频响"）。
21. **单点爆破**：锁定一个对方完全没有的关键环节（"力锤标定"）集中打。
22. **"常规设计"物理关联反驳**：用具体物理结构/受力路径的关联性打掉"常规设计"标签。
23. **结构化排比对比**：如"目的不同、判据不同、处理动作不同"，用于从权/细节对比，清晰有节奏。

> 组合规则：先 F 定主攻类型 → 再从 A–E/G/H 取**经事实验真**的 2–4 环（可更少）排成递进序（推荐范文的三层：首先破上位概括 → 其次破方向动机含合力反转 → 再次破从思想到方案的鸿沟含无规则反衬）→ 每环回扣“如 2.1 所述技术问题有无技术启示”→ 视需要用 I 的表达手法为关键环节加"外壳"。一通把该链写入 2.2；二通把该链写入第一部分，2.2 只收束。不要用效果收束代替非显而易见论证，也不要用“未公开”代替“无启示”。**禁止为凑满示例链或“其一其二其三”而写本案撑不住的论断；表达手法不得替代事实论证。**
