# 创造性答复意见陈述书撰写规范

本 skill 用于**分步**撰写（或按用户反馈逐步修订）针对创造性缺陷的意见陈述书。最终目标是完整可提交正文，但**禁止在同一次回复中一次性写出全文**。

## 核心结构（务必按此布局）

答复以每项独权的创造性论述为主体，不得把创造性论证拆散成平行、重复的大节：

1. **独权创造性论述——固定三节**（参照岩芯模版，标题与顺序不可改）：
   - **2.1、**申请人认为，权利要求1与最接近的现有技术对比文件1相比，至少具有以下区别技术特征：
   - **2.2、**权利要求1所保护的技术方案是非显而易见的。
   - **2.3** 修改后的（或本申请）权利要求1的技术方案具有显著的技术进步
2. **针对审查意见原文的逐条答复归入 2.2**：在“非显而易见性”节中，完整照录审查员结论 → “对此，申请人答复如下：”→ 其一/其二… 针对性拆解，并与 D1 单独、其他文件、组合无启示等体系化论证融合；不另设一个与独权创造性并列的大节。
3. **其他独权、从属权利要求** + 开场/结尾。

**有益效果归属（硬性）**：凡 2.2 针对性答复中本来会写的“本申请能够……效果”“取得预料不到的技术效果”等，**一律放到对应独权的技术进步一节**（第一项独权为 2.3，第二项独权为 3.3）集中论述，不在逐条答复的“其一/其二/其三”里做效果收束。

### 两项或多项独立权利要求的处理（硬性）

若权利要求书包含两项独立权利要求（常见为“方法独权 + 系统/装置独权”），不得只完整论证第一项独权，再用一句“另一独权与其对应，故也具备创造性”带过。应当：

1. **逐项识别创造性基础**：分别确定每项独权的最接近现有技术、区别技术特征、实际技术问题及审查员对该项独权的评述。两项独权共用同一对比文件时也要分别核对其公开内容。
2. **逐项设置三节论证**：第一项独权写“2.1 / 2.2 / 2.3”；第二项独权另设下一大节，并写“3.1 / 3.2 / 3.3”（后续从权章节顺延）。每项都分别包含区别特征、充分的非显而易见性、单独且深入的技术进步。
3. **系统独权不得只作形式映射**：若第二项独权是与方法独权对应的系统/装置，须逐一说明其模块/单元如何对应实施方法步骤，并核对系统独权是否还包含独有的结构、连接、数据流或协同关系；对这些独有特征单独论证。可引用前文已充分论证的共同发明构思，但必须补足系统实现层面的差异，不能只有“同理可得”。
4. **独立发明构思分别论证**：若两项独权解决不同技术问题或采用不同核心手段，不得合并论述；分别建立完整创造性逻辑链。
5. **plan 按独权拆项**：每项独权的 2.1/2.2/2.3（或 3.1/3.2/3.3）各列为独立待办，逐项交付、逐项确认。

### 2.2 内的针对性答复段模板（不含有益效果）

```text
第【序】、针对审查意见指出的：【完整照录该条审查意见原文，不得压缩概括】
对此，申请人答复如下：
其一、【就该条审查结论拆解：可指向区别特征（N）、纠正问题界定、指出对比文件未公开等】
其二、【无动机 / 无启示 / 相反教导 / 过度概括 / 退一步…——用第1步为本条选定的手法】
（视需要再写其三；强结论用“退一步来讲……再退一步来讲……”设防）
【此处不要写有益效果；效果统一写入后文 2.3】
```

### 独权创造性三节模板（2.1 → 2.2 → 2.3）

**2.1 区别技术特征（编号 + 权利要求原文）+ 实际技术问题 + 与审查员概括对比**

> 2.1、申请人认为，权利要求1与最接近的现有技术对比文件1相比，至少具有以下区别技术特征：
>
> 结合修改后的权利要求1，申请人认为，修改后的权利要求1相对于对比文件1至少进一步具有以下区别技术特征：
>
> 区别特征（1）：【权利要求原文，不得自行概括压缩】
>
> 区别特征（2）：【权利要求原文】
>
> ……
>
> 【简要说明对比文件1已公开何处、未公开上述哪一项】
>
> 基于上述区别技术特征，并结合本申请的背景技术，申请人认为，修改后的权利要求1实际要解决的技术问题应当为：
> 【申请人界定的具体技术问题】
>
> 具体而言，【由区别特征如何达成该问题的简要展开】。
>
> 审查员将实际解决的技术问题概括为“【审查员原概括用语】”，该概括未能具体反映区别技术特征所产生的【……效果】。相较而言，上述技术问题更能客观体现修改后的权利要求1相对于最接近现有技术所实现的技术改进。

**2.2 非显而易见性（四层递进，必须充分，逐条针对性答复在本节展开）**

> 2.2、权利要求1所保护的技术方案是非显而易见的。
>
> 首先，……（最接近对比文件单独不能给出）
> 其次，……（其他对比文件不能弥补）
> 再次，……（结合后仍不能自然得到；连续改造清单；禁止事后拼贴）
> 最后，……（适用时反驳“常规步骤/公知常识”）

凡在针对性答复中已充分展开的点，此处做**体系化归纳**并可用“如本意见前文针对审查意见第×点所述……”指引，避免逐字重复；但 **2.2 标题与四层骨架必须保留**。

**2.2 的内部次序：**

1. 先按审查意见核心结论逐条写“完整照录原文 + 对此答复”，分别击破问题上位、结合动机、技术启示、公知常识等；
2. 再按“首先/其次/再次/最后”将前述答复归纳为完整的非显而易见性逻辑；若逐条答复本身已按该顺序组织，可边答边完成四层，但章节标题仍为 2.2；
3. 2.2 只论证“为什么不能显而易见地得到”，不在本节集中展开有益效果。

**充分论证的最低要求：**

1. **对比文件1单独**：不仅写“未公开”，还要说明其技术目的、工作原理、完整方案及客观教导停留在何处；逐项对应区别特征（N），解释为何其不能单独得到本申请。
2. **其他对比文件分别审查**：逐篇说明其技术场景、必要前提、数据/结构来源、实际实施方式，以及为何不能弥补对比文件1。对比文件较多时不得合并成一句“其他文件均未公开”。
3. **组合推理闭合**：分别回答“为什么会找该文件”“为什么会抽取该特征”“为什么会按权利要求顺序/关系组合”“组合后为什么能够正常工作”。审查意见缺少其中任一环，应明确指出。
4. **连续改造清单**：列出从对比文件组合到本申请仍需完成的具体改造，并说明这些改造不是彼此独立的常规替换，而受同一技术问题和数据/结构关系约束。
5. **整体性与协同关系**：说明区别特征之间如何互相提供前提、输入、约束或反馈，为什么不能拆成若干孤立常规手段逐项评价。
6. **公知常识/常规手段须有依据**：审查员仅作结论时，指出其未证明该手段在本案具体技术场景、具体关系和具体用途下属于公知常识；不能只争论手段名称是否已知。
7. **结论必须落回权利要求整体**：最终明确，本领域技术人员在申请日前没有合理成功预期，不能在不付出创造性劳动的情况下得到该项独权的整体方案。

**2.3 显著的技术进步（单独成节；有益效果统一写在这里；必须深入）**

> 2.3 修改后的权利要求1的技术方案具有显著的技术进步
>
> 【按独权特征的数据流/结构顺序复述闭环：先…再…从而…】
> 【每一个效果必须挂在区别特征（N）上，并与 2.1 的实际技术问题镜像对应】
> 【明确：该效果不是最接近现有技术的简单公开方式所能直接体现的】
>
> 因此，修改后的权利要求1相对于现有技术具有显著的技术进步。
>
> 综上所述，修改后的权利要求1具有突出的实质性特点和显著的进步，具备创造性，符合专利法第二十二条第三款的规定。

**深入论述的固定层次：**

1. **区别特征到运行机制**：按权利要求的数据流、控制流或结构配合顺序，说明每项区别特征实际如何工作，不得只复述权利要求原文。
2. **运行机制到直接效果**：逐项写明“区别特征（N）→ 中间机制/状态变化 → 直接技术效果”，避免从特征直接跳到笼统的“提高准确性/效率”。
3. **特征协同到整体效果**：说明各区别特征如何耦合协作，缺少任一特征会出现什么不足，整体效果为何不是各已知手段效果的简单相加。
4. **与 2.1 技术问题镜像对应**：2.1 提出的每个具体问题，都应在本节得到明确回应；不得新增独权未解决的效果，也不得把从权效果写成独权效果。
5. **相对现有技术的进步**：分别对比最接近现有技术及审查员组合方案能够达到的效果，说明本申请多实现了什么、避免了什么、改善了什么，不能只写“现有技术不能实现”。
6. **显著性/不可预期性**：结合现有技术的预期方向说明为何该效果不是常规替换的自然结果。只有原始申请文件或客观材料有支持时，才使用“预料不到”“唯一”等强表述；无证据时改用审慎措辞。
7. **证据与支持**：优先引用说明书实施例、实验数据、附图关系或已记载机理；无数据不得编造定量幅度，可作有说明书支持的定性论证。

**禁止浅表写法：**

- 仅把区别特征换一种说法复述后直接写“因此效果更好”；
- 只列“提高精度、降低成本、提升效率”三个结果而不解释技术因果；
- 两项独权共用一段效果文字、不区分方法执行机制与系统结构协同；
- 把针对性答复中尚未由独权特征支撑的效果搬入 2.3。

### 区别技术特征列举硬性要求

- 用**权利要求原文**列出，不得压缩成自己的概括句。
- 用**编号**列出：`区别特征（1）：……` `区别特征（2）：……`；2.1 一次性列出全部；针对性答复与 2.2/2.3 引用时写“上述区别特征（N）”。

### 技术问题界定后必须对比审查员概括

1. 先写申请人界定的实际技术问题。
2. “具体而言，……”简要展开。
3. **紧跟**：`审查员将实际解决的技术问题概括为“【审查员用语】”，该概括未能具体反映……。相较而言，上述技术问题更能客观体现……`
4. 不得只写自己的问题而不点出审查员概括何处不当。

## 情形分支：是否修改权利要求

- **情形一｜修改了权利要求**：先写「一、修改说明」——将拟补充内容**接到原独立权利要求末尾**添加（并入来源 + 完整新增原文 + 法33条 + 细则51条）；再写针对性答复（若有）与「二、修改后的权利要求1具备创造性」下的 2.1→2.2→2.3。
- **情形二｜未修改**：无修改说明，开头一句“申请人坚持原权利要求”；针对性答复可为正文主干的一部分，但其后仍须有完整的独权创造性三节 2.1→2.2→2.3（用语改为“本申请权利要求1”）。

## 答复要点：核心必达 + 视情选用

**A. 核心必达**

1. **区别技术特征**（落在 2.1，编号 + 原文）
2. **实际解决的技术问题** + **与审查员概括对比**（落在 2.1）
3. **非显而易见性**（落在 2.2；针对性答复可先打具体靶子）
4. **有益效果 / 显著的技术进步**（只落在对应独权的技术进步一节，不在 2.2 针对性答复段内收束）
5. **修改说明**（仅情形一）

**B. 视情况选用**（按审查意见与案情挑用，不必全覆盖）

- 结合动机、技术启示、相反技术教导、过度概括、事后分析、正常研发路径、领域可迁移、退一步/再退一步、结合障碍等——按第1步选法对照表选定，用得上才写，并在 2.2 的针对性答复中展开。

## 硬性流程规则（最高优先级）

1. **禁止一次生成答复全文**。
2. **先分析选法、再落笔**：摘出审查员结论性论断，为每条挑选 2–4 种贴合手法；**并非每种都用**；选法须先交用户确认。工具箱只收通用手法，案情特有表述不得默认选用。
3. **用 plan/待办清单驱动全过程**：列出全部步骤与子项（每条针对性答复各一子项；每项独权的“区别特征/非显而易见/技术进步”各一步），逐步更新状态，始终只有一个“进行中”。
4. **一步只做一个交付物**；须等用户确认再进入下一步；按用户提示推进。
5. 用户要求“一口气写完”时仍分步；修订也按步。

若有两项独权，第3条中的“三节各一步”适用于**每一项独权**：权利要求1的 2.1/2.2/2.3 完成并确认后，再依次撰写第二项独权的 3.1/3.2/3.3；不得一轮同时交付两项独权的全部创造性正文。

每步输出格式：`【第X步 / 共N步】步骤名称` + 本步交付物 + 请用户确认 + 预告下一步 + 更新 plan 状态。

---

## 适用场景

- 审查意见指出不符合专利法第22条第3款（创造性）需书面答复。
- 第N次通知书；**次数、发文日、申请号按用户材料填写**。
- 不区分结构类/方法类/算法类。

不适用：仅新颖性、超范围、不清楚、得不到说明书支持、单一性等且不夹带创造性评述。

## AI 能力边界

- 不得编造对比文件未记载内容、段落号、效果或公知常识出处。
- 不得把本申请特征事后拆开贴回对比文件当作“启示”。
- 信息不足时只论证已能支撑的部分，并列出待核引用。

## 分步流程（必须按序执行）

```text
第1步 材料核对 + 审查意见结论清单 + 逐条选定反驳手法 + 判明情形 →（建立 plan 清单）
  ↓ 用户确认
第2步 修改说明（情形一）/ 坚持原权利要求（情形二）
  ↓ 用户确认
第3步 撰写独权创造性「2.1 区别技术特征 + 技术问题 + 与审查员概括对比」
  ↓ 用户确认
第4步-①/②/③… 撰写「2.2 非显而易见性」中的逐条针对性答复（完整照录 + 驳斥；不含有益效果）
  ↓ 每条各自确认
第4步-末 撰写 2.2 体系化归纳（D1单独 / 其他文件分别 / 组合无启示 / 整体协同）
  ↓ 用户确认
第5步 撰写独权创造性「2.3 显著的技术进步」（有益效果统一写在此步）
  ↓ 用户确认
第6步（存在第二项独权时）依次撰写第二项独权「3.1 区别特征 / 3.2 非显而易见 / 3.3 显著技术进步」
  ↓ 各节分别确认
第7步 其他权利要求
  ↓ 用户确认
第8步 组装全文 + 检查清单
```

### 第4步（多子项）：2.2 非显而易见性（含针对性答复）

严格按上文「2.2 模板」及“充分论证的最低要求”输出。不得只满足标题形式而内容空泛。标题用：

> 2.2、权利要求1所保护的技术方案是非显而易见的。

本步按审查意见结论拆成多个子项，逐条交付：

- “针对审查意见指出的：”后**完整照录**该条原文；
- “对此，申请人答复如下：”后用第1步选定手法拆解，可指向 2.1 的区别特征（N）、纠正问题界定、论证无动机/无启示等；
- **禁止写有益效果收束**；相关效果素材记录下来，留到第5步 2.3；
- 各条确认后，再写本节体系化归纳。至少完成：D1 单独分析、D2/D3 分别分析、组合动机与技术启示审查、连续改造清单、整体协同关系及合理成功预期分析。

### 第5步：2.3 显著的技术进步（有益效果统一在此）

本节必须**单独成节并深入论述**，严格按上文「2.3 模板」及“深入论述的固定层次”输出。标题用：

> 2.3 修改后的权利要求1的技术方案具有显著的技术进步

- 效果挂在区别特征（N）上，与 2.1 技术问题镜像对应。
- 写入第4步 2.2 针对性答复时**刻意留到本步**的全部有益效果要点。
- 至少写清“区别特征 → 运行机制 → 直接效果 → 特征协同 → 相对现有技术的显著进步”；不得只罗列效果名称。
- 以创造性符合法22.3的收束句结束本节。

### 第6步：第二项独立权利要求（存在时，拆为三个子步骤）

若本案有第二项独权，对其分别撰写：

- **3.1 区别技术特征 + 实际技术问题 + 与审查员概括对比**；
- **3.2 非显而易见性**：充分程度不低于第一项独权；共同论点可准确引用 2.2，但须补充第二项独权特有的结构/模块/数据流与现有技术差异；
- **3.3 显著的技术进步**：单独成节，写清第二项独权自身“特征 → 机制 → 效果”链。方法独权的效果不能机械复制为系统独权效果。

若两项独权分别为方法与系统，3.2/3.3 至少核对：

1. 系统模块是否只是功能名称，还是限定了特定连接/数据流/协同；
2. 模块组合如何实际承载方法中的关键步骤；
3. 系统独有结构是否产生额外技术作用；
4. 对比文件是否真正公开相同模块关系，而非仅公开相似功能；
5. 即使共同发明构思已在 2.2 论证，系统化实现是否仍需无启示的结构改造。

不存在第二项独权时，在 plan 中将本步标记为“不适用”，不得虚构。

## 反驳手法工具箱（第1步据此选法；2.2/3.2 取用）

**备选清单而非必用清单**。只收可跨案件复用的通用手法。

> 个别案件审查意见里的特有表述（例如“对比文件没有排除某手段，故有动机……”）属**案情特例**，不是通用手法，不得收入工具箱、更不得默认选用；仅当本案确有同类表述时，可按事后分析（第1项）等通用逻辑就事论事反驳。

### A. 结合动机 / 显而易见

1. **反对事后分析（事后诸葛）**
2. **正常研发路径叙事**
3. **根本差异 → 相反教导 / 互斥方案（teaching away）**
4. **自洽闭环、缺失动机与核心功能**

### B. 技术问题界定

5. **纠正技术问题过度上位**

### C. 简单迁移 / 常规调整

6. **反对过度概括对比文件**
7. **相反方向 / 无逆向动机**

### D. 公知常识 / 跨领域

8. **领域可迁移性反驳**

### E. 递进退让

9. **退一步：从思想到方案的鸿沟**
10. **再退一步：结合后冲突无规则**
11. **结合障碍：违背对比文件初衷**

> 使用建议：C/D 常配 6/7/8 再用 E（9→10→11）；A 常配 1/2 再用 3/4；B 用 5 并与 2.1 技术问题呼应。手法在 **2.2/3.2 的针对性答复与体系化论证**中展开；不要用效果收束代替非显而易见论证。
