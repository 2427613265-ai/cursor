# 创造性答复意见陈述书撰写规范

本 skill 用于撰写（或按用户反馈修订）**针对创造性缺陷**的意见陈述书正文，输出须为**完整可提交**的陈述书文字，不是提纲或要点列表。

## 适用场景

- 审查意见指出权利要求不符合专利法第22条第3款（创造性），需要书面答复。
- 适用于第N次审查意见通知书（一通、二通或更多次）；**通知书次数、发文日、申请号一律按用户材料填写**，不得沿用其他案件或模版中的次数表述。
- 不区分结构类/方法类/算法类；创造性论证逻辑通用。

不适用（应另用对应专题 skill，尚无 skill 时如实说明并按常规法理答复，勿硬套本骨架的全部节次）：

- 仅涉及新颖性、修改超范围、不清楚、得不到说明书支持、单一性等形式或其它实质问题，且不夹带创造性评述。
- 若通知书同时评述创造性与其他问题：创造性部分按本 skill；其他问题单独分节答复，不强行塞进第二节。

## 前置输入（缺一则先追问）

开始撰写前，须确认用户已提供或可据此推断下列材料；缺失则先提问，不凭空编造：

1. **审查意见通知书**：全文，或至少含对比文件编号、评述逻辑、针对哪些权利要求的摘要。
2. **对比文件要点**：审查员援引的段落/图号及公开内容；若仅有摘要而无段落原文，论证中须标注“据用户提供摘录，段落号待核”。
3. **当前权利要求书**（通知书所针对的版本）。
4. **拟修改方案**（若本次修改）：哪些特征并入独权、来源于哪条原从权/说明书何处；若坚持不改，用户须明确说明。

## AI 能力边界

- **不得编造**对比文件未记载的技术内容、段落号、效果或“本领域公知常识”的具体出处。
- 不得在已知本申请方案后，把本申请的具体限定**事后拆开再贴回**对比文件，当作本领域技术人员申请日前已有的“启示”。
- 创造性评估依赖用户提供的对比文件信息；信息不足时只论证已能支撑的部分，并列出待用户补充核验的引用。

## 全文固定骨架

按下列顺序输出（有修改时用“修改后的权利要求1”；无修改时标题与正文改为“本申请权利要求1”，并省略或改写第一节中的“修改内容”为坚持原权利要求的说明）：

```text
尊敬的审查员：
您好！
开场（收到第N次审查意见通知书 + 引出答复）

一、修改说明：（有修改时）/ 坚持原权利要求的说明（无修改时）

二、修改后的权利要求1具备专利法第22条第3款规定的创造性：
  2.1 区别技术特征 + 实际解决的技术问题
  2.2 非显而易见性（四层递进）
  2.3 显著的技术进步
  小结：符合专利法第22条第3款

三、其他权利要求具备创造性（概括 + 择要强化）

结尾客套
此致
```

>

## 第一节：修改说明

### 有修改时

1. **写明修改动作**：对哪一（几）条独立/从属权利要求作了何种修改（例如：将原权利要求X的附加技术特征并入原独立权利要求1）。
2. **完整引用所添加或替换的文字**（用引号给出修改后独权中新增的关键限定，便于审查员对照）。
3. **法33条**：说明上述修改未超出原权利要求书和说明书记载的范围。
4. **细则第51条**：说明修改是针对审查意见通知书中关于创造性（或明确指出的条款）作出的。

句式参考：

> 1、申请人对原独立权利要求1进行了修改：根据原权利要求【X】的附加技术特征，在原独立权利要求1中添加以下内容：
> “【完整新增文字】”
> 从而形成新的权利要求1。
>
> 上述修改未超出原权利要求书和说明书所记载的范围，符合专利法第33条的规定。
>
> 申请人的上述修改，是针对审查意见通知书中关于权利要求【…】不符合创造性所作出，因此，上述修改也符合专利法实施细则第51条的相关规定。

### 无修改时

用简短一节说明申请人经研究后坚持原权利要求，并直接转入第二节创造性论证；不要虚构“已修改”内容。

---

## 第二节：创造性论证

### 2.1 区别技术特征与实际解决的技术问题

**固定顺序：**

1. **列出区别技术特征**（可编号为区别特征（1）（2）…）：相对于最接近现有技术（通常为对比文件1），修改后独权至少还具有哪些特征；表述应尽量贴近权利要求用语，可作不影响含义的压缩，但不得删掉构成完整技术方案所必需的数据关系/步骤链条。
2. **对照最接近现有技术说明“未公开何处”**：承认其已公开的部分，明确指出未公开本区别特征或未公开到本申请所要求的具体程度（例如仅公开“调用封装函数完成某处理”，而未公开本申请逐步限定的完整处理过程）。
3. **归纳实际解决的技术问题**：须由区别特征及其在本申请中的作用客观导出，具体、可检验；避免空泛的“如何得到更准确/更优的结果”之类审查员式过宽概括。
4. **必要时纠正审查员的问题概括**：若通知书中的“实际解决的技术问题”过宽、过空或未能反映区别特征带来的具体技术效果，应明确指出其不足，并给出申请人认为更恰当的问题表述，说明为何更能体现相对于最接近现有技术的改进。

区别特征、技术问题、后文效果三者必须**同一条逻辑链**，禁止问题写A、论证却绕到B。

### 2.2 非显而易见性（四层递进，固定执行）

在标题“权利要求1所保护的技术方案是非显而易见的”之下，按**首先→其次→再次→最后**四层展开。对比文件多于两篇时，将“其次”扩展为对D2、D3…的分别评述，再进入“结合也不能得到”；逻辑层次仍保持：**单独不足 → 其他文件不能补 → 结合无启示 → 反驳常规步骤定性**。

#### 第一层：最接近对比文件单独不能给出独权方案

- 概括该对比文件的**技术目的**与公开层次。
- 引用其具体段落/图（仅限用户材料中有的），说明其止步于何处。
- 对照本申请，列出其**没有公开或启示**的完整限定链条（按权利要求中的数据处理/结构连接顺序罗列，形成“完整逻辑链条”印象）。
- 点明：该对比文件未发现或未试图解决与区别特征对应的具体技术问题。

#### 第二层：其他对比文件不能弥补上述缺陷

对审查员用来“补强”的对比文件：

- **区分上位思想与具体方案**：例如仅公开“可采用某类处理/可选方法之一”，不等于公开本申请限定的完整数据关系、步骤顺序或结构配合。
- **实施例实际采用手段 vs 所列举的可选项**：若对比文件在概述中列举多种可选手段，但具体实施例采用的是另一种，须拆开论述——列举不等于针对本申请场景给出选择本申请手段的启示，更不等于公开本申请的参数分解、邻域/部件配合、权重或连接关系等进一步限定。
- **映射/结构来源不同**：即使上位概念相似（如都有“由目标反求源”），若数据来源、矩阵/模型建立方式、坐标或连接关系不同，应明确说明“不相同、不能直接移植”。

#### 第三层：结合后仍不能自然得到修改后的独权

- 分别简述“以D1为起点本领域技术人员会得到什么”以及“进一步参考D2至多得知什么”。
- 用**连续改造清单**列出：要从现有技术落到修改后独权，仍需在无明确启示的情况下连续完成哪些改造步骤。
- **整体判断原则**：创造性针对权利要求限定的技术方案**整体**；不能仅因其中部分手段分别已知，即否定各步骤之间特定数据关系/连接关系所形成的整体方案。
- **禁止事后拼贴**：明确指出，若审查意见是在知晓本申请后将完整方案拆成局部动作（函数调用、可选方法、常规步骤等）再分别评价，尚不足以说明申请日前为何会按本申请限定的数据关系与顺序建立该完整过程。

#### 第四层：反驳“常规处理步骤 / 公知常识”定性（在适用时）

当通知书曾将某原从权（或其附加特征）概括为“常规步骤/常规选择”，而该内容现已并入独权时：

- 说明原从权**并非**仅有一个上位标签（如“采用某某插值/采用某某常规构件”），而是限定了由多个相互衔接的技术特征构成的**完整过程或结构配合**。
- 指出通知书未具体说明该完整过程分别由哪篇对比文件公开，也未说明基于何种启示会按本申请顺序应用于最接近现有技术。
- 主张：核心内容并入独权后，须以**修改后独权整体**重新进行创造性判断，不宜继续沿用对原从权的“常规步骤”定性直接否定新独权。

若本次未并入从权、通知书也未作“常规步骤”认定，可省略本层，但仍须保证前三层完整。

### 2.3 显著的技术进步

- 按独权技术特征的**数据流或结构装配顺序**复述方案如何闭环工作（先…再…从而…）。
- 每一个效果表述必须**挂在已写入独权的特征上**，说明“正是因为这些特征的配合才产生该效果”。
- 效果应与2.1归纳的实际技术问题镜像对应；禁止空喊“更准确、更精细、效果更好”而不绑定具体特征。
- 对比说明：该效果不是最接近现有技术的简单公开方式（如仅调用封装功能、仅列举可选项）所能直接体现的。

### 第二节收束

> 综上所述，修改后的权利要求1具有突出的实质性特点和显著的进步，具备创造性，符合专利法第二十二条第三款的规定。

（无修改时删除“修改后的”等用语，与全文一致。）

## 第三节：其他权利要求

1. **概括依附**：从属权利要求直接或间接引用独立权利要求，独立权利要求具备创造性的，从属权利要求亦具备创造性。
2. **择要强化**：对通知书重点攻击、或确有附加区别特征与效果的从权，用“值得注意的是：”单独展开——说明其并非仅有表面上的简单附加，而是进一步限定了何种技术内容；对比文件未公开何处；因而该从权也具备创造性。
3. 不要把全部从权逐条注水重复第二节；无额外可争点时，概括一段即可。

## 微型示例（虚构，仅示语气与层次）

> 对比文件1公开了通过人机交互选取目标区域四角点，并调用图形库透视变换函数完成校正。该文件将像素重映射整体封装于所述函数中，未公开“由目标矩形尺寸创建输出图像矩阵→对变换矩阵求逆→逐目标像素反求源点浮点坐标→分解整数坐标与小数偏差→按邻域加权赋值”的完整数据处理链条。
>
> 对比文件2虽在一般意义上列举了若干插值可选方式，但其实施例实际采用的是另一类插值，且坐标映射关系由成像姿态参数建立，与本申请基于源四边形与目标矩形点对点映射再求逆的路径不同。对比文件2至多给出“非整数源坐标可插值”的上位思想，不能给出本申请限定的完整像素重建数据链。
>
> 将两篇结合，本领域技术人员至多得到“选取角点并调用函数校正 + 可选插值”的拼合，仍需在无明确启示下连续放弃封装式调用、展开求逆与邻域权重赋值等多重改造，不能在不付出创造性劳动的情况下得到修改后的权利要求1。

正式案件中须替换为用户材料中的真实对比文件与权利要求用语；本示例不得当作事实引用。

## 输出前检查清单

- [ ] 通知书次数、发文日、申请号、对比文件编号与用户材料一致
- [ ] 有修改时：修改来源、新增文字、法33条与细则51条均已写明
- [ ] 2.1 区别特征、实际技术问题、后文效果三者对应；必要时已纠正审查员过宽概括
- [ ] 2.2 按“单独→其他不能补→结合无启示→（如适用）反驳常规步骤”递进，而非三点平行堆砌
- [ ] 未编造对比文件段落、效果或启示；摘要性引用已标明待核
- [ ] 未把本申请特征事后拆开拼贴为对比文件的结合启示
- [ ] 2.3 效果挂在独权特征上，非空泛赞美
- [ ] 第三节有概括；对重点从权有择要强化（若存在）
- [ ] 输出为完整意见陈述书正文，含开场与结尾，不是提纲
