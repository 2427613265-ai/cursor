# 权利要求书质检规范

本skill用于**整体复核**已经撰写完成的权利要求书（区别于撰写阶段单条撰写时使用的三个规范：`patent-structural-claim-drafting`、`patent-avoid-unnecessary-functional-limitation`、`patent-directional-terms-need-reference`——质检时要把这三个也当作检查项之一并入使用）。

按下面四个维度检查，**优先输出维度二第1点这类可以机械核对、非黑即白的硬伤**，再讨论范围/创造性/梯度这类需要专业判断的问题。

## 维度一：独立权利要求——保护范围与创造性的平衡

1. **必要技术特征是否齐全**：独权是否包含了解决其声称的技术问题所必需的全部技术特征？少了任何一个，方案就不完整、可能得不到说明书支持。
2. **有没有可以删掉但不影响创造性的多余限定**：独权里每一个技术特征，如果删掉后方案仍然新颖、非显而易见、仍能解决背景技术指出的问题，这个特征就应该降级到从权，不要留在独权里不必要地缩小保护范围。
3. **核心区别特征是否真的能支撑创造性**：与背景技术提到的最接近现有技术相比，独权的区别技术特征是不是本领域技术人员容易想到的常规变型？这个区别特征带来的效果是不是真正非显而易见？
4. **保护范围有没有被不必要的具体数值/具体实现方式绑死**：独权里如果出现了具体角度数值、具体一种机构（比如限位槽+限位销这种具体结构），要考虑这些是否该降级到从权，让独权保持更概括的表达。
5. **独权里保留的功能性限定/效果陈述，是否构成了不必要的隐含限定**：如果独权里写了"从而实现XX效果"，理论上会被解释为一个隐含的限定条件（即必须真正产生这个效果才落入保护范围），如果这是撰写者主动的取舍要如实告知，不是撰写者本意就要建议删除。

## 维度二：从属权利要求——依附关系与梯度防御布局

1. **依附编号是否正确——逐条机械核对**：对每一条从权，列出它这一条新用到的所有名词，反查这些名词分别是在哪一条权利要求里第一次出现的，取其中编号最大的那条，从权的依附对象必须等于或晚于（即依附链条上包含）这条。**只要从权用到了某个词，而依附链条里找不到这个词的出处，就是编号错误，必须改**。这是最容易在插入/删除/重排权利要求后出错、也是最容易被审查员一眼挑出来的问题，每次调整编号后都要重新全篇核对一遍，不能只抽查改过的那几条。
2. **梯度是否合理——由概括到具体，由结构到数值**：好的布局应该是"先用一条从权概括性地限定技术手段 → 再用下一条限定其具体结构 → 再用更下一条限定具体数值/档位/锁定方式"。检查是否存在"一步到位塞入所有细节"（层次不够）或者"该合并的却拆得太细"（浪费权利要求数量）。如果几个并列分支的精细程度不对称（比如一个分支拆了三层、另一个分支只有两层），要指出来，由用户决定是否要为了对称性再调整。
3. **从权分支之间有没有实质重复**：检查是否存在两条从权本质上用的是同一种技术手段（哪怕依附对象、措辞不同），如果一条已经隐含覆盖了另一条的保护范围，后者对创造性没有增量贡献，应删除或改成确实不同的技术方案。
4. **并列的互斥实施例分支有没有交叉污染**：比如两种互斥的替代实施例（分支A、分支B）应该各自独立依附同一个上位权利要求，检查有没有从权错误地跨分支引用了另一分支才有的术语。

## 维度三：逻辑完善性

1. **术语前后一致**：同一个技术特征自始至终只用同一个名称指代，不能一会儿叫"夹角"一会儿叫"倾斜夹角"、一会儿叫"基准角度"。
2. **每个新名词是否有清楚的定义和明确的首次出现位置**：不能让后面的从权凭空引用一个之前权利要求里从未出现过的词；如果某个词是从已有部件的固有几何属性中自然引出的（比如"某根管子的轴线""某个环状部件的中心轴线"），无需单独定义，但如果这个词承载了具体的、后续要精确引用的技术含义（比如某种特定转动/调节所绕的轴），就必须有明确的首次定义语句。
3. **引用编号与术语来源要对应**（与维度二第1点是同一件事，从"术语"和"编号"两个角度双重核对，确保不遗漏）。

## 维度四：表达清楚性

直接复用以下三个撰写规范作为检查清单：
- `patent-structural-claim-drafting`：结构类权利要求是否先总述部件、后展开结构。
- `patent-avoid-unnecessary-functional-limitation`：功能性限定/技术效果表述是否用得恰当。
- `patent-directional-terms-need-reference`：方向、方位词是否都有明确参照物。

## 输出格式

质检结果按以下顺序输出：
1. 硬伤：依附编号错误（逐条列出"第几条权利要求 → 应改为依附第几条 → 原因"）
2. 实质重复问题
3. 独权范围与创造性观察
4. 从权梯度布局观察
5. 表达清楚性核对（如有遗漏）
