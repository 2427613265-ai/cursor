# 交底书评估与创造性补强（撰写前处理，所有案件类型通用）

## 适用场景

- 拿到发明人/申请人提供的**交底书**、准备正式落笔撰写专利申请文件**之前**使用。
- 解决的是"这份交底书能不能直接写、够不够格、要不要补强、独权怎么搭骨架"这类**撰写前处理**问题，处在整个撰写流程的最前端，先于 `专利申请文件撰写策略`（方向布局）和各类具体文字撰写。
- 不区分结构类/方法类/算法类，本流程对任意技术方案类型都适用。

## 一个前提：交底书是否浅显到必须先找发明人

发明人的交底书常常不够完善，大部分情况下需要撰写者自行补充完善；**只有当交底书实在过于浅显时，才需要退回去请发明人补充**。因此拿到交底书后先做一个粗判断：

- 如果交底书连"要解决什么问题、用什么手段、核心结构/步骤是什么"这类**最基本的信息都缺失或语焉不详**，导致无法据此展开撰写，则应当先与发明人沟通、请其补充，本skill后续步骤暂不展开。
- 如果交底书**不算太浅显**（技术问题、技术方案、核心创新点大体能看明白，只是存在细节缺漏或表述问题），则不必急于打扰发明人，进入下面的评估—补强—搭建独权流程，由撰写者自行补充完善。

## AI 使用本skill的能力边界（重要）

- **创造性评估依赖现有技术检索，AI 本身不具备专利数据库检索能力**。在没有检索结果、也没有用户提供的现有技术情况时，AI 不能凭空断言"这个方案有/没有创造性"，更不能编造并不存在的"对比文件""检索结论"。此时应如实告知这一限制，请用户补充检索结果或现有技术概况后再评估；用户明确表示无法提供时，只能基于常识和对话中已有信息做**倾向性判断**，并标注该判断的不确定性。
- 补强环节 AI 提出的"更深入的技术问题""新构思的技术方案""极端场景",本质是基于已知技术特征和常识的**合理推演与建议方向**，不是既定技术事实。提出后应交用户/发明人确认技术上是否可行、是否符合发明人真实意图，确认后再写入正式文本，不能未经确认就当作定案。

## 第一步：交底书体检（逻辑 / 形式 / 技术方案完善性）

在撰写独权和评估创造性之前，先给交底书做一次"体检"，找出并（在能力范围内）修补以下三类问题：

- **逻辑错误**：技术问题与技术方案是否对得上（声称要解决的问题，方案是否真的解决了它）；前后描述是否自相矛盾；因果关系、动作先后是否说得通；是否违背物理/数学常识。
- **形式错误**：术语前后是否一致、部件/步骤命名是否统一、附图与文字是否对应、编号是否混乱等。
- **技术方案不完善**：是否存在缺环（某个部件/步骤悬空、缺少必要的连接或前置条件）、关键参数或实现方式缺失、只给了单一实施例而缺少替代方案等。

对能自行补齐的缺漏，撰写者补充完善；对涉及发明人真实技术选择、AI 无法凭空确定的关键缺环（如具体结构参数、真实实现方式），列成问题清单向用户/发明人确认，不要脑补编造。

## 第二步：按交底书内容撰写初版独权并输出（必做交付物）

体检完成、交底书方案基本可读后，**必须先按交底书内容忠实撰写一版独立权利要求，并完整输出给用户**，再进入后续创造性评估与补强。这是本skill的**第一个必做交付物**，不得跳过。

撰写要求：

- **忠实于交底书**：把交底书中发明人描述的技术问题、技术方案、核心结构/步骤/部件，如实转化为一条独立权利要求的正式文字；此阶段**不刻意拔高、不引入补强构思的新特征**，只把交底书已有内容写成规范的独权表述。
- **写成完整独权正文**：不是列提纲或骨架，而是输出可直接阅读的独立权利要求全文（含前序部分与"其特征在于"之后的特征部分）；具体措辞、句式、部件铺陈顺序等，按案件类型参考同分支下 `结构类`/`方法类`/`算法类` 的权利要求撰写规范，并同步对照通用分支下的 `权利要求主谓宾结构规范`、`权利要求指代与参照清晰性规范`、`自定义名词就地定义` 等。
- **明确标注为"交底书初版独权"**：输出时须标明这是基于交底书原文忠实转化的初版，尚未做创造性补强，便于用户对照交底书审阅、确认发明人真实技术贡献是否被完整覆盖。
- **作为后续工作的基线**：后续创造性评估、补强构思、必要技术特征核对，均以这一版独权为起点和参照；用户若对初版有修改意见，应先调整初版再继续后续步骤。

## 第三步：创造性评估（是否够发明 / 实用新型）

在初版独权输出之后，以交底书方案及该初版独权为对象，评估其创造性水平，判断适合申请哪种专利类型：

- **发明**：要求相对现有技术具有**突出的实质性特点和显著的进步**，创造性门槛更高。
- **实用新型**：要求具有**实质性特点和进步**，门槛相对较低，且仅保护产品的形状、构造或其结合（方法、材料配方等不能申请实用新型）。

评估要点：

- 结合现有技术（检索结果或用户提供的现有技术情况），判断交底书方案（及初版独权所覆盖的方案）相对现有技术的**区别技术特征**是什么、这些区别带来了什么技术效果、对本领域技术人员而言是否显而易见。
- 得出结论：创造性是否足以支撑**发明**申请；若不足以支撑发明，是否至少满足**实用新型**；若连实用新型都难以满足，则需要进入第四步补强，或与发明人沟通。
- 评估依赖检索，AI 无检索能力时按上文"能力边界"处理，只给出带不确定性标注的倾向性判断。

## 第四步：按评估结果分流

- **若创造性足够**（满足拟申请的专利类型）→ 以第二步输出的**交底书初版独权**为基础，直接进入第六步做必要技术特征核对与下放，无需刻意补强拔高。
- **若创造性不足** → 不要直接放弃或原样上报，先尝试**补强**：以交底书已有的技术问题、技术方案及初版独权为基础，沿下面两条路径深挖，重新构思出创造性更高的技术方案，再进入第五步。

### 补强路径一：深化技术问题（问题 → 方案）

- 以交底书原有的技术问题、技术方案为起点，**进一步追问更深入的技术问题**——原方案解决了表层问题，那么它还遗留了什么更本质、更难、更有价值的问题？
- **以新提出的、更深入的技术问题为出发点，重新构思技术方案**。围绕新问题构思的方案，往往能引入原交底书没有的区别技术特征，从而提升创造性高度。
- 深化后的技术问题应当能反映到独权解决的问题上，并与说明书"技术问题陈述"呼应（参见 `说明书技术问题陈述撰写`）。

### 补强路径二：从极端应用场景出发（场景 → 意想不到的效果）

- 设想该技术方案的**极端应用场景**（极端工况、极端规模、极端环境、极限性能要求等），思考在这些极端场景下原方案会暴露什么短板。
- 从极端场景的需求出发去构思技术方案，往往能逼出非常规的技术手段，进而获得**意想不到的技术效果**（超出本领域技术人员预期的效果），这正是支撑创造性的有力论据。

两条路径可以单用也可以结合。补强产出的新问题、新方案、新场景均属"建议方向"，须按能力边界交用户/发明人确认后再落地。

## 第五步：以"交底书初版独权 + AI 构思"构建补强后的独立权利要求

**仅当第四步判定创造性不足、需要补强时执行本步。** 若创造性已足够，跳过本步，直接以第二步的交底书初版独权进入第六步。

- **补强后独权 = 以第二步交底书初版独权为基础 + 以 AI 构思的技术方案（第四步补强产出的新特征/新手段）为补充**。
- 基础部分忠实于交底书发明人的真实技术贡献；补充部分是补强得到、且经用户/发明人确认可行的技术特征。两者融合，形成一条既有创造性、又落地可行的独权。
- 输出时须标明这是"补强后独权"，并与第二步的"交底书初版独权"对照，说明新增了哪些特征、各自解决什么问题、带来何种技术效果。

## 第六步：必要技术特征核对与下放从权（与已有 skill 整合）

对当前待定的独权（创造性足够时为第二步的交底书初版独权，补强时为第五步的补强后独权），**逐一核对其中每一个技术特征是否为解决本案（可能已被补强/深化过的）技术问题所必需的必要技术特征**：

- **是必要技术特征** → 保留在独权中（独权必须记载解决其技术问题所必需的全部技术特征，缺一不可）。
- **不是必要技术特征**（去掉后核心技术效果仍成立的具体实现方式、数量词、排列方式、具体机构等）→ **先从独权中摘除，下放到从属权利要求**，既让独权保护范围尽可能宽，又用从权形成梯度防御、占位阻断规避设计。

> 这一步的详细判断方法（逐一特征反问"去掉会怎样"、三类最常见可精简项、精简后方案完整性反向验证等）与本skill**重合**，不在此重复，直接调用同为通用分支的 `独立权利要求必要技术特征检查` 执行；下放后的从权写法参考对应案件类型的从权撰写规范（如 `结构类权利要求撰写`）。

## 与其他 skill 的衔接顺序

本skill位于整个撰写流程最前端，典型衔接如下：

```
本skill（交底书体检 → 按交底书撰写初版独权并输出 → 创造性评估 → 分流/补强 → 搭建补强后独权（如需） → 必要技术特征下放）
  → 专利申请文件撰写策略（在已确定独权骨架的基础上，做商业/范围/规避/布局五维度规划）
  → 通用/说明书撰写（发明内容、附图说明、具体实施例）
  → 结构类/方法类/算法类（具体权利要求文字撰写）
  → 权利要求质检/权利要求书质检（定稿复核，其中必要技术特征专项由"独立权利要求必要技术特征检查"承担）
```

- 与 `专利申请文件撰写策略` 的分工：本skill解决"这份交底书够不够格、要不要补强、独权含哪些特征"；`专利申请文件撰写策略` 解决"这件专利往哪个方向布局、防谁、如何避免被规避"。前者先行，后者承接。
- 与 `独立权利要求必要技术特征检查` 的分工：第六步的必要技术特征核对与下放，完全复用该skill，不重复其判断细则。
- 与 `说明书技术问题陈述撰写` 的分工：补强路径一深化出的技术问题，最终要落到该skill规范的技术问题陈述句式中。
