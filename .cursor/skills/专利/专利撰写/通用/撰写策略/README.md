# 通用·撰写策略

本分支用于存放**不区分结构类/方法类/算法类、处在正式落笔撰写之前**的策略层 skill：解决"这份交底书够不够格、要不要补强、这件专利往哪个方向布局"这类整体方向问题，不涉及具体某一句权利要求/说明书文字怎么措辞。

## 已有 skill

| skill | 用途 | 典型触发 |
|-------|------|----------|
| `交底书评估与创造性补强` | 拿到交底书、撰写前处理：体检（逻辑/形式/技术方案完善性）→ 创造性评估 → 分流/补强 → 搭建独权 → 必要技术特征下放 | "帮我看看这份交底书能不能写""创造性够不够""交底书不够怎么补强" |
| `专利申请文件撰写策略` | 落笔前整体方向规划：商业前景与抵御方向、保护范围与稳定性、侵权判定友好性、竞争对手规避、权利要求布局五维度 | "撰写策略""怎么布局""从战略/商业角度考虑" |

## 推荐调用顺序

```
交底书评估与创造性补强（撰写前处理，最先执行）
  → 专利申请文件撰写策略（在已确定独权骨架基础上做五维度布局）
  → 通用/说明书撰写（发明内容、附图说明、具体实施例）
  → 结构类/方法类/算法类（具体权利要求文字撰写）
  → 权利要求质检/权利要求书质检（定稿复核）
```

前者先判断"交底书能不能写、够不够格、独权含哪些特征"，后者承接后再规划"往哪个方向布局、防谁"。

新增相关 skill 时，请在本目录下按照 `.cursor/skills/专利/专利撰写/通用/撰写策略/<中文名称>/SKILL.md` 的结构添加，并更新本 README。
