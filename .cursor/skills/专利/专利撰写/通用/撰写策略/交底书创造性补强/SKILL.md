# 交底书创造性补强：产出"最有可能授权的独权技术方案细化方案"

## 这个skill要解决的问题与最终交付物（先读这一节）

客户/发明人交给你一份技术交底书，往往不完善——通常只写了发明人最先想到的那一个粗略方案，技术问题描述得笼统，实现细节缺失，场景单一。**本skill的目标不是产出一份"发散建议清单"，而是产出一份可以直接拿去撰写独立权利要求的、具体的技术方案。**

**最终交付物（必须给出，不能省略、不能只给方向）：**

一份《最有可能授权的独权技术方案细化方案》，至少包含：

1. **核心发明点**：一句话点明这件交底书里真正非显而易见、值得作为创造性支点的技术贡献（不是整套系统的罗列，而是那个"别人不容易想到"的具体点）。
2. **推定最接近的现有技术**：基于本领域公知常识推定（**注明：AI 无专利检索能力，此为常识推定而非检索结论**），说明最接近现有技术大概是什么、通常怎么做。
3. **区别技术特征**：本方案相对该推定现有技术，多出/不同的技术特征有哪些。
4. **三步法创造性自评**：对准备写入独权的方案，逐条说明"区别特征 → 实际解决的技术问题 → 为什么本领域技术人员不容易想到（无动机/有技术障碍/产生意想不到效果）"。这是判断能否授权的核心。
5. **两档独权草案**：
   - **稳妥版（授权概率优先）**：把足够的、协同的细化特征写进独权，使其明显区别于现有技术、创造性扎实，优先保证能授权；
   - **进取版（保护范围优先）**：尽量上位、特征最少的独权，作为争取更大保护范围的起点/审查博弈的起点。
   - 并明确说明两版在"授权概率 vs 保护范围"上的取舍。
6. **配套从权梯度**：把没有写进稳妥版独权的有价值细化特征，按梯度布置为从权（既是保护梯度，也是答复审查意见时把独权进一步限缩的"退路"）。

用户如果只要其中一部分（例如"只给我一版最稳的独权"），按需裁剪，但方法内核不变。

## "最有可能授权"到底由什么决定（核心方法论）

授权的实质门槛主要是**创造性**（新颖性通常更容易满足）。围绕创造性，独权的写法存在一条基本权衡：

- 独权**越宽**（技术特征越少、越上位）→ 越容易与现有技术重叠 → 创造性越难成立 → **授权概率越低**，但一旦授权保护范围越大；
- 独权**越细**（技术特征越多、越具体）→ 越容易与现有技术拉开距离 → 创造性越容易成立 → **授权概率越高**，但保护范围越小、越容易被规避。

所谓"最有可能授权的独权"，**不是无脑往独权里堆特征**，而是找到那个平衡点：

> 用**尽量少、但组合起来非显而易见**的技术特征集，使独权①明显区别于推定的最接近现有技术、②清楚地解决一个明确的技术问题、③每个特征都能得到说明书支持，从而让创造性"站得住"。

**"细化"在其中的作用（这是本skill的关键动作）**：当核心区别特征单独来看有可能被审查员认定为"公知常识/容易想到"时，不要硬扛，而是主动向独权补入与之**协同**的细化特征，使若干特征组合成一个"非显而易见的整体方案"，或使方案产生"**意想不到的技术效果**"（意想不到的技术效果是创造性最有力的论据）。这就是"把独权细化到最可能授权"的具体手段——细化的落点是创造性，而不是为细化而细化。

判断"细化到什么粒度"的准则：
- 加入某个特征后，若它对拉开与现有技术的距离、或对产生协同/意想不到效果**没有贡献**，就不要放进独权，下放到从权；
- 加入某个特征后，若它是"若删去则创造性明显变弱"的特征，就应留在独权；
- 稳妥版独权 = 恰好覆盖这些"删去则创造性变弱"的特征；进取版独权 = 只保留其中最核心的一两个，赌审查员认可其创造性。

## 补强创造性的三个着力点（都是为"充实独权/从权的有创造性特征"服务）

下面三个方面是**寻找和补充"能提高独权创造性的技术特征"的素材来源**，不是三个独立的写作任务。每挖到一个点，都要回头问：它应该进独权（稳妥版）、还是进从权、还是只写进说明书？

### 着力点一：技术问题的界定——让区别特征显得非显而易见

在三步法里，"发明实际解决的技术问题"界定得是否恰当，直接决定区别特征会不会被认定为"容易想到"。因此重新、准确地界定技术问题，是提高授权概率最省力的一步。

- **深挖到更本质的技术问题**：发明人写的常是现象层问题（"效率低""不好用"）；追问其技术根因，把问题界定到能凸显区别特征独特作用的那个层次。技术问题越贴合区别特征的独特作用，方案越显得非显而易见。
- **挖掘"隐藏的"技术问题**：同一方案可能同时解决多个层次的问题，其中一些发明人自己都没意识到。识别出来后，既能支撑从权，也能为独权的创造性提供更有力的论据。
- **让技术问题与独权区别特征严格对应**：独权里保留的每个区别特征，都应能对应到一个被清楚界定、且非显而易见的技术问题；对应不上的特征，说明它对创造性没贡献，考虑下放从权。
- **警惕把纯目的/效果/非技术问题当技术问题**。

### 着力点二：技术方案的细化——为独权补入协同的、拉开创造性距离的特征

这是本skill最主要的动作来源。针对交底书的粗略骨架逐点细化，产出可写入独权/从权的具体特征：

- **补充替代实施方式**：针对每个关键特征追问"还能用什么结构/器件/步骤/算法实现"，为上位概括和功能性限定积累实施例支撑（否则独权得不到说明书支持而无法上位）。
- **补齐实现细节到可实施**：补上被交底书省略的中间步骤、连接关系、控制逻辑，达到本领域技术人员照着能实现的程度。
- **挖掘协同关系与意想不到的技术效果**：分析特征之间"为什么配合起来有用"，找出能产生超预期效果的特征组合——这类组合往往就是稳妥版独权的核心。
- **补充参数范围与优选值**：把单点参数扩展为合理取值范围并标优选值（**具体数值须发明人确认或来自其实验，AI 不编造数据；未确认时按合理默认值给出并显式标注为待核实假设**）。
- **补充改进方案作为从权与退路**：在核心方案之上提出进一步改进，布局为从权形成梯度，同时作为答复审查意见时限缩独权的退路。

### 着力点三：具体应用场景的拓展——扩大保护范围、并催生新的可保护特征

- **横向迁移核心原理到其他领域/场景**，找出商业前景更好或更能体现普适性的场景。
- **重点找会"倒逼方案做适应性改进"的场景**：这些适应性改进本身就是新的可保护技术特征，可布为针对特定场景的从权甚至独立权利要求。单纯罗列场景对创造性帮助有限。
- **考虑把核心装置/方法上位为不受原场景限制的独立权利要求**：若核心发明点脱离交底书原场景仍成立，单独布一条不带原场景限定的独权，往往是扩大保护范围最有价值的一步（进取版独权的一种来源）。

## AI 的能力边界（重新界定：主动产出方案，只对数据守底线）

本skill要求 AI **主动、决断地给出上面那份细化方案**，这是核心价值，不能因为"稳妥"就把内容都退回发明人。同时守住一条清晰的底线：

- **可以且应当做的**：基于交底书已披露特征和公知原理，做合理技术推演，**直接给出**核心发明点提炼、区别特征分析、三步法自评、两档独权草案和从权梯度。这是撰写者"二次创造"的正常职责。
- **不可以做的（唯一硬底线）**：不编造**具体实验数据、物性参数数值、性能提升百分比、效果对比结果、现有技术检索结论**，也不把技术上根本不可行、违背物理/数学常识的"改进"当作创新点。
- **遇到需要真实数据/物性/可行性判断而交底书未给时的正确做法**：不要因此拒绝产出方案，而是**照常给出方案框架**，把其中依赖真实数据的具体数值按合理默认值填入并**显式标注为"待发明人核实的假设"**，同时在交付物末尾用一份简短的"需向发明人确认清单"集中列出这些假设点（可行性、材料/参数、真实效果数据）。

> 一句话原则：**方案框架和创造性论证由 AI 决断给出；只有"具体实验数据/物性参数/检索结论"以发明人确认为准，未确认时以显式标注的假设形式给出，绝不伪装成既定事实。**

## 标准工作流程

1. **通读交底书**，识别其"单一方案骨架 + 表面技术问题 + 单一场景"的不完善之处，以及前后不一致处。
2. **提炼核心发明点**：找出真正非显而易见的那个技术贡献。
3. **推定最接近现有技术、确定区别特征、做三步法自评**（注明检索限制）。
4. **过三个着力点**，为独权/从权收集有创造性的技术特征，逐个判断该进独权、进从权还是只进说明书。
5. **产出两档独权草案 + 从权梯度**，并说明授权概率与保护范围的取舍。
6. **末尾附"需向发明人确认清单"**：集中列出方案里按假设填入、待核实的数据/参数/可行性点。
7. 交付后如需继续，进入 `专利申请文件撰写策略`（整体布局）与 `结构类/方法类/算法类`、`通用/说明书撰写`（具体撰写），并用 `权利要求质检` 复核。

## 交付物自检清单

- [ ] 是否给出了**具体的独权技术方案**（而不是停留在"建议方向/待发明人补充"）？
- [ ] 是否点明了**唯一的核心发明点**，而不是把整套系统平铺？
- [ ] 是否推定了最接近现有技术，并做了**三步法创造性自评**（区别特征→技术问题→为何非显而易见）？
- [ ] 稳妥版独权里的每个特征，是否都对创造性有贡献（删去则创造性变弱）？有没有可下放从权的冗余特征？
- [ ] 是否同时给出了进取版独权，并说明了两版在授权概率与保护范围上的取舍？
- [ ] 从权梯度是否覆盖了独权之外有价值的细化特征，可作为审查退路？
- [ ] 独权里打算上位概括/功能性限定的地方，是否有足够实施例支撑（否则会因得不到说明书支持而不能授权）？
- [ ] 是否**没有**把编造的实验数据/物性参数/检索结论当作既定事实？依赖真实数据处是否已按假设标注并汇入确认清单？

## 一句话总结

交底书是起点不是终点：从技术问题界定、技术方案细化、应用场景拓展三个方面补强创造性，**最终落到一份具体的、最有可能授权的独权技术方案细化方案**（核心发明点 + 区别特征 + 三步法自评 + 稳妥/进取两档独权 + 从权梯度）；AI 主动决断地给出方案，只对具体实验数据/物性参数/检索结论守住"不编造、以发明人确认为准"的底线。
