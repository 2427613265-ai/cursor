# 交底书创造性补强：独权聚焦单一核心问题，纵向细化留答辩空间

## 两个必须避免的错误（先读，这是本skill的立身之本）

在给交底书补强创造性时，最容易犯、也最要命的两个错误：

1. **独权贪多**：让一条独立权利要求同时解决好几个技术问题、堆进过多技术特征。后果是**保护范围过小、容易被规避，且答复审查意见时无从退守（没有答辩空间）**——独权里已经塞满了限定，再想通过增加特征来克服创造性质疑时已经没有牌可打。
   - 正确做法：**独权只锁定一个（清单中最有创造性的）核心技术问题，用解决它所必需的最少技术特征来写，尽量做宽。**

2. **从权发散**：把清单里每个技术问题都各配一条直接依赖独权的平行从权，横向铺开一大片。后果是**从权之间互不相干、没有梯度，答辩时无退路可退**，而且真正该深挖的独权核心方案反而没被细化。
   - 正确做法：**"细化"是指围绕独权的那个核心技术问题/核心技术方案作纵向深化**，形成层层递进（逐级限缩）的从权梯度——这条梯度就是答复审查意见时的答辩退路。

> 一句话：**独权要"窄问题、宽范围"（只解决一个核心问题、特征尽量少），从权要"一条线纵向递进"（对核心方案逐级细化限缩），而不是"独权贪多、从权发散"。**

## 提高创造性的思路顺序（服务于"聚焦"）

1. **先找有创造性的技术问题，并从中选出唯一的核心问题**作为独权落点（其余问题另作处理，绝不塞进主独权）。
2. **对该核心技术问题对应的技术方案作纵向细化**（不局限于交底书、可 AI 编写），形成递进从权梯度。
3. **最后用应用场景变化**换取不同/更好的技术效果，进一步提高创造性。

## 第一步：列技术问题清单 → 选定唯一核心技术问题

- 通读交底书，列出：(A) 交底书独权明示要解决的技术问题；(B) 跳出交底书挖掘的、有创造性的技术问题（**即使某问题暂无完善方案也要列**）。评估各自创造性高低。
- **本步的关键落点：从清单中挑出创造性最高、且最适合作独权支点的那一个，定为"核心技术问题"。独权只解决它。**
  - 若交底书独权明示的问题本身创造性较高 → 以交底书独权为基础修改（但仍要按第二步把它"减负做宽"）。
  - 若某个新界定的问题更有创造性 → 以它为核心、先初步撰写对应独权。
- **其余问题分类处理，避免它们污染独权或散成平行从权：**
  - **与核心同一技术线**（是核心方案的进一步限定、优选、协同细化）→ 留作第二步的**梯度从权素材**。
  - **与核心不同方向、但同样有较高创造性**→ 考虑布为**并列的独立权利要求**，或记入说明书/留作另案；不要塞进主独权，也不要作为一堆平行从权喧宾夺主。
  - 创造性不足、只是常规选择的 → 记入说明书即可。

自检：
- [ ] 是否明确选出了**唯一**核心技术问题，而不是让独权背上一串问题？
- [ ] 非核心问题是否已分流（并列独权 / 说明书 / 另案），而不是全塞进从权？

## 第二步：写"宽"独权 + 对核心方案作纵向细化的从权梯度

### 2.1 独权：只解决核心问题，特征精简，留答辩空间

- 独权 = 解决核心技术问题所必需的**最少技术特征集**。
- **逐个特征做"删除测试"**：删掉某特征后，独权是否仍新颖、仍能解决该核心问题、仍有创造性？若删掉仍成立，就把它移出独权、下沉到从权梯度。
- **单一问题检验**：如果一条独权实际在解决好几个技术问题，就是贪多了——把非核心问题对应的特征拆到从权或并列独权，让主独权回到只解决一个核心问题的状态。
- 记住：独权此刻**故意留白**（不把所有细化特征写满），正是为了在答复审查意见时有特征可加、有退路可退。

### 2.2 从权：沿核心方案纵向递进，构成答辩退路

- 从权沿"独权核心技术方案"这一条线**逐级细化、逐级限缩**：从权 2 在独权上加一个限定，从权 3 在从权 2 上再加一个，层层递进（引用结构上串联依赖，形成梯度）。
- 这条梯度的作用是**答辩退路**：独权被驳 → 退到从权 2 → 再退到从权 3……每退一级，多一点区别特征、创造性更强一点，直到能克服审查意见为止。
- **对"独权核心技术问题对应的技术方案"要真正做深、做细**（实现细节、优选结构/器件、参数取值区间、特征间协同机理、优选实施方式），这些纵深细化点就是梯度的各级——这才是用户要的"对独权技术方案作细化"，而不是绕开核心去补别的问题。
- **反面示例（要避免）**：从权 2 解决 A 问题、从权 3 解决 B 问题、从权 4 解决 C 问题，且都直接依赖独权、彼此平行——这是"发散"，答辩时没有可逐级退守的梯度。若确有互不相干但有价值的改进，用**少量**并列从权或并列独权承载，但主线必须是那条纵向梯度。

自检：
- [ ] 独权是否只锁定一个核心技术问题？有没有因贪多而范围过小？
- [ ] 独权中任一特征做删除测试后是否仍成立（即是否已足够宽、留了白）？
- [ ] 从权是不是沿核心方案纵向递进的梯度（可逐级退守），而不是每问题一个的横向平行从权？
- [ ] 是否真正把独权核心技术方案做了纵深细化，而不是把精力散到别的技术问题上？
- [ ] 答复审查意见时是否有清晰的逐级答辩退路？

## 第三步：改变、丰富应用场景，进一步提高创造性

在独权与从权梯度成型后，用应用场景做最后的创造性增益：

- 改变或丰富应用场景，看方案能否实现**不同的技术效果**（解决新场景特有问题）或**更好的技术效果**（同一效果更突出）。
- 新场景倒逼的适应性改进是新的可保护特征——但仍按第一步的原则判断：能纳入核心技术线的作梯度从权，方向不同的作并列独权，不要又散开。
- 若核心方案脱离原场景仍成立，可把核心装置/方法上位为不受原场景限制的独立权利要求，扩大保护范围。

## AI 的能力边界（简短，但要守住）

本skill鼓励 AI 主动编写、补充技术方案（尤其第二步的纵深细化），不要只复述交底书。唯一底线：AI 编写的方案须技术上成立、不违背物理/数学常识；**不编造**具体实验数据、物性参数数值、性能提升百分比、效果对比结果、现有技术检索结论；依赖真实数据而交底书未给时，按合理默认值写入并**显式标注为"待发明人核实的假设"**，集中列入交付物末尾的确认清单，绝不伪装成既定事实。

## 交付物（聚焦，不发散）

1. **技术问题清单** + **选定的唯一核心技术问题**（说明为何选它、非核心问题的去向）。
2. **一条尽量宽的独权**：只解决核心问题、特征精简，并说明"留白/答辩空间"体现在哪里。
3. **一条纵向递进的从权梯度**：逐级限缩、作为答辩退路，逐级说明每一级加了什么区别特征、对应加强了哪一点创造性。
4. **非核心创造性问题的处理去向**（并列独权 / 说明书 / 另案），必要时给出并列独权草案。
5. **场景拓展带来的新技术效果**及相应权利要求安排。
6. **需向发明人确认清单**：集中列出按假设填入、待核实的数据/参数/可行性点。

## 一句话总结

补强创造性 = **先选出唯一最有创造性的核心技术问题 → 独权只解决它、特征精简做宽、留足答辩空间 → 对该核心技术方案作纵向细化形成层层递进的从权梯度（答辩退路）→ 最后用应用场景换取不同/更好的技术效果**；切忌"独权贪多致范围过小、从权按问题平行发散致无退路"，唯一底线是不编造具体实验数据/物性参数/检索结论。
