# 专利申请文件撰写策略：六个维度

## 适用场景

- 在正式落笔撰写某一件专利申请具体权利要求/说明书文字**之前**，或撰写过程中需要跳出文字细节、重新审视整体方向时使用。
- 解决的是"这件专利应该往什么方向布局、覆盖哪些方案、防谁"这类**策略层**问题，不涉及具体某一句权利要求怎么措辞。
- 不区分结构类/方法类/算法类，六个维度对任意技术方案类型都适用；具体到某一类技术方案的文字撰写规范，参考同级的 `结构类`/`方法类`/`算法类` 分支，具体到说明书某一固定段落的写法，参考同级 `通用/说明书撰写` 分支。
- 与 `权利要求质检/patent-claims-quality-review` 的区别：质检是对**已经写完**的权利要求书做复核；本 skill 是在**动笔之前**做策略规划，二者可以配合——先用本 skill 规划方向，写完后再用质检 skill 复核落地效果。

## 维度一：适当的保护范围——检索在先、上位概括、多实施例支撑

- 撰写前应先充分检索国内外专利文献，摸清现有技术水平，据此确定权利要求的保护范围边界，让专利具备更高的稳定性——理想情况下，撰写者应该能大致"预测"审查阶段可能被检索出的对比文件。
- 撰写专利申请文件的过程是一次"二次创造"：需要和发明人一起讨论解决技术问题的**各种实施方式**，把具体方案的技术特征进行概括、抽象（上位概括或功能性限定），才能确定权利要求，而不是把发明人提供的单一具体方案照抄进独权。
- 上位概括/功能性限定需要**足够多的实施例**支撑，否则权利要求可能得不到说明书支持；功能性特征在侵权判定阶段会被解释为说明书披露的具体实施方式及其等同方式，而"是否等同"本身容易产生争议，所以撰写阶段就应该尽量挖掘更多的实施方式，为将来的解释留出空间。

自检：
- [ ] 撰写前是否已经做过现有技术检索？检索结果是否已用来调整独权的保护范围？
- [ ] 是否已经和发明人一起讨论过，除了最初方案外还有哪些替代实施方式？
- [ ] 说明书中的实施例数量是否足以支撑权利要求里的上位概括表述？
- [ ] 权利要求中用到功能性限定的地方，说明书是否给出了不止一种具体实施方式？

## 维度二：从有利于侵权判定的角度撰写

- 基于**全面覆盖原则**，独立权利要求只写解决技术问题所必需的最少技术特征集；任何非必要技术特征写进独权，只会不必要地缩小保护范围，应当放到从属权利要求中。
- 容易被忽视、但实际会缩小保护范围的写法，撰写时要仔细斟酌是否真的必要：
  - 发明名称本身带来的隐含限定
  - 使用环境特征
  - 以制备方法界定产品的技术特征（product-by-process）
  - 方法权利要求中的步骤顺序限定
- 权利要求中如果出现了容易产生歧义、而说明书里又没有解释的术语，会给日后侵权判定阶段对权利要求的解释造成障碍——应当在说明书中提前给出清楚的定义。

自检：
- [ ] 独权中的每一个技术特征，是否都是解决其声称技术问题所必需的？删掉是否仍不影响创造性？
- [ ] 是否存在发明名称、使用环境限定、product-by-process、方法步骤顺序等可能不必要缩小保护范围的写法？是否已仔细斟酌其必要性？
- [ ] 权利要求中有没有可能引起歧义的术语？说明书中是否已给出明确定义？

## 维度三：商业前景

- 和发明人沟通的过程中，留意技术方案是否可以应用到比最初设想**更有商业前景**的领域（例如方案最初只考虑PC端，但发现移动端应用前景更可观）。
- 一旦发现潜在的更优应用场景，要评估技术方案迁移到新场景是否需要调整（新场景可能对技术方案提出不同要求），并据此**及时调整申请策略**——例如是否需要针对新场景补充撰写方案或单独布局权利要求。

自检：
- [ ] 除发明人最初描述的应用场景外，是否讨论过其他潜在应用领域？
- [ ] 若技术方案可迁移到其他场景，是否需要针对该场景调整技术方案、补充说明书内容或增加权利要求？

## 维度四：是否容易被竞争对手规避

- 与申请人一起主动拓展两类方案：
  - **改进技术方案**：可以作为专利审查阶段应对审查意见的"退路"（用于修改权利要求或补充为从权），同时也能提前占位、阻止竞争对手把这个改进方案抢先申请专利。
  - **变劣技术方案**：根据北京市高级人民法院《专利侵权判定指南》，变劣技术方案不构成等同侵权，因此撰写时也要一并考虑覆盖变劣方案，避免竞争对手用"故意变劣"的方式规避侵权。

自检：
- [ ] 是否已经和申请人讨论过针对当前方案的可能改进方向？是否已经布局为从权，或至少记录在说明书里作为审查阶段的"退路"？
- [ ] 是否讨论过可能的"变劣"替代方案？是否需要额外的从权覆盖，防止竞争对手借此规避？

## 维度五：有利于取证维权

- 若申请人从事产品生产业务，从侵权赔偿角度出发，**以产品制造商为主要抵御对象**撰写权利要求可能对权利人更有利——因为销售商、进口商、使用者通常更容易主张"合法来源"抗辩，难以有效追责。此时可以考虑同时撰写**产品权利要求**和**产品制备方法权利要求**。
- 在通信、网络、计算机等领域，侵权主体可能涉及制造商、销售商、使用者、平台方等多方，撰写前应先明确**希望抵御的具体对象**，再针对性设计权利要求（不同的抵御对象可能对应不同的权利要求设计）。
- 一般而言：
  - 不需要拆解产品、消费者正常使用即可获知的技术特征，更有利于取证。
  - 需要通过反向工程才能确认的技术特征，要评估反向工程的难度和成本：
    - 化学领域：优先选择能用普通测量仪器/常规技术验证的技术特征。
    - 半导体/芯片领域：反向工程容易实现时，可以详细描述元件及其连接方式；反向工程难度大或成本高时，需要考虑调整撰写方式（转向更容易从外部观测、验证的技术特征）。

自检：
- [ ] 是否已经明确本专利主要希望抵御的侵权主体（生产商/销售商/使用者/平台方）？
- [ ] 权利要求中的技术特征，是否需要拆解产品或依赖反向工程才能验证？验证难度和成本如何？
- [ ] 是否考虑了产品权利要求 + 制备方法权利要求的组合，以覆盖不同的侵权主体？

## 维度六：布局策略

- 重视**从属权利要求**的价值：把独权之外、仍具备新颖性和创造性的技术特征布置到从权中，用于阻断竞争对手的规避设计，不要为了追求独权"简洁"而把有价值的区别特征全部舍弃。
- 当某些技术特征之间**互不兼容**（比如两种互斥的实施方案）时，可以考虑构建多条保护范围不同的独立权利要求分别覆盖，避免将来因为修改（例如答复审查意见时被迫删除或合并权利要求）而被迫放弃其中一部分保护范围。
- 方法权利要求和产品权利要求的保护范围可以**有意设计出层次差异**（例如方法权利要求的保护范围大于产品权利要求），以实现不同层次的保护，而不是让两者的保护范围完全重合。

自检：
- [ ] 从权是否已经覆盖了独权之外、具备新颖性和创造性的技术特征，用来阻断规避设计？
- [ ] 是否存在互不兼容的技术特征？是否需要拆分成多条独立权利要求分别保护，而不是勉强合并成一条？
- [ ] 方法权利要求与产品权利要求的保护范围，是否经过有意识的层次化设计？

## 使用建议

在正式撰写某一件专利申请的具体文字之前，建议按以上六个维度逐一过一遍，和申请人/发明人确认清楚后再形成撰写策略，进入具体的权利要求和说明书撰写阶段。具体到某一类技术方案的文字撰写规范，参考同分支下的 `结构类`/`方法类`/`算法类` 以及 `通用/说明书撰写` 目录下的 skill；写完后可以用 `权利要求质检/patent-claims-quality-review` 做整体复核。
