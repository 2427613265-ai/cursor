# 说明书"背景技术"部分撰写规范（所有案件类型通用）

## 段落结构（四段式）

背景技术部分按以下四段展开，段落顺序和各自的作用不能颠倒：

### 第一段：具体使用场景

交代一个具体、真实的应用场景/行业背景，**不能写得抽象笼统**。场景要写到让读者立刻理解"这个技术具体用在什么情境下"，包含必要的环境/工况特征描述（比如"高压富水断层发育地带""深部复杂地质环境"这种具体到能让人感受到难度和特殊性的措辞），避免只写"在XX领域中"这类放之四海而皆准的空话。

### 第二段：现有技术方案（纯客观描述，不夹带任何评价）

概述现有技术里针对这个场景采用的1～2类主流做法（如果存在多种技术路线，逐一介绍）。**这一段只客观描述现有技术是什么、怎么做的，不评价好坏、不提前透露任何不足**——所有的问题和缺陷都留到第三段"然而"之后才第一次提出。

**这一点很重要，容易写错**：如果在这一段里提前把每种做法的缺点都写了，第三段开头的"然而"就没有新内容可以转折，读起来会显得重复、生硬，"然而"这个转折词也失去了意义。第二段和第三段的分工必须清楚——第二段负责"是什么"，第三段负责"有什么问题"，不能混在一起。

**不要编造或引用具体的专利文号/公告号**（比如"专利公告号CNxxxxxxxU"）。除非用户明确提供了经过核实的对比文件及其准确文号，否则一律用"现有的XX装置""一部分现有技术""目前常见的做法"这类概括性表述来描述某一类做法，不要虚构一个看似真实、实际未经核实的专利号——写错或编造文号是实质性错误，比不写具体文号的风险大得多。

### 第三段：用"然而"转折，第一次提出2～3个技术问题

承接第二段客观描述的现有技术，用"然而……"开头，**第一次**引出对现有技术的评价和批评，把问题归纳为**2～3个更明确、更具体的技术问题**，通常用"一方面……另一方面……"或分条的方式组织。

写每一个技术问题时，要能让读者看出它对应的是第二段里哪一类现有做法（比如"采用单一处理头转动扫描的装置，由于……"），保持前后呼应，但**呼应的内容本身必须是第一次出现，不能是第二段已经说过的话原样重复**。

**这2～3个技术问题，必须和本专利权利要求实际解决的技术问题一一对应**——不能写一堆背景技术缺陷，但独权/从权的技术方案根本没有针对性地解决；也不要为了追求"问题多"而堆砌无关的缺陷。撰写之前要先想清楚：独权解决了哪个问题，排在前面的关键从权分别解决了哪个问题，再回来对照着写这一段。

### 第四段：固定收尾句

单独一句，格式固定：

**"鉴于此，有必要提出（专利名称）以解决或至少缓解上述缺陷。"**

这里的"（专利名称）"必须与"技术领域"部分、权利要求书标题三处保持逐字一致（参见同分支下的 `说明书技术领域部分撰写`）。

## 语言风格：自然、通顺，避免生硬堆砌

- **句子长度适中**：不要把过多的定语、状语、并列分句硬塞进同一句话里，读起来喘不过气。一句话尽量只表达一个核心意思，多个意思拆成多句，用句号或分号分开，而不是全部揉进一个长句。
- **优先用日常、行业内自然的表达和常见连接词**（"而""但""不过""并且""此外"等），不要为了显得"专业"就堆砌生僻、过度书面化或刻意拗口的修饰语和连接词。
- **逻辑推进要顺畅、符合自然的叙述节奏**：场景→现有做法→具体不足→归纳问题→提出方案，读起来应该像是在讲一件事的来龙去脉，而不是机械地套模板、每一句都刻意对应某个"规定动作"。写完之后通读一遍，如果感觉某句话是"为了完成格式要求而硬凑出来的"，就要重新组织语言。

## 参考模板（用户提供的隧道施工案例，供对照句式和段落节奏）

> 在地下工程及交通基础设施建设中，随着隧道掘进作业不断向深部及复杂地质环境延伸，不可避免地会穿越高压富水断层发育地带。在这种极端水文地质条件下进行掘进施工时，工作面往往面临突发性围岩水压力和持续的渗水。
>
> 目前，在应对此类高压富水断层时，现有的隧道施工多采用单一的以堵为主或以排为主的常规处治工法。其中，以堵为主的施工方法通常是在隧道主洞开挖轮廓外围一次性实施注浆封堵，并施作重型衬砌结构，试图通过增强各施工阶段的结构刚度来强行抵抗周边围岩水压，但是否具备开挖条件往往依赖施工经验进行主观判断，缺乏客观量化的判定依据，也缺少与主洞掘进进度动态协同、循环推进的注浆工艺；而以排为主的施工方法则多采取被动式作业策略，主要依赖在主洞工作面敷设抽排管路或利用常规水泵进行积水抽排，即便设置辅助导洞，也多止步于简单的超前探测，未能形成独立完整、可将渗水引导至洞外的立体排水通道，更缺乏针对高压水流跌落冲击的消能防护结构。
>
> 然而，上述常规施工方法在实际面临持续且巨大的高压补给水源冲击时，存在显著的工序协同与施工作业空间局限性。一方面，单一的封堵工序无法在主洞开挖前实现地层水压的提前释放，且缺乏客观、可量化的安全判据支撑开挖时机的确定，致使水压力随施工推进不断在支护结构外侧聚集，容易在后续作业期间引发涌水及支护破坏；另一方面，单纯依赖主洞内部的被动响应式排水工法，难以在主洞掘进前对前方水文地质危险源形成预知，也难以随掘进进程动态圈定断层破碎带的具体位置；大量的水流汇集与抽排作业只能被迫与主洞开挖作业挤占同一有限的工作空间，导致排水与掘进工序严重交叉干扰，容易引发全线作业的停滞；且排水路径与主洞掘进空间未完全分离，各排水构件之间又缺乏有效的消能与沉淤设计，容易导致高压涌水携带泥沙倒灌、冲毁排水通道的施工隐患。
>
> 鉴于此，有必要提出一种高压富水断层发育隧道的施工方法以解决或至少缓解上述缺陷。

对照分析：
- 第一段只交代场景（深部隧道掘进、高压富水断层），不涉及任何技术方案或缺陷。
- 第二段介绍了"以堵为主"和"以排为主"两类做法。
- 第三段用"然而……一方面……另一方面……"把问题归纳成了三个更聚焦的技术问题。
- 第四段是固定句式，专利名称与题目一致。

> **注意**：这份参考模板的第二段里，其实紧跟着每种做法的介绍就顺带提了一句该做法的不足（比如"但是否具备开挖条件……缺乏客观量化的判定依据"）。经实践发现，这种写法容易导致第三段"然而"开头显得重复、转折生硬——因为问题已经在第二段说过一遍了。**因此本skill的正式规则（见上面第二段/第三段的说明）与这份模板的具体处理略有出入：以上面"第二段纯客观描述、第三段才第一次提出问题"的规则为准，不要机械照抄这份模板里第二段夹带缺点的写法。**

## 撰写/审核检查清单

- [ ] 第一段的场景是否具体（删掉行业术语后是不是就变成了放之四海而皆准的空话）？
- [ ] 第二段是否只客观描述了现有技术"是什么"，完全没有提前透露任何不足/问题？
- [ ] 第三段的每个技术问题，是不是第一次出现，而不是第二段已经写过的内容原样重复？
- [ ] 第三段归纳的技术问题数量是否落在2～3个（太少显得创造性单薄，太多则显得权利要求覆盖不全或背景技术堆砌）？
- [ ] 第三段的每一个技术问题，能不能在权利要求书里找到对应解决它的技术特征（尤其是独权和排在前面的关键从权）？
- [ ] 结尾句的专利名称，是否和"技术领域"部分、权利要求书标题完全一致？
- [ ] 有没有编造/引用未经用户核实的具体专利文号？
- [ ] 通读一遍，句子是否自然通顺，有没有堆砌过长、过度书面化的长句？

## 应用范围

本规则适用于撰写或审核任意类型专利说明书的"背景技术"部分。与 `说明书技术领域部分撰写`、`专利/权利要求质检` 下的 `权利要求书质检` 配合使用：建议先质检权利要求书、明确独权和关键从权分别解决什么技术问题，再撰写"技术领域"和"背景技术"两部分，确保专利名称、技术问题与权利要求技术特征三者相互对应。
