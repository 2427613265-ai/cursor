# 具体实施方式：先释义新名词，再展开实施例

## 核心规则

在"具体实施方式"部分为**某一条权利要求**撰写实施例时，如果这条权利要求里含有**不清楚的、新定义的名词**（撰写者自造的术语、被赋予特殊含义的普通词、本发明特有的参数或量），必须按以下顺序动笔：

```
第一步：先逐一解释这条权利要求涉及的每一个新定义名词——它是什么、怎么计算、由哪些已知量构成
第二步：名词都解释清楚后，再按"复述权利要求 → 具体而言展开"的方式撰写该条权利要求的具体实施例
```

**不要**在还没解释名词的情况下直接开写实施例，逼着读者一边读实施例一边猜名词含义；也不要把名词的解释拖到实施例后面才补。

## 为什么要先释义

- 实施例是用来把权利要求讲透的，如果实施例里用到的核心名词读者还不理解，整段实施例就读不通，起不到"讲透"的作用。
- 先把名词钉死，后面展开原理和操作流程时可以直接引用这些名词，行文连贯、不必反复打断去解释。
- 说明书里对新名词的这段解释，也正是权利要求中该名词"得到说明书支持"的落脚点。

## 与权利要求"其中"句就地定义的分工

- 权利要求内部：非公知的自定义名词要用"其中，……"句**就地给出简明定义**（见 `自定义名词就地定义`），保证权利要求自足可读。
- 说明书具体实施方式：在此基础上**更充分地展开解释**——给出计算方式、取值范围、物理含义、举例，让名词的内涵彻底清楚。
- 两者不重复也不矛盾：权利要求是精炼的法律定义，说明书是详细的技术解释；本 skill 要求的就是把这段详细解释放在对应实施例的**最前面**。

## 撰写结构示范

```
【先释义】
本实施例中，所述调节比是指调节杆的伸出长度与壳体高度的比值；该比值用于表征调节杆
的伸出程度，比值越大表示伸出越多。在本发明中，当该比值达到1.5及以上时，调节杆已进
入需要锁定的行程区间。

【再展开该条权利要求的实施例】
（复述权利要求）……当所述调节比大于1.5时，锁定所述调节杆；
具体而言，……（按"具体而言"展开原理与操作，此处可直接使用"调节比"一词，无需再解释）
```

即：新名词的释义段落排在该条权利要求实施例（复述+展开，见 `说明书具体实施方式撰写`）之前，作为读懂后续实施例的前提。

## 检查清单

- [ ] 该条权利要求涉及的每一个新定义名词，是否都在对应实施例**之前**先解释清楚了？
- [ ] 有没有出现"名词还没解释、实施例里就直接使用"的情况？
- [ ] 有没有把名词解释错误地拖到实施例之后才补？
- [ ] 释义是否比权利要求"其中"句更充分（给出了计算方式/取值/物理含义/举例），而不是简单照抄？
- [ ] 释义中是否又引入了别的未解释的新名词（避免"用未定义解释未定义"）？

## 应用范围

适用于任意技术方案类型（结构类/方法类/算法类）说明书"具体实施方式"部分的撰写与质检。撰写时按"先释义、后展开"的顺序逐条动笔；质检说明书时，作为 `说明书质检/说明书具体实施方式清楚性与逻辑一致性规范` 的检查项之一，核对新名词是否在对应实施例之前先行解释。与 `说明书具体实施方式撰写`（复述+展开的两段结构）配合使用：本 skill 负责"释义排在实施例之前"这一顺序，后者负责实施例本身"怎么复述、怎么展开"。
