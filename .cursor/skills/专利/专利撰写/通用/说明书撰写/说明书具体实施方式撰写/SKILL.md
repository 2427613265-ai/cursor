# 说明书"具体实施方式"部分撰写规范（所有案件类型通用）

## 核心结构：先复述，再展开

撰写具体实施方式时，针对每一条权利要求（或方法权利要求里的每一个步骤），按以下两段展开：

### 第一段：复述权利要求的技术方案（必须原样复述，不得改动）

**每一条权利要求的具体实施例，第一段都必须原样复述该权利要求的技术方案，不得改动、删减、增补或调整其技术特征的文字表述**——照抄权利要求书里的文字，**保留"所述"这种严谨、结构化的措辞风格**，作为这部分实施例的开头句。这一段是实施例与权利要求逐字对应的落脚点，不允许在此段做任何"顺句子""改措辞"的加工，任何解释、补充、附图标记都留到第二段展开时再写。

**唯一的例外是"其特征在于"这个措辞要删除**："其特征在于"是权利要求的形式用语（用于划分前序部分与特征部分），不是技术特征本身，复述到说明书时应当去掉，让句子直接连成通顺的技术叙述。删除"其特征在于"不算改动技术特征，除此之外的技术方案文字仍要原样照抄。

- 段首可加固定引导语，例如"在本实施例中，"或"在本实施例中，如图X（至图Y）所示，"，随后紧跟原样复述的权利要求文字；引导语只是框架，不改变被复述的权利要求内容本身。
- 方法类专利：常用"S+数字、XXX"的步骤编号格式，直接原样复述该步骤对应权利要求的文字。
- 结构类专利：复述"（一种XXX，）包括XXX；所述XXX设置在……"这种结构表述（去掉"其特征在于"），不需要编号。
- 从属权利要求：第一段同样复述该从权全文，**保留"根据权利要求X所述的……"的引用关系**（同样删除其后的"其特征在于"），不得省略引用部分。

### 第二段：用"具体而言，"过渡，逐条展开

第二段固定用过渡语"**具体而言，**"引出详细说明（"在本实施例中，""如图X所示，"等属于第一段复述的段首引导语，不要拿到这里当展开段的过渡语混用），然后**严格按照第一段复述内容里技术特征出现的先后顺序，逐一展开**：

1. 这个技术特征**为什么**要这样设计——解决什么问题、起什么作用、如果不这样做会有什么后果；
2. 结合附图标记，具体**怎么**实施——真实的结构细节、操作流程、部件之间的配合关系；

**不能遗漏**复述部分提到的任何一个技术特征，也**不能在展开时夹带**复述部分没有出现过的全新技术特征——如果确实需要补充一个复述部分没有的优选细节，必须用"进一步地""在一个可选的实施方式中"等词单独标注，跟主线内容分开，不能悄悄混进去当成权利要求本身就有的内容。

### 两段的语言风格差异

- 第一段（复述）：保持跟权利要求书一致的严谨书面语，不需要额外解释。
- 第二段（"具体而言"展开）：可以放开写得更自然、更叙事化，像是在解释工程原理和实际操作流程，语言风格参照 `说明书背景技术部分撰写` 里"语言风格"章节的要求（句子长度适中，避免生硬堆砌）。

## 从属权利要求的实施例写法

独立权利要求之外，**每一条从属权利要求也要各自撰写对应的实施例**，同样遵循"复述+展开"的两段结构，但展开部分的重点不同：

1. **第一段**：复述该从权全文（保留"根据权利要求X所述的……"的引用关系，删除其后的"其特征在于"），除删除"其特征在于"外不得改动技术特征文字。
2. **第二段"具体而言，"**：**只展开该从权相对于其引用的基础权利要求新增或进一步限定的技术特征**，不要把基础权利要求里已经详细说明过的共同特征重新展开一遍；确需带到基础特征时，用"如前所述""如上所述"一句带过即可。
3. 若从权限定的是**数值范围、比例、数量关系、角度**等参数，展开时应说明三件事：为什么要落在这个范围内、低于下限或高于上限分别会带来什么不利后果、在该范围内能取得的技术效果——把参数的取值依据讲清楚，而不是只重复一遍数字。
4. 组织方式：通常以"实施例1对应独立权利要求"为主线，各从属权利要求可逐条对应追加段落；若多条从权属于同一优选实施方式，可在同一段里用"进一步地"合并叙述，但不得遗漏任何一条从权的限定特征。

## 附图引用与附图标记规范

具体实施方式要结合附图撰写，引用附图和附图标记时遵循以下写法：

- **何时写"如图所示"**：描述整体方案或某条实施例开头时，用"如图X所示"或"如图X至图Y所示"（放在第一段复述的段首引导语中，如"在本实施例中，如图1至图3所示，……"）；单独描述某个局部结构时，就近标注该结构所在的图，如"如图3所示，所述导流齿9设置于所述环形外沿4的外侧面"。
- **附图标记格式**：统一采用"**部件名称+标号**"的连写形式（如"环形外沿4""导流齿9"），**不要写成"4、环形外沿"这种附图标记说明里的列举格式**。附图标记只出现在第二段展开中；第一段是原样复述的权利要求文字，权利要求本身不带标号，故第一段不加附图标记。
- **一致性**：实施例中用到的每一个附图标记，其编号和名称都必须与 `说明书附图说明撰写` 里的"附图标记说明"完全一致，不得自创、错位或引用一个附图说明里不存在的标号。

## 与"有益效果"的分工边界

具体实施方式在展开时会说明"这个特征起什么作用"，但要与"实用新型内容/发明内容"里的"有益效果"分工清楚：

- 具体实施方式重点写**结构关系、连接配合、工作原理、如何实现**，可以顺带点出该特征在运行中起的作用；
- 但不要整段照搬"有益效果"部分的宣传式表述——有益效果的归纳留在发明内容部分，实施例只服务于**充分公开和支持权利要求**这一目的。

## 参考示范一：方法类（用户提供的隧道施工案例）

> S2、当所述地下水状态满足预设的安全指标时，对所述主洞隧道的围岩进行帷幕注浆；开挖所述主洞隧道，并施作衬砌；并在所述主洞隧道内施作中心水沟；其中，所述主洞隧道沿所述泄水洞平行施作，且所述主洞隧道的底面标高高于所述泄水洞的底面标高；
>
> 具体而言，在所述泄水洞3持续进行泄排水与超前探测的过程中，需对主洞隧道1预掘进区段的地下水状态进行多维度客观评估，判断其是否已下降并稳定至预设的安全指标；只有在确认满足所述安全指标后，方可启动主洞隧道1的后续施工作业，从而避免在地下水压力仍处于高位时贸然施工而引发涌水、突泥等灾害性事故。
>
> 在确认满足所述安全指标后，首先针对主洞隧道1待开挖段的围岩实施帷幕注浆，通过预先注浆固结的方式在主洞隧道1开挖轮廓外围构建起加固阻水防线，避免在后续开挖过程中围岩因卸荷或渗水作用而发生失稳破坏。
>
> 待帷幕注浆固结完成后，沿所述泄水洞3的走向平行开挖施作主洞隧道1，且在施作过程中，通过测量放样控制使主洞隧道1的底面标高始终高于泄水洞3的底面标高，从而在两者之间构建起稳定的高差关系，为后续横向排水管网提供重力自流的高差基础，无需依赖机械抽排即可实现渗水的顺畅导排。
>
> 在主洞隧道1开挖成型后，随即在其内部施作衬砌，以对开挖揭示的围岩提供支护并阻隔残余渗水，保障主洞隧道1的结构安全与长期运营稳定。最后，在主洞隧道1底部的纵向中心线上施作中心水沟2，中心水沟2采用现浇混凝土结构沿主洞隧道1全线连续设置，用于收集汇聚经衬砌渗入主洞隧道1内部的残余渗漏水，构建起主洞隧道1内部渗漏水的初始汇流通道。

对照分析：第一段复述了S2步骤的四个技术特征（①判断地下水安全指标、②帷幕注浆、③平行开挖且底面标高高于泄水洞、④施作衬砌和中心水沟）；第二段严格按①②③④的顺序逐一展开，每一点都先说"为什么"（避免涌水突泥/避免失稳破坏/无需机械抽排即可导排/保障结构安全并收集残余渗漏水），再说"具体怎么做"，且引入了复述部分没有点名的附图标记（1、2、3）和补充细节（现浇混凝土结构、全线连续设置），这些细节是对复述内容的具体化，不是引入新的技术特征。

## 参考示范二：结构类（本项目喷雾降尘装置独权示例）

> 一种喷雾降尘装置，包括外罩、风叶和降尘组件；所述风叶安装在所述外罩的内壁，能够转动以产生轴向气流；所述降尘组件设置在所述风叶的一侧，所述降尘组件包括环管和多个喷雾管；所述环管同轴设置在所述风叶的出风侧；多个所述喷雾管呈放射状固定连通于所述环管，每个所述喷雾管的管壁上开设有多个喷孔；所述喷孔的开口方向与所述风叶所在平面之间形成有夹角。
>
> 具体而言，外罩1的内壁安装有风叶2，风叶2由电机驱动转动，转动过程中沿转轴方向产生轴向气流，用于将后续雾化水体向外罩1前方及外部推送。风叶2的一侧设有降尘组件，降尘组件包括环管3和多个喷雾管4：环管3同轴设置在风叶2的出风侧，用于承接外部输入的加压水源，并向各喷雾管4分配水流；多个喷雾管4呈放射状固定连通于环管3，每个喷雾管4的管壁上开设有多个喷孔5，用于将环管3内的水流雾化喷出。喷孔5的开口方向与风叶2所在平面之间形成有夹角，使喷孔5喷出的雾化水体能够获得风叶2产生的轴向气流的风压推动，在自身水压的基础上叠加获得额外的推进力，从而增大雾化水体的有效喷射距离，扩大对设备前方及周边粉尘扩散区域的覆盖能力。

对照分析：第一段直接复述权利要求原文；第二段按"风叶→降尘组件（环管→喷雾管→喷孔）→喷孔夹角"的原有顺序逐一展开，每引入一个部件就说明它的作用（风叶产生轴向气流用于推送水雾、环管分配水流、喷雾管雾化喷出、夹角使雾化水体获得风压推动增大射程），并补上了附图标记（1～5），没有引入复述部分没有的新技术特征。

## 检查清单

- [ ] 每一条权利要求的实施例，第一段是否原样复述了对应权利要求（或步骤）的技术方案，除删除"其特征在于"外没有改动、删减措辞，也没有遗漏任何技术特征（从权是否保留了"根据权利要求X所述的……"引用关系）？
- [ ] 第一段复述是否已删除权利要求形式用语"其特征在于"，使句子连成通顺的技术叙述？
- [ ] 第二段"具体而言"之后的展开，是否覆盖了复述部分提到的每一个技术特征，且顺序保持一致？
- [ ] 从属权利要求的实施例，第二段是否只展开了该从权新增/限定的特征，而没有把基础权利要求已详细说明过的共同特征重复展开？
- [ ] 数值范围、比例、数量、角度类从权，是否说明了取值依据、上下限的不利后果以及在该范围内的技术效果，而不是只重复数字？
- [ ] 有没有在展开时不加区分地引入复述部分没有的新技术特征？如果有，是否用"进一步地/在一个可选的实施方式中"等词单独标注、与主线内容分开？
- [ ] 是否在合适位置使用了"如图X所示"/"如图X至图Y所示"？附图标记是否统一为"部件名称+标号"（如"环形外沿4"）的连写格式，且编号、名称与附图标记说明完全一致？
- [ ] 过渡语是否规范：第一段段首用"在本实施例中，（如图X所示，）"，第二段展开固定用"具体而言，"，二者没有混用？
- [ ] 第二段的语言是否自然流畅，不是干巴巴地重复第一段的措辞，而是加入了"为什么这样做"的解释？
- [ ] 有没有把发明内容里的"有益效果"整段照搬进实施例？

## 应用范围

本规则适用于撰写或审核任意类型专利说明书"具体实施方式"部分。与 `专利/权利要求质检` 下的 `权利要求书质检` 配合使用：建议先确认权利要求书逐条定稿，再按本skill逐条撰写对应的具体实施方式，保证说明书和权利要求书的技术特征、措辞完全对应。

若某条权利要求涉及新定义名词，动笔前先按 `具体实施方式名词先释义后展开` 把名词解释清楚，再按本 skill"复述+展开"的两段结构撰写实施例——本 skill 负责实施例本身怎么写，那个 skill 负责"名词释义排在实施例之前"这一顺序。
