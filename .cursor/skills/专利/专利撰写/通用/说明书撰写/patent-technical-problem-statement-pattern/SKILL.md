# 说明书"发明内容"部分——技术问题陈述撰写规范（所有案件类型通用）

## 固定句式

"发明内容"部分开头，紧跟在背景技术之后，用这句话陈述本专利要解决的技术问题：

**"本发明/实用新型的目的在于提供一种（专利名称），以解决现有技术中的XXXX的问题。"**

- "（专利名称）"必须与"技术领域"部分、权利要求书标题三处保持逐字一致（参见同分支下的 `patent-technical-field-sentence-pattern`）。
- "XXXX"是这句话的核心，见下面的规则。

## 核心规则：XXXX 必须对应独立权利要求解决的技术问题，不是背景技术里全部问题的堆砌

背景技术部分（参见 `patent-background-technology-drafting`）通常会归纳出2～3个技术问题，分别由独立权利要求和排在前面的关键从权解决。但**这里的"目的陈述"，只需要点出独立权利要求（权1）核心技术特征实际解决的那一个（或那几个直接相关的）技术问题**，不要把背景技术里罗列的全部问题都堆到这一句里。

判断方法：
1. 先看独立权利要求的核心技术特征是什么（通常是"其特征在于"之后最关键的那一处结构限定）。
2. 回头看背景技术第三段归纳的几个技术问题，找出哪一个问题是**专门靠独权这个核心技术特征解决的**（不是靠从权才解决的）。
3. 把这一个（或这几个直接相关的）问题浓缩成一句话，填进"XXXX"里。

其余由从权解决的技术问题，不需要在这句"目的陈述"里提及——可以在"技术方案"段落展开从权具体内容时再自然带出，或者放到"有益效果"部分分点说明。**这样"目的陈述"才能跟独权的保护范围精确对应，不会显得独权"文不对题"（背景技术提了三个问题，独权却只解决了其中一个，如果目的陈述把三个都写上，会让人怀疑独权是不是遗漏了必要技术特征）**。

## 示范（承接本项目背景技术）

假设背景技术归纳了三个问题：① 喷射距离受限（水压未配合风压）；② 喷雾角度和覆盖直径固定不可调；③ 喷孔布置未结合风压分布差异、雾化不均。

独立权利要求（权1）的核心技术特征是"喷孔开口方向与风叶所在平面形成夹角，使水压与风压共同推动，增大喷射距离"——这精准对应的是**问题①**，跟问题②③没有直接关系（问题②③分别是从权2～7、从权8～9解决的）。

因此目的陈述应该写成：

> 本实用新型的目的在于提供一种喷雾降尘装置，以解决现有技术中雾化水体仅依靠自身水压驱动、喷射距离有限，难以覆盖设备较远粉尘扩散区域的问题。

**不要**写成把三个问题都罗列进去的版本（错误示范）：

> ~~本实用新型的目的在于提供一种喷雾降尘装置，以解决现有技术中喷射距离有限、喷雾角度和覆盖直径固定不可调、喷孔布置未结合风压分布差异导致雾化不均的问题。~~

这种写法看起来"全面"，但会让读者误以为独权本身就同时解决了三个问题，而实际上独权只对应第一个，另外两个是从权才解决的，容易造成独权与背景技术的问题、以及独权与其技术特征之间对应关系不清晰。

## 检查清单

- [ ] 专利名称是否与"技术领域"部分、权利要求书标题完全一致？
- [ ] XXXX描述的问题，是否正好是独立权利要求核心技术特征解决的问题，而不是背景技术里全部问题的简单堆砌？
- [ ] 如果独权确实是通过一个综合性技术方案同时解决多个问题，XXXX里提及的每一个问题，是否都能在独权的技术特征里找到对应的解决手段（而不是只有第一个问题有对应，其余是"搭便车"式地写上去）？
- [ ] 语言是否自然通顺（参照 `patent-background-technology-drafting` 里"语言风格"章节的要求，避免堆砌生硬长句）？

## 应用范围

本规则适用于撰写或审核任意类型专利说明书"发明内容"部分的技术问题陈述。撰写顺序建议：先完成 `patent-background-technology-drafting`（背景技术，归纳2～3个问题）→ 用 `patent-claims-quality-review` 确认独权和各从权分别对应哪个问题 → 再用本skill撰写"目的陈述"，只保留独权对应的那部分。
