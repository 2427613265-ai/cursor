# 通用·说明书撰写

本分支用于存放**不区分结构类/方法类/算法类，所有案件类型通用**的说明书撰写相关 skill（比如"技术领域"部分的固定句式这类跟案件具体技术方案无关的通用规范）。

案件类型专属的说明书撰写规范，请分别放到 `专利/专利撰写/结构类/说明书撰写/`、`专利/专利撰写/方法类/说明书撰写/`、`专利/专利撰写/算法类/说明书撰写/` 对应目录下。

## 已有 skill

- `说明书技术领域部分撰写`（说明书技术领域部分撰写）：说明书"技术领域"部分的固定句式，以及专利名称与权利要求书标题一致性、用词统一性的检查清单。
- `说明书背景技术部分撰写`（说明书背景技术部分撰写）：背景技术部分的四段式结构（具体使用场景→现有技术及各自不足→归纳2～3个技术问题→固定收尾句），要求技术问题与权利要求实际解决的问题一一对应。
- `说明书技术问题陈述撰写`（说明书技术问题陈述撰写）：发明内容部分"本发明/实用新型的目的在于提供一种（专利名称），以解决现有技术中的XXXX的问题"固定句式，要求XXXX只对应独立权利要求实际解决的技术问题，不堆砌背景技术里的全部问题。
- `说明书有益效果撰写`（说明书有益效果撰写）：发明内容部分"有益效果"只写独立权利要求核心技术特征带来的效果，不混入从权效果，且要与技术问题陈述一一对应、形成镜像呼应。
- `说明书具体实施方式撰写`（说明书具体实施方式撰写）：每一条权利要求（含从属权利要求）的实施例第一段都原样复述对应权利要求/步骤的技术方案、除删除"其特征在于"这一形式用语外不得改动，再用"具体而言，"过渡，按原有顺序逐条展开每个技术特征的原理和具体实施方式，不遗漏、不夹带新特征；并规范从属权利要求实施例写法、"如图所示"及附图标记（部件名称+标号）的引用格式。
- `具体实施方式名词先释义后展开`（具体实施方式名词先释义后展开）：为某条权利要求写实施例时，若涉及新定义名词，先把这些名词逐一解释清楚，再按"复述+展开"撰写该条实施例，保证读者读实施例前已理解相关名词。
- `说明书附图说明撰写`（说明书附图说明撰写）：附图说明只能依据交底书已有附图撰写，不能凭空编造新图；核心发明点缺少对应附图时，写完后要单独提醒撰写人向发明人申请补充；方法类发明第一张图必须是方法流程图。

后续新增通用说明书撰写相关 skill 时，请在本目录下按照 `.cursor/skills/专利/专利撰写/通用/说明书撰写/<中文名称>/SKILL.md` 的结构添加。
