# 通用·说明书撰写

本分支用于存放**不区分结构类/方法类/算法类，所有案件类型通用**的说明书撰写相关 skill。

案件类型专属的说明书撰写规范，请分别放到 `专利/专利撰写/结构类/说明书撰写/`、`专利/专利撰写/方法类/说明书撰写/`、`专利/专利撰写/算法类/说明书撰写/` 对应目录下。

## 子分支

说明书撰写按说明书各主要部分分为三个子分支：

| 子分支 | 对应说明书部分 | 目录 |
|--------|---------------|------|
| **发明内容** | 技术领域、背景技术、技术问题陈述、有益效果等 | `发明内容/` |
| **附图说明** | 附图说明、附图标记说明 | `附图说明/` |
| **具体实施例** | 具体实施方式（各权利要求实施例及补充方案） | `具体实施例/` |

各子分支下的 skill 清单详见对应子目录的 `README.md`。

## 撰写顺序建议

1. `发明内容/` — 技术领域 → 背景技术 → 技术问题陈述 → 有益效果
2. `附图说明/` — 附图说明
3. `具体实施例/` — 逐条权利要求实施例 → 未入权补充方案编排

后续新增通用说明书撰写相关 skill 时，请先判断属于哪个子分支，再在该子目录下按照 `.cursor/skills/专利/专利撰写/通用/说明书撰写/<子分支>/<中文名称>/SKILL.md` 的结构添加。
