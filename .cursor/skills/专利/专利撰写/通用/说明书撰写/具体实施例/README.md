# 通用·说明书撰写·具体实施例

本分支存放说明书**"具体实施方式"**（具体实施例）部分的通用撰写规范，不区分结构类/方法类/算法类。

## 已有 skill

- `说明书具体实施方式撰写`（说明书具体实施方式撰写）：每一条权利要求（含从属权利要求）的实施例第一段都原样复述对应权利要求/步骤的技术方案、除删除"其特征在于"这一形式用语外不得改动，再用"具体而言，"过渡，按原有顺序逐条展开每个技术特征的原理和具体实施方式，不遗漏、不夹带新特征；并规范从属权利要求实施例写法、"如图所示"及附图标记（部件名称+标号）的引用格式。
- `具体实施方式名词先释义后展开`（具体实施方式名词先释义后展开）：为某条权利要求写实施例时，若涉及新定义名词，先把这些名词逐一解释清楚，再按"复述+展开"撰写该条实施例，保证读者读实施例前已理解相关名词。
- `未写入权利要求的技术方案在说明书中的编排`（未写入权利要求的技术方案在说明书中的编排）：写完各权利要求实施例后，交底书/AI/人为补充但未写入权利要求的技术方案也要写进说明书充分公开；与某条权利要求强相关的紧跟其实施例之后，独立、关联不大的集中放到整个具体实施方式最后，并用"在一个可选的实施方式中/进一步地"等标注为可选内容，不得混入权利要求实施例主线。
- `具体实施方式补充整体装配方式`（具体实施方式补充整体装配方式）：权利要求只保护部件/组件本身、未限定其与外部部件装配连接关系时，具体实施方式要补充整体装配方式，采用"权1实施例展开段一句总述+文末可选实施例详述"的分层写法，使附图中的安装类标记不成孤岛，且不把装配写进权1复述段、不在各从权实施例重复。
