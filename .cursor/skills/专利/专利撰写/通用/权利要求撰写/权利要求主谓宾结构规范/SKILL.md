# 权利要求主谓宾结构规范

## 核心规则

**权利要求中的每个动作、状态、关系描述都必须有明确的主语，避免因省略主语导致指代不明、执行者不清楚，从而造成权利要求不清楚。**

### 基本原则

1. **动作必有执行者** - 每个动词前应有明确的主语（部件名称、模块名称、系统名称等）
2. **状态必有归属** - 每个状态描述应明确是哪个对象的状态
3. **关系必有双方** - 描述连接、配合等关系时，应明确关系的主体和客体
4. **允许省略的情况** - 仅当上下文中主语明确且唯一时，才可省略

---

## 常见问题类型

### 问题1：动作的执行者不明确

#### 错误示范

❌ **缺少主语导致执行者不明：**
```
S3：接收用户输入的数据，进行加密处理，并存储至数据库；
```

**问题分析：**
- "接收"的主语是谁？控制器？输入模块？
- "进行加密处理"的主语是谁？加密模块？处理器？
- "存储至数据库"的主语是谁？存储模块？控制器？
- 三个动作的执行者都不明确

#### 正确示范

✅ **明确各动作的主语：**
```
S3：所述输入模块接收用户输入的数据，所述加密模块对所述数据进行加密处理，所述存储模块将加密后的数据存储至数据库；
```

**改进说明：**
- 明确了"接收"的主语是"输入模块"
- 明确了"加密处理"的主语是"加密模块"
- 明确了"存储"的主语是"存储模块"

### 问题2：状态的归属对象不明确

#### 错误示范

❌ **缺少主语导致状态归属不明：**
```
其特征在于，处于工作状态时，温度控制在30-50℃范围内
```

**问题分析：**
- "处于工作状态"的主语是谁？整个装置？某个模块？
- "温度控制在..."的主语是谁？哪个部件的温度？

#### 正确示范

✅ **明确状态的归属对象：**
```
其特征在于，所述加热模块处于工作状态时，所述加热模块的温度控制在30-50℃范围内
```

或者：
```
其特征在于，所述反应腔处于工作状态时，所述反应腔内的温度控制在30-50℃范围内
```

### 问题3：关系描述中的主体或客体不明确

#### 错误示范

❌ **连接关系主客体不明：**
```
通过弹性连接件连接
```

**问题分析：**
- 谁和谁通过弹性连接件连接？缺少连接关系的双方

#### 正确示范

✅ **明确连接关系的双方：**
```
所述固定座与所述活动臂通过弹性连接件连接
```

### 问题4：并列动作主语混淆

#### 错误示范

❌ **并列动作主语不一致但省略导致混淆：**
```
S3：所述控制器接收传感器信号，分析信号特征，发出控制指令，执行相应动作；
```

**问题分析：**
- "接收"、"分析"、"发出"的主语是控制器 ✓
- "执行相应动作"的主语是谁？还是控制器吗？还是执行机构？
- 如果主语变了但没有明确指出，会造成理解混乱

#### 正确示范

✅ **方案A - 主语相同，明确承前省略：**
```
S3：所述控制器接收传感器信号，分析信号特征，并发出控制指令；所述执行机构根据所述控制指令执行相应动作；
```

✅ **方案B - 拆分为多个步骤：**
```
S3：所述控制器接收传感器信号；
S4：所述控制器分析所述信号的特征，并发出控制指令；
S5：所述执行机构根据所述控制指令执行相应动作；
```

## 检查方法

### 第一步：标记所有动词和状态词

阅读权利要求，用下划线标记所有动词（接收、发送、处理、存储、控制、判断等）和状态词（处于、位于、保持等）。

### 第二步：为每个动词/状态词寻找主语

逐个检查每个标记的词，问自己：
1. 这个动作/状态的执行者/归属者是谁？
2. 在当前句子中能否找到明确的主语？
3. 如果主语被省略，是否能从前文唯一确定？

### 第三步：识别问题句式

特别关注以下容易省略主语的句式：

#### 句式1：直接以动词开头
```
❌ 接收数据，进行处理...
✅ 所述处理器接收数据，进行处理...
```

#### 句式2："通过...方式"开头
```
❌ 通过加密算法对数据进行加密
✅ 所述加密模块通过加密算法对数据进行加密
```

#### 句式3：被动语态缺少施动者
```
❌ 数据被加密后存储
✅ 数据被所述加密模块加密后，由所述存储模块存储
```

#### 句式4：并列动作主语变化
```
❌ 所述控制器接收信号、处理信号，触发执行（执行者是谁？）
✅ 所述控制器接收信号、处理信号，并触发所述执行机构执行相应动作
```

### 第四步：修正问题句式

对于识别出的问题，按以下原则修正：

1. **补充明确的主语** - 优先方案
2. **调整句子结构** - 使主语自然出现
3. **拆分复杂句子** - 避免一个句子中主语频繁变化

---

## 检查清单

在撰写或审核权利要求时，逐项检查：

### 结构类权利要求

- [ ] 部件的连接关系是否明确了连接双方（A与B连接）
- [ ] 部件的位置关系是否明确了位置主体（A位于B的上方）
- [ ] 部件的功能描述是否明确了功能执行者（所述A用于...）
- [ ] "其中"、"且"、"并"之后的动作是否主语明确
- [ ] 被动语态是否必要时补充了施动者

### 方法类权利要求

- [ ] 每个步骤的动作执行者是否明确（由谁执行）
- [ ] 判断、比较等操作是否明确了操作主体
- [ ] 获取、接收等动作是否明确了执行者
- [ ] 并列动作之间主语变化是否明确标出
- [ ] 条件判断的判断主体是否明确

### 算法类权利要求

- [ ] 计算、处理等操作是否明确了执行主体（处理器、模块、系统）
- [ ] 数据流转过程中各环节的执行者是否明确
- [ ] 判断、比较、提取等操作的主体是否明确
- [ ] 输出、返回等动作的执行者是否明确

## 典型案例

### 案例1：结构类权利要求中的主语省略

**场景**：描述一个数据处理装置的结构

❌ **错误版本：**
```
权利要求1：一种数据处理装置，包括：
处理器、存储器和通信模块；
其特征在于，还包括加密模块，
接收所述处理器发送的数据，
进行加密处理，
并将加密后的数据发送至所述通信模块。
```

**问题分析：**
- "接收"的主语是谁？加密模块吗？
- "进行加密处理"的主语是谁？加密模块吗？
- "发送至通信模块"的主语是谁？加密模块吗？
- 虽然从上下文能推断出是加密模块，但主语缺失导致表达不够清楚

✅ **正确版本：**
```
权利要求1：一种数据处理装置，包括：
处理器、存储器和通信模块；
其特征在于，还包括加密模块，
所述加密模块接收所述处理器发送的数据，
所述加密模块对所述数据进行加密处理，
所述加密模块将加密后的数据发送至所述通信模块。
```

或者更简洁（允许承前省略）：
```
权利要求1：一种数据处理装置，包括：
处理器、存储器和通信模块；
其特征在于，还包括加密模块，
所述加密模块用于接收所述处理器发送的数据，对所述数据进行加密处理，并将加密后的数据发送至所述通信模块。
```

### 案例2：方法类权利要求中的主语省略

**场景**：描述一个数据处理方法

❌ **错误版本：**
```
S3：获取用户输入，验证输入格式，判断是否有效，若是，则存储至数据库；
```

**问题分析：**
- "获取"的主语是谁？服务器？输入模块？
- "验证"的主语是谁？验证模块？服务器？
- "判断"的主语是谁？
- "存储"的主语是谁？
- 四个动作的执行者都不明确

✅ **正确版本：**
```
S3：所述服务器获取用户输入，所述服务器验证所述输入的格式，判断所述输入是否有效，若是，则所述服务器将所述输入存储至数据库；
```

或者拆分为多个步骤：
```
S3：所述服务器获取用户输入；
S4：所述服务器验证所述输入的格式，判断所述输入是否有效；
S5：若判断结果为有效，则所述服务器将所述输入存储至数据库；
```

### 案例3：多主体交互场景

**场景**：客户端和服务器之间的交互

❌ **错误版本：**
```
S3：发送请求至服务器，接收服务器返回的响应，解析响应数据，并显示在界面上；
```

**问题分析：**
- "发送请求"的主语是客户端
- "接收响应"的主语是客户端
- "解析数据"的主语是客户端
- "显示在界面"的主语是客户端
- 虽然主语相同，但没有明确，容易让读者困惑

✅ **正确版本：**
```
S3：所述客户端发送请求至服务器，接收服务器返回的响应，解析所述响应中的数据，并将解析结果显示在界面上；
```

### 案例4：状态描述中的主语省略

**场景**：描述设备的工作状态

❌ **错误版本：**
```
其特征在于，处于待机状态时，降低功耗；处于工作状态时，提高处理速度。
```

**问题分析：**
- "处于待机状态"的主语是谁？整个装置？某个模块？
- "降低功耗"的主语是谁？
- "处于工作状态"的主语是谁？
- "提高处理速度"的主语是谁？

✅ **正确版本：**
```
其特征在于，所述处理器处于待机状态时，所述处理器降低功耗；所述处理器处于工作状态时，所述处理器提高处理速度。
```

或者：
```
其特征在于，所述处理器根据工作状态调节性能：处于待机状态时降低功耗，处于工作状态时提高处理速度。
```
（这种情况下，"所述处理器"作为总领主语，后续可以承前省略）

## 允许省略主语的情况

虽然原则上要求主语明确，但以下情况可以承前省略：

### 情况1：同一句子内的并列动作（主语相同）

✅ **允许省略：**
```
所述控制器接收信号、分析信号、并发出指令
```
（"接收"、"分析"、"发出"的主语都是"控制器"，承前省略）

### 情况2："用于"结构引导的功能描述

✅ **允许省略：**
```
所述处理模块用于接收数据、处理数据并输出结果
```
（"接收"、"处理"、"输出"的主语都是"处理模块"，通过"用于"引导）

### 情况3：明确的定语从句结构

✅ **允许省略：**
```
所述加密模块包括密钥生成单元，所述密钥生成单元用于生成密钥
```
（"生成"的主语通过"用于"结构明确是"密钥生成单元"）

### 判断标准

允许省略的关键判断标准：
1. **唯一性** - 上下文中只有一个可能的主语
2. **邻近性** - 主语在前文紧邻位置出现
3. **结构性** - 通过"用于"等结构明确了主语
4. **不引起歧义** - 读者无需推理即可明确主语

## 修正建议格式

当发现主语缺失问题时，按以下格式提出修正建议：

```
【问题句子】：接收数据，进行处理，并存储
【问题类型】：动作执行者不明确
【歧义分析】："接收"、"处理"、"存储"的执行者不清楚，可能是处理器、控制器或其他模块
【修正建议】：所述处理器接收数据，对所述数据进行处理，并将处理结果存储至所述存储器
【修正理由】：明确各动作的执行主体，避免指代不明导致权利要求不清楚
```

## 与其他技能的配合

本技能应与以下技能配合使用：

1. **`权利要求清楚性与逻辑一致性规范`** - 本技能是清楚性检查的一个重要维度
2. **`结构类权利要求撰写`** - 结构类权利要求撰写时同步检查主语
3. **`条件判断语句规范`** - 判断语句中也要注意判断主体的明确性

推荐使用顺序：
- **撰写阶段**：边写边注意主谓宾结构（本技能）
- **自查阶段**：应用本技能逐句检查主语
- **质检阶段**：结合`权利要求清楚性与逻辑一致性规范`全面检查

## 注意事项

1. **不要过度重复主语** - 在同一句子内，主语相同的并列动作可以承前省略，过度重复反而影响可读性

2. **平衡清晰性和简洁性** - 目标是清楚，不是冗长。在确保清晰的前提下尽量简洁

3. **考虑专利审查员的阅读体验** - 审查员需要快速理解技术方案，主语不明会增加理解成本

4. **说明书要与权利要求对应** - 说明书中对权利要求的解释应该明确各动作的执行主体，与权利要求中的主语保持一致

5. **不同技术领域的习惯** - 某些技术领域可能有特定的表达习惯，但清楚性原则不变
