# 通用·权利要求撰写

本分支用于存放**不区分结构类/方法类/算法类，所有案件类型通用**的权利要求撰写相关 skill（比如通用的用词规范、格式规范等，与具体案件类型无关）。

案件类型专属的权利要求撰写规范，请分别放到 `专利/专利撰写/结构类/权利要求撰写/`、`专利/专利撰写/方法类/权利要求撰写/`、`专利/专利撰写/算法类/权利要求撰写/` 对应目录下。

## 已有 skill

- `patent-claims-formal-compliance`（权利要求形式合规性规范）：依据《专利法实施细则》检查从属权利要求引用方向（只能引用在前权利要求）、多项从属权利要求的引用方式（择一引用、不得作为另一多项从权的基础）、引用部分编号与主题名称、编号连续性、固定句式格式等硬性格式规则。
- `patent-claims-clarity-and-logic`（权利要求清楚性与逻辑一致性规范）：检查孤岛特征、突然出现且缺乏解释的人为定义术语、模糊用语、歧义与反例、清楚简洁性、物理/数学常识与前后逻辑矛盾，以及术语一致性、引用编号来源对应。
- `patent-independent-claim-necessary-features-check`（独立权利要求必要技术特征检查）：逐一检查独权里的每个技术特征是否是解决声称技术问题所必需的，重点排查数量词、排列方式、具体机构这三类常见的可精简项，避免独权保护范围被不必要地缩小。

后续新增相关 skill 时，请在本目录下按照 `.cursor/skills/专利/专利撰写/通用/权利要求撰写/<skill-name>/SKILL.md` 的结构添加。
