# 通用·权利要求撰写

本分支用于存放**不区分结构类/方法类/算法类，所有案件类型通用**的权利要求撰写相关 skill（比如通用的用词规范、格式规范等，与具体案件类型无关）。

案件类型专属的权利要求撰写规范，请分别放到 `专利/专利撰写/结构类/权利要求撰写/`、`专利/专利撰写/方法类/权利要求撰写/`、`专利/专利撰写/算法类/权利要求撰写/` 对应目录下。

## 已有 skill

| skill | 简要说明 |
|-------|----------|
| `权利要求形式合规性规范` | 从权引用方向、多项从权择一引用、编号连续性等《专利法实施细则》硬性格式规则 |
| `权利要求清楚性与逻辑一致性规范` | 孤岛特征、人为定义术语、模糊用语、歧义与反例、物理/数学常识、术语一致性 |
| `独立权利要求必要技术特征检查` | 逐一检查独权各特征是否必要，重点排查数量词、排列方式、具体机构 |
| `权利要求主谓宾结构规范` | 每个动作/状态/关系是否有明确主语，避免省略主语导致指代不明 |
| `权利要求指代与参照清晰性规范` | 指代词（"该""其"）和方位词（周向、轴向等）的清晰性与参照物 |
| `条件判断语句规范` | 方法类/算法类权利要求中条件判断的固定句式（"判断是否…，若是，则…；若否，则…"） |
| `预设值与参数来源检查` | 机器人视角自查预设值/阈值/参数在使用前是否有明确来源，是否满足"先获取、后使用" |
| `自定义名词就地定义` | 非公知的自定义名词必须在权利要求内部用"其中"句就地定义，而不是只写在说明书里 |

## 与质检分支的关系

- 撰写阶段：边写边对照 `权利要求清楚性与逻辑一致性规范` 等自查
- 定稿质检：由 `专利/权利要求质检/权利要求书质检` 主控，按"必读/按需"分级加载本目录下的子 skill

后续新增相关 skill 时，请在本目录下按照 `.cursor/skills/专利/专利撰写/通用/权利要求撰写/<中文名称>/SKILL.md` 的结构添加，并更新本 README。
