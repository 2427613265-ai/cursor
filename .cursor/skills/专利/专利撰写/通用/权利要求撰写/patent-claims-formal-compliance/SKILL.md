# 权利要求形式合规性规范（所有案件类型通用）

## 适用场景

- 撰写从属权利要求时，或对已撰写完成的权利要求书做形式核对时使用。
- 本 skill 检查的是《专利法实施细则》规定的**硬性格式规则**，属于"非黑即白"的合规问题，没有专业判断的商榷空间——一旦违反，审查员会直接在审查意见中指出形式缺陷。
- 与 `专利质检/权利要求质检/patent-claims-quality-review` 配合使用：质检时应把本 skill 的检查项作为**最优先**核对的"硬伤"类问题，确认无形式缺陷后，再讨论保护范围、创造性这类需要专业判断的实质问题。

## 规则一：从属权利要求只能引用"在前"的权利要求

依据《专利法实施细则》（2023年修订）第二十五条第二款："从属权利要求只能引用在前的权利要求。"

- 权利要求 N 的依附对象，编号必须**严格小于 N**——只能引用排在它前面的权利要求，不能引用编号等于（引用自己）或大于自己（引用排在它后面）的权利要求。
- 这是最基础、最容易机械核对的一条规则：扫描每一条从权的引用部分，取出其中引用的权利要求编号，只要发现编号 ≥ 自身编号，就是格式错误，必须改。

正确示范：
```
权利要求5：根据权利要求1至4中任一项所述的装置，其特征在于，……
```
（引用的1～4均小于5，合规）

错误示范：
```
权利要求3：根据权利要求5所述的装置，其特征在于，……
```
（权利要求3引用了排在它后面的权利要求5，违反规则，必须改成引用编号更小的权利要求，或调整整体编号顺序）

## 规则二：多项从属权利要求只能"择一方式"引用在前权利要求

依据同一条款："引用两项以上权利要求的多项从属权利要求，只能以择一方式引用在前的权利要求。"

- 如果一条从权同时引用了两项以上的在前权利要求，引用部分必须使用"或""任一项"这种**择一（选择性）**表述，不能使用"和""及""与"这种要求同时满足多个引用对象的表述。
- 判断方法：读引用部分的连接词——"根据权利要求1或2所述"合规（择一）；"根据权利要求1和2所述"不合规（要求同时满足两者的限定，不是择一）。

正确示范：
```
根据权利要求1或权利要求2所述的装置，……
根据权利要求1至3中任一项所述的装置，……
```

错误示范：
```
根据权利要求1和权利要求2所述的装置，……
```
（这种写法在逻辑上要求同时依附两项权利要求的全部限定，不符合"择一"要求，必须改成"或"/"任一项"）

## 规则三：多项从属权利要求不得作为另一项多项从属权利要求的基础

依据同一条款："并不得作为另一项多项从属权利要求的基础。"

- 先判断一条从权是不是"多项从属权利要求"——只要它的引用部分包含两项以上权利要求（用了"或""任一项"这种择一表述），它就是多项从属权利要求。
- 判断规则：**任何一条多项从属权利要求，都不能被另一条多项从属权利要求引用**。也就是说，如果权利要求5是"根据权利要求1或2所述……"（多项从权），那么后面任何一条权利要求都不能写成"根据权利要求5或6所述……"这种同时以择一方式引用了权利要求5的多项从权。但是**单独、非择一地引用**权利要求5本身（即"根据权利要求5所述……"，只单独引用这一项）是允许的。

正确示范：
```
权利要求3：根据权利要求1或2所述的装置，其特征在于……（多项从权，合规，因为它引用的权利要求1、2都不是多项从权）
权利要求4：根据权利要求3所述的装置，其特征在于……（单独引用权利要求3这一项，不是"多项择一"引用，即使权利要求3本身是多项从权，这里也合规）
```

错误示范：
```
权利要求3：根据权利要求1或2所述的装置，其特征在于……（多项从权）
权利要求4：根据权利要求3或5所述的装置，其特征在于……
```
（权利要求4是多项从权，同时以择一方式引用了权利要求3（本身是多项从权）和权利要求5，属于"多项从权作为另一多项从权的基础"，违反规则。修改方式：可以把权利要求4拆成两条分别单独引用权利要求3、权利要求5的从权，或者把权利要求3改写成非多项从权）

## 规则四：从权引用部分应写明编号及主题名称

依据《专利法实施细则》第二十五条第一款：从属权利要求的引用部分应"写明引用的权利要求的编号**及其主题名称**"。

- 从权的引用部分不能只写编号，还应带上被引用权利要求的主题名称（即独权/上级从权保护的对象名称，比如"装置""方法""系统"）。
- 主题名称应当和独权的名称保持一致，不能中途变换措辞（比如独权是"一种XX装置"，从权引用时也应是"根据权利要求1所述的装置"，不能写成"根据权利要求1所述的设备"）。

正确示范：
```
权利要求1：一种XX装置，其特征在于，……
权利要求2：根据权利要求1所述的装置，其特征在于，……
```

错误示范：
```
权利要求2：根据权利要求1所述，其特征在于，……   （缺少主题名称"装置"）
权利要求2：根据权利要求1所述的设备，其特征在于，……   （主题名称和独权"装置"不一致，前后用词不统一）
```

## 规则五：编号连续性与固定句式格式

- **编号连续、无重复、无跳号**：权利要求书从1开始连续编号，不能跳号（比如从3直接跳到5）、不能有重复编号；调整、增删权利要求后必须重新核对全篇编号是否连续。
- **每项权利要求独立成段，以句号结尾**：每一项权利要求单独一段，最后以句号收尾，不能跟下一项权利要求粘连或漏掉句号。
- **固定引导句式**：
  - 独立权利要求：`一种/一个 XXX，其特征在于，……`（结构类/产品类）或按方法类固定句式（步骤类，以"包括以下步骤："引导）。
  - 从属权利要求：`根据权利要求X所述的XXX，其特征在于，……`（单项引用）或 `根据权利要求X至Y中任一项所述的XXX，其特征在于，……`（多项择一引用）。
  - 全篇权利要求书的引导句式应当统一，不要在"根据""按照""依据"这类近义表述之间来回切换。

## 检查方法（逐条机械核对）

对权利要求书从第一条独权开始，逐条向后核对每一条从权：
1. 提取该从权引用部分里的所有编号，检查是否都严格小于自身编号（规则一）。
2. 如果引用了两项以上编号，检查连接词是否为择一表述（规则二）。
3. 如果引用了两项以上编号，检查被引用的这些编号中，有没有任何一个本身也是"多项从属权利要求"（规则三）。
4. 检查引用部分是否同时写明了编号和主题名称，且主题名称与独权保持一致（规则四）。
5. 检查全篇编号是否连续、无跳号无重复；逐项检查句末是否有句号；引导句式是否统一（规则五）。

## 检查清单

- [ ] 是否存在从权引用了编号等于或大于自身的权利要求？
- [ ] 多项从权的引用部分是否都使用了"或"/"任一项"这种择一表述，而非"和"/"及"？
- [ ] 是否存在多项从权同时引用了另一条本身也是多项从权的权利要求？
- [ ] 每一条从权的引用部分是否都同时写明了编号和主题名称？主题名称是否与独权保持一致？
- [ ] 权利要求编号是否连续、无跳号、无重复？每项是否以句号结尾？引导句式是否全篇统一？

## 应用范围

本规则适用于任意技术方案类型（结构类/方法类/算法类）的权利要求书撰写与质检，是所有权利要求都必须遵守的形式底线，撰写从权时就应当直接遵守，而不是等质检阶段才发现问题。与 `专利质检/权利要求质检/patent-claims-quality-review` 配合使用：质检时应优先核对本 skill 的检查项。
