# 独立权利要求必要技术特征检查规范（所有案件类型通用）

## 法律依据与核心原则

依据《专利法实施细则》，独立权利要求应当从整体上反映发明或者实用新型的技术方案，记载解决其技术问题**所必需的技术特征**。

核心原则：独权里的每一个技术特征，都应该是"解决声称的技术问题所必需的"，既不能少（少了会导致技术方案不完整、无法解决问题），也不能多（多了会不必要地缩小保护范围，让本该落入保护范围的等同/替代实施方式被排除在外）。**本skill专门用于排查"多"的问题**——即独权里有没有夹带非必要的技术特征。

## 判断方法：逐一特征反问"去掉会怎样"

对独权里的每一个技术特征（部件、连接关系、数量词、排列方式、具体机构、参数范围等），逐一提问：

1. **明确这条独权对应解决的技术问题是什么**（通常是 `patent-technical-problem-statement-pattern` 里已经确定的内容——独权核心技术特征直接对应解决的那个问题）。
2. **对每个技术特征反问**："如果去掉这个特征，或者把它替换成一个更宽泛/更简单的说法，声称的技术问题还能解决吗？核心技术效果还成立吗？"
   - 答案是"能，问题依然解决"——这个特征**不是必要技术特征**，应当考虑删除、概括处理，或降级到从权。
   - 答案是"不能，去掉后问题就解决不了了"——这个特征**是必要技术特征**，必须保留在独权里。
3. **反向验证方案完整性**：特征拿掉之后，独权描述的技术方案是否仍然完整、能够独立实施？如果拿掉后方案出现结构悬空、逻辑说不通、或者变得含糊到无法实施，说明这个特征其实是必要的，不能拿掉——必要技术特征的判断标准始终是"能否解决声称的技术问题"，不是"越少越好"，删减不能矫枉过正。

## 三类最常见的"可精简项"

撰写独权时要特别警惕以下三类容易被不必要地写死、但实际上大多可以概括或删除的内容：

### 1. 数量词（"多个""至少两个"）

除非问题的解决确实依赖于"多个"这个数量本身（比如"多个"才能形成阵列效果、或者"多个"才能实现某种协同作用），否则大多数情况下单个也能实现同样的核心效果，应当去掉数量词（用不区分单复数的裸名词）或改成"一个或多个"这种不限定数量的说法。

**示例**：一种喷雾装置的独权，如果核心创造点是"喷孔开口方向与某个基准方向形成夹角，使雾化水体获得风压助推"，这个效果单个喷孔就能实现，不需要"多个喷孔"才成立——"多个"应当从独权里去掉，留给需要"多孔联动/多孔阵列"效果的从权去限定。

### 2. 排列方式（"呈放射状""均匀分布""对称设置"）

这类描述往往只是众多可行实施方式里的**一种优选方案**，除非排列方式本身就是解决技术问题的手段（比如"均匀分布"是为了让某种力学效应均衡分布，这种情况下排列方式本身就是必要技术特征），否则应当从独权里去掉或降级到从权。

**示例**：多个部件"固定连通"于某个主体结构上，如果核心效果跟这些部件具体是"呈放射状"排列、还是随意分布在主体结构上没有关系，那么"呈放射状"就是可以从独权里去掉的非必要限定。

### 3. 具体机构/具体实现方式（"通过螺栓固定""采用注塑一体成型""通过电机驱动"）

具体的连接方式、成型工艺、驱动方式通常是众多可选实现细节中的一种，除非这种具体机构本身就是发明点所在，否则应当用更概括的说法（"固定连接"而不是"通过螺栓固定"），把具体机构降级到从权。

## 检查清单

- [ ] 是否已经明确独权对应解决的技术问题是什么？
- [ ] 独权里每一个技术特征，去掉之后核心技术效果是否依然成立？如果成立，是否已经降级处理？
- [ ] 是否存在可以去掉、但目前仍然写在独权里的数量词限定？
- [ ] 是否存在可以去掉、但目前仍然写在独权里的排列方式限定？
- [ ] 是否存在可以概括、但目前仍然写成具体机构/具体实现方式的限定？
- [ ] 精简之后的独权，技术方案是否仍然完整、能够独立实施，没有出现结构悬空或逻辑不完整的问题？

## 与已有skill的关系

- 与同分支的 `patent-claims-clarity-and-logic` 配合：那个skill检查的是"表达是否清楚、有没有孤岛特征"，本skill检查的是"这个特征该不该出现在独权里"，两者角度不同，都要过一遍。
- 与 `专利/权利要求质检/patent-claims-quality-review` 配合：该skill"维度一"第2点已经点出"有没有可以删掉但不影响创造性的多余限定"这个大方向，本skill是针对这一点的详细展开和具体操作方法。
- 精简之后如果需要把被删除的具体实现方式、数量限定、排列方式等内容保留下来（作为优选实施方式），应当把它们改写成从权，而不是彻底丢弃，参见 `patent-structural-claim-drafting`（结构类从权撰写）等对应分支的撰写规范。

## 应用范围

本规则适用于任意技术方案类型（结构类/方法类/算法类）独立权利要求的撰写与审核，从最开始撰写独权时就应当对照检查，而不是等到质检阶段才发现独权范围被不必要地缩小了。
