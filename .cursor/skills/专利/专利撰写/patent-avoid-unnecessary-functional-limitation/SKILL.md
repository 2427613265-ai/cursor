# 权利要求撰写规范：功能性限定 / 技术效果表述的使用边界

## 核心规则

撰写权利要求时，**默认优先使用结构特征、位置关系、连接关系来限定技术方案**。

只有满足以下**两种情形之一**，才可以使用功能性限定或补充技术效果说明：

1. **结构描述本身无法说清楚**：仅凭结构、位置、连接关系的描述，读者/审查员无法清楚判断该技术特征的含义、边界，或者看不出它与前面已限定特征之间的关联关系。
2. **功能性表述能表达得更清楚、更简洁**：该技术特征即使勉强用纯结构语言也能描述出来，但那样写会更冗长、更绕、可读性更差；而用功能性表述（如"可转动地""定位锁紧""选择性地插设"）本领域技术人员一看就懂，且不会因此不当扩大或模糊保护范围——这种情况下应当优先选用更清楚的功能性表述，不必强行改写成纯结构语言。

**不允许**的是另一种情况——在结构关系已经用最清楚的方式（无论是结构语言还是功能语言）描述完毕之后，再画蛇添足地加一句"从而实现XX有益效果""以达到XX目的""进而调节XX"这类总结性、宣传性的效果陈述——这类内容属于说明书"发明内容""有益效果"部分应当承担的工作，写进权利要求里不会增加或明确保护范围，只会让权利要求变得冗长。

## 判断标准（自检清单）

给一句权利要求收尾加"以使……""从而……""用于……""进而……"这类效果/目的状语之前，先问自己：

1. **去掉这句话，前面的结构描述是否已经完整、无歧义？**
   - 是——这句效果陈述大概率多余，删除。
   - 否（比如前面只说"能够转动"，但读者根本不知道转动之后改变的是哪个已定义的特征、或转动本身是否有方向/轴线上的特殊限定）——保留，用来把"新增的动作能力"和"需要明确的技术特征"关联起来。

2. **这句话是否在定义一个后面权利要求还要引用的新名词？**
   - 是（比如新引入"张开夹角"这个词，后面从权还要用"所述张开夹角"限定具体数值）——保留，这里的功能性表述承担的是"定义新特征"的角色，不是单纯讲效果。
   - 这个名词/效果后面不会再被引用，只是本条内部的收尾总结——大概率可以删除。

3. **这句话讲的是"结构本身导致的显而易见的直接结果"，还是"需要专门说明才能理解的机理/关联"？**
   - 显而易见的直接结果（比如"多个限位槽+限位销、选择性插设"结构，自然就是"多档位锁定"，不需要再补一句"以实现多档位锁定"；"整体转动"这种刚体转动，自然不会改变刚体内部各点的相对角度关系，不需要再补一句"且不改变……的相对关系"）——删除。
   - 需要专门解释、不是一望而知的关联（比如"绕自身轴线转动"并不能让人一眼看出转动改变的是哪个已定义的夹角）——保留。

4. **这个技术特征改用纯结构语言描述，是否会比现在的功能性表述更啰嗦、更难懂，且不会带来保护范围上的实质差别？**
   - 是（比如与其把"可转动地插接"拆成"两者的配合面均为圆柱面、过渡配合"这种绕弯子的结构描述，不如直接写"可转动地插接"，本领域技术人员一看就懂，保护范围也没有变宽或变窄）——保留功能性表述，不必强行改写成纯结构语言。
   - 否（纯结构语言完全可以同样简洁地表达，只是撰写者图省事才用了功能性表述）——改写为结构语言。

## 正确示范

结构已经讲清楚，不需要再补效果：

```
所述内雾化孔的开口方向与所述风叶所在平面之间形成有夹角。
```

不需要再加"以使雾化水体的水压与轴向气流的风压叠加、增大喷射距离"——夹角本身已经是完整、清楚的结构限定，效果放到说明书里说明即可。

结构本身无法说清楚"转动改变的是什么"，需要补一句功能性表述把新结构和已定义特征关联起来：

```
所述内喷雾管通过所述第一旋转组件能够绕自身轴线转动并定位锁紧，以调节所述夹角的大小。
```

"以调节所述夹角的大小"不能删——因为"绕自身轴线转动"本身不能告诉读者这个转动改变的是哪个已经定义的技术特征。

功能性表述比结构性表述更清楚、应当保留的情形：

```
所述旋转接头与所述内喷雾管的端部固定连接，且同轴可转动地插接于所述固定套筒内。
```

"可转动地插接"是功能/状态性表述，但比强行改写成"所述旋转接头的外壁与所述固定套筒的内壁均为圆柱面且过渡配合"这类纯结构描述更清楚、更符合本领域技术人员的阅读习惯，保护范围也没有实质差别，因此应当保留功能性表述，不必刻意改写成结构语言。

## 错误示范（应避免）

```
……所述限位销选择性地插设于其中一个所述限位槽内，以实现所述夹角的多档位定位锁紧。
```

"以实现所述夹角的多档位定位锁紧"多余：前面"多个限位槽"+"选择性地插设于其中一个"已经讲清楚了"多档位"这层意思，且"锁紧的对象是夹角"这一点在前一条从权里已经通过"定位锁紧，以调节所述夹角的大小"交代过了，这里不需要重复，应直接删除、句号收尾。

```
……以调节所述张开夹角的大小，进而调节由多个所述内喷雾管共同形成的喷雾覆盖直径。
```

前半句"调节张开夹角的大小"要保留（定义新名词"张开夹角"）；后半句"进而调节喷雾覆盖直径"是显而易见的几何结果（张开角变了，喷雾覆盖范围自然跟着变），不需要在权利要求里专门说明，应删除，放到说明书里解释。

## 应用范围

本规则适用于独立权利要求和从属权利要求的撰写与审校，尤其要重点检查每条权利要求"其特征在于"之后的最后一句，是否存在可以删除的效果性收尾。与 `patent-structural-claim-drafting` skill配合使用：前者管"部件怎么排列展开"，本skill管"哪些收尾的效果/功能表述该留、该删"。
