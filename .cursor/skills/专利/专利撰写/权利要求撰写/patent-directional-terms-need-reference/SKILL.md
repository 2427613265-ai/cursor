# 权利要求撰写规范：方向、方位词必须有明确参照物

## 核心规则

权利要求中凡是出现**依托某个参照系才有意义**的方向、方位类词汇——常见的有"周向""轴向""径向""转动""移动""平行""垂直""倾斜""偏向"等——都必须在文字上明确点出这个参照系是什么（哪根轴线、哪个平面、哪个部件），不能让读者/审查员自己去猜或者靠上下文隐含推断。

**判断方法**：读到一处方向/方位词，追问一句"相对于什么？"——如果权利要求文字本身能直接回答（前面已经点名了轴线/平面/部件，这里明确引用），就是清楚的；如果需要读者自己联系上下文才能猜出参照物，或者干脆没有任何参照物可循，就属于不清楚，必须修改。

## 常见问题模式与修正方法

1. **"沿周向""绕周向"没有点名绕哪根轴**：
   - 不清楚："设有沿周向间隔分布的多个限位槽"（周向是绕谁的圆周？）
   - 清楚："设有沿**所述枢转轴**的周向间隔分布的多个限位槽"

2. **"转动""移动"没有点名沿/绕什么轴或方向**：
   - 不清楚："能够相对于所述环管整体转动"（绕哪根轴转？）
   - 清楚："能够绕**自身轴线**整体转动"

3. **形状类方位词（"螺旋状""弯曲""弧形"等）没有点名绕/沿什么参照物**：
   - 不清楚："呈螺旋状缠绕延伸"（螺旋缠绕的是哪根轴？）
   - 清楚："绕**自身轴线**呈螺旋状分布"

4. **"平行""垂直""倾斜"类角度关系词，参照物本身要先定义清楚**：
   - 这类词通常没有"绕哪根轴"的问题，但要检查作参照的平面/轴线/部件本身是否已经在前文给出清楚定义（比如"风叶所在平面"要在首次出现处说明是"风叶转动时叶片扫掠形成的平面，垂直于风叶转动轴线"），否则同样属于参照物不清楚。

5. **"一者……另一者……"句式中，方向词要挂在能唯一确定参照系的对象上**：
   - 两个零件即使"同轴"配合，也优先选择其中**天然具有唯一确定轴线的那个零件**（比如轴类零件本身、或者前文已经用专门名词定义过的"自身轴线"）作为方向词的参照对象，而不是笼统地说"周向"、寄望读者自己联系上下文中"同轴"这个词去推断。

## 检查清单

审校每一条权利要求时，逐句扫描是否出现以下词汇，并逐一确认参照物是否已经写明：

- [ ] 周向 / 径向 / 轴向
- [ ] 转动 / 转 / 枢转 / 摆动
- [ ] 移动 / 滑动 / 伸缩
- [ ] 平行 / 垂直 / 倾斜 / 呈……角度
- [ ] 螺旋状 / 弧形 / 弯曲等形状类方位描述
- [ ] "该方向""该轴线"这类代词式引用——确认前面确实明确定义过，不是凭空出现

## 应用范围

本规则适用于独立权利要求和从属权利要求的撰写与审校，也适用于说明书具体实施方式中涉及方向、方位关系的描述。与 `patent-structural-claim-drafting`、`patent-avoid-unnecessary-functional-limitation` 两个skill配合使用：前两者管"部件怎么排列展开""哪些功能性收尾该留该删"，本skill专门管"方向、方位词是否清楚、有没有参照物"。
