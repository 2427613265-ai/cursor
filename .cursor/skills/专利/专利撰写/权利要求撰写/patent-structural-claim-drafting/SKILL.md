# 结构类权利要求撰写规范：先总述部件，后展开结构

## 适用场景

- 撰写"一种XX装置/机构/系统"这类**装置类（结构类）独立权利要求**。
- 撰写从属权利要求中，一次性引入**两个或以上新部件**的场合（例如"XX还包括部件甲和部件乙"）。
- 不适用于方法类权利要求（方法权利要求按步骤顺序撰写，不适用本规则）。

## 核心规则

在"其特征在于"（或"XX包括"）之后，**先用一个独立的分句把本层级涉及的全部部件一次性列全**（例如"包括外罩、风叶和降尘组件"），这句话到此为止，**不得夹带**任何部件的内部结构、连接关系、位置关系描述。

列完部件清单之后，再按清单顺序，逐一用后续分句展开每个部件的：
- 安装方式/位置关系（装在哪里、设在哪一侧、是否同轴等）；
- 连接关系（固定连通、固定连接、转动连接等）；
- 自身的结构特征（管壁开孔、截面形状等）。

如果某个部件本身又由多个子部件组成，则在展开该部件时，**递归应用同一规则**：先用一句把子部件清单列全，再逐一展开子部件。层级可以嵌套多层，但每一层进入下一层展开之前，都必须先把该层的部件名单说完整。

## 正确示范

```
一种XX装置，其特征在于，包括外罩、风叶和降尘组件；
所述风叶安装在所述外罩的内壁，能够转动以产生轴向气流；
所述降尘组件设置在所述风叶的一侧，所述降尘组件包括环管和多个内喷雾管；
所述环管同轴设置在所述风叶的出风侧；
多个所述内喷雾管呈放射状固定连通于所述环管的内壁，每个所述内喷雾管的管壁上开设有多个内雾化孔；
所述内雾化孔的开口方向与所述风叶所在平面之间形成有夹角……
```

要点解析：
1. 第一句"包括外罩、风叶和降尘组件"一次性点名三个顶层部件，句末即止。
2. 第二句才回过头交代"风叶"的安装方式。
3. 第三句交代"降尘组件"的位置，并且因为"降尘组件"本身还有子部件，所以紧接着用同一分句里的"包括环管和多个内喷雾管"再次先列全子部件清单，之后才展开"环管""内喷雾管"各自的结构。

## 错误示范（应避免）

```
一种XX装置，其特征在于，包括外罩，所述外罩的内壁安装有风叶，所述风叶能够转动以产生轴向气流；
所述风叶的一侧设有降尘组件，所述降尘组件包括环管和多个内喷雾管；……
```

问题：只列了"外罩"一个部件，"风叶"是被夹带在展开"外罩"结构时顺带带出来的，并未出现在开头的部件清单句里，读者无法从第一句直接看出装置的完整顶层构成，逻辑层次也变得不清晰。

同理，从权中若一句话里同时列出两个新部件，也不要把其中一个部件的连接关系直接写在列举分句里，例如应避免：

```
……包括固定连通于所述环管内壁的固定套筒，以及与所述内喷雾管端部固定连接、同轴可转动地插接于所述固定套筒内的旋转接头；
```

应改写为：

```
……包括固定套筒和旋转接头；
所述固定套筒固定连通于所述环管的内壁；
所述旋转接头与所述内喷雾管的端部固定连接，且同轴可转动地插接于所述固定套筒内；
```

## 撰写/审核检查清单

- [ ] "其特征在于"之后的第一句，是否已经用"包括A、B和C"的句式把该层级的全部部件一次性列全？
- [ ] 这句部件清单里，有没有夹带任何部件的内部结构/连接关系描述？如果有，挪到后面单独成句。
- [ ] 从权中一次性引入多个新部件时，是否也先用一句把新部件列全，再分句展开？
- [ ] 多层级部件（部件里还有子部件）是否在展开该部件时，同样先列子部件清单、再展开子部件？
- [ ] 部件清单的先后顺序，是否和后续展开描述的顺序保持一致，便于对照阅读？
