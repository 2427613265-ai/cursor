# 【第3步-④ / 共11步】驳斥结论④（CIE搭配/补偿亮度/分段函数=公知或常规）

第四、针对审查意见指出的：权利要求4引用权利要求3，对比文件1已经建立了根据照度偏差调节亮度的闭环，对比文件2给出了需根据体感调节色温的技术启示，本领域技术人员知晓，在照明设计中，色温与照度（亮度）存在公认的搭配关系以实现视觉舒适（例如，CIE标准等文献中建议高色温宜搭配较高照度，低色温宜搭配较低照度）。因此，当系统根据体感温差E决定目标色温后，为了维持或达到最佳的视觉舒适效果，自然需要根据当前的实际照度情况，对亮度进行相应的补偿性调节。这是照明工程领域的公知常识。本领域技术人员在掌握了对比文件2的体感-色温映射关系和对比文件1的照度-亮度调节原理，通过有限次试验和参数调试将体感温差E、实时照度Lrmk+1、目标色温SKTk+1和最终亮度调整量通过一个数学关系式（如分段函数）关联起来，属于一种具体的、常规的算法实现方式。因此，在其引用的权利要求不具备创造性时，该从属权利要求也不具备专利法第二十二条第三款规定的创造性。

对此，申请人答复如下：

其一、审查意见将CIE等文献中关于“高色温宜搭配较高照度、低色温宜搭配较低照度”的静态设计建议，过度概括为足以直接得到本申请动态控制律的“公知常识”。照明设计中的色温—照度搭配关系，充其量提供了一种静态、经验性的匹配方向，并不能等同于自动控制中如下具体技术手段：以体感温差E作为主分段变量，在过冷区间进一步嵌套实时照度与照度标准的比较关系，联立输出色温目标期望与耦合亮度调整量，并嵌入“照度闭环达初步期望→体感耦合达目标期望→时序循环维持”的控制流程。公知的设计搭配原则，不能被抽掉适用场景、输入变量、输出结构与控制时序后，直接当作本申请照度色温耦合调控模型的完整技术教导。

其二、所谓“根据体感温差E决定目标色温后，再按实际照度对亮度作补偿性调节”，是在已经知晓本申请技术方案之后所作的事后重构。对比文件2存在“体感/舒适评价→色温”的路径，但其光量控制服务于色温合成，并非基于实时照度达标状况的亮度补偿；对比文件1存在“照度偏差→亮度步长”的路径，但完全不涉及体感温差与色温。二者均亦未给出“先由E决定色温目标、再据照度补偿亮度、并以统一分段函数联立二者”这一控制逻辑。以本申请为指引，将CIE静态建议、对比文件1的照度—亮度原理与对比文件2的体感—色温映射事后拼贴为“自然需要补偿调节”，属于典型的事后分析，不能证明该联立调控在申请日前是显而易见的。

其三、退一步讲，即便承认照明领域存在色温与照度宜相匹配的一般认识，从该一般认识到本申请（现已并入独立权利要求1的）具体分段映射之间仍存在明显技术鸿沟：选择以体感温差E而非单纯目标色温作为分段主变量；在E＜−1.25/2℃时嵌套三档照度条件以联立亮度调节；在中间温差区间采用与|E|相关的色温连续调节并同步给出耦合亮度调整量；在过热区间将色温置于6000K并将亮度调至最大值；以及上述映射如何与步骤S2的初步期望调节、步骤S6的预设时长循环相衔接。对比文件1、2及所谓公知常识对上述关键选择均无明示或暗示；“有限次试验和参数调试”不能替代现有技术对上述具体控制结构给出的技术启示。因此，原权利要求4的附加技术特征并入独立权利要求1后，修改后的权利要求1整体相对于对比文件1、2及公知常识的结合仍非显而易见。
