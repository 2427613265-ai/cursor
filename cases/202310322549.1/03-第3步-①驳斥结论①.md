# 【第3步-① / 共11步】二、针对审查意见核心观点的逐条答复（结论①）

二、针对审查意见核心观点的逐条答复

第一、针对审查意见指出的：由此可见，该权利要求要求保护的技术方案与对比文件1的区别特征在于：该权利要求要求保护一种家庭光环境与色温智能控制方法，响应目标灯光控制请求而触发；S3，获取所述室内目标空间在第k+1时刻的实时灯光色温SKTk+1、实时空气干球温度Tairk+1、实时空气相对湿度RHairk+1；S4，根据实时空气干球温度Tairk+1以及实时空气相对湿度RHairk+1获取第k+1时刻的实际体感温度与目标体感温度的温差值E；S5，根据所述温差值E输入至预先设定的照度色温耦合调控模型，得到所述温差值E对应的色温调控步长以及耦合亮度调节步长，以将所述室内目标空间的实时灯光色温SKTk+1以及实际灯光亮度Llmk+1调整为目标期望值；其中，所述照度色温耦合调控模型包含实时灯光色温、实际灯光亮度以及色温调控步长之间的映射关系；S6，等待预设时长，然后重复步骤S1～S5，以将所述室内目标空间维持在“灯光智控”模式。基于上述区别特征可以确定，该权利要求要求保护的技术方案实际要解决的技术问题是：如何调节灯光以主动改善人体的热舒适感。

对此，申请人答复如下：

其一、审查意见将本申请实际解决的技术问题概括为“如何调节灯光以主动改善人体的热舒适感”，属于对技术问题的过度上位。按照该上位界定，凡是利用照明参数对人体热舒适或体感温度施加影响的改进——例如仅调节色温、仅调节亮度，甚或不同控制对象、不同控制时序与不同映射结构的方案——都可能被轻易认定为“显而易见”，从而使创造性判断失去对本申请具体技术方案的针对性。

其二、区别技术特征本身所客观体现的，并非抽象的“用灯光改善热舒适”，而是一套有明确结构的控制问题：在已通过灯光亮度调控模型将实际灯光亮度调整至初步期望值的基础上，如何进一步根据体感温差值E，在照度色温耦合调控模型中**联立**给出色温调控步长与耦合亮度调节步长，使实时灯光色温与实际灯光亮度同步调整至目标期望值，并在预设时长后循环执行，从而将室内目标空间维持在“灯光智控”模式。尤其在申请人已将原权利要求4并入独立权利要求1之后，区别特征还进一步体现为：上述耦合调控并非停留在“有映射关系”的上位表述，而是以体感温差E为主分段条件，并在E处于过冷区间时嵌套实时照度条件，具体联立确定色温目标期望与耦合亮度调整量。

其三、因此，本申请实际解决的技术问题应更贴切地界定为：**如何在室内照度闭环调节达初步期望的基础上，进一步基于体感温差对灯光色温与亮度进行耦合联立调控，并实现可持续维持的智控循环**。申请人将围绕该具体技术问题，进一步说明对比文件1、2单独或其结合均未给出得到修改后权利要求1的技术启示。
