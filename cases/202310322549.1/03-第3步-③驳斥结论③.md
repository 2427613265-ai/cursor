# 【第3步-③ / 共11步】驳斥结论③（结合动机 / 功能集成易想到）

第三、针对审查意见指出的：在对比文件1已公开基于照度闭环调节亮度的基础上，本领域技术人员在面对如何进一步提升室内环境舒适度这一技术问题时，有动机结合对比文件2的教导。对比文件2明确教导了可以利用温湿度计算体感指标并据此调节光源色温来改善体感温度。将这一色温调节功能与对比文件1的照度调节功能相结合，形成一个先调亮度初步满足照度，再根据体感调色温并进行亮度微调的系统功能集成思路对本领域技术人员而言是容易想到的。对比文件1已经公开了基于一个模型和计算出的步长实现对亮度的调整，采用类似的方法来实现后续步骤中色温、以及亮度的精细化调整，并循环该过程从而保证家庭室内目标空间维持在灯光智控模式对本领域技术人员而言是容易想到的。响应于控制请求触发控制方法属于本领域的惯用技术手段。因此，在对比文件1的基础上结合对比文件2得出该权利要求的技术方案，对于本领域技术人员来说是显而易见的，该权利要求不具有突出的实质性特点和显著的进步，不具备专利法第二十二条第三款规定的创造性。

对此，申请人答复如下：

其一、对比文件1、2各自“未排除”引入另一手段，或者一般性地存在“进一步提升舒适度”的愿望，并不等于本领域技术人员有动机得到本申请修改后权利要求1的具体方案。“未排除”与“有教导、有动机得到本申请”属于不同层次；以对比文件1未涉及色温、对比文件2未排除照度闭环为由，直接推断二者将被结合成“先调亮度初步满足照度，再根据体感调色温并进行亮度微调”的完整控制结构，是将结论建立在事后叙述之上，而非建立在现有技术自身的客观教导之上。

其二、审查意见关于“功能集成思路容易想到”的认定，实质上是在已知本申请技术方案之后，将完整方案拆解为“照度调亮度”“体感调色温”“再对亮度微调”“循环维持”等局部动作，再分别到对比文件1、2中寻找对应片段并反向拼接。本领域技术人员在申请日前面对对比文件1时，其所解决的是照度达标与亮度平缓调节问题，其技术方案已构成照度—亮度闭环的自洽体系，并不存在必须引入温湿度检测、体感温差及色温—亮度耦合控制的内在需求；面对对比文件2时，其所解决的是基于舒适评价调节光色/色温的问题，亦无建立照度初步期望值、更无“耦合亮度调节步长”这一与照度条件联动的控制量。以本申请为蓝图对两篇对比文件进行事后拆拼，不能证明结合是显而易见的。

其三、对比文件1在其发明目的范围内已经能够独立完成“使目标区域照度恒定、亮度调节平缓”的既定目标，属于自洽闭环；本领域技术人员没有动机为完成该目标再额外引入温湿度传感器、体感温差计算以及照度色温耦合调控模型，否则将增加系统复杂度与控制环节，偏离对比文件1减负、自适应照度调节的技术路线。同时，对比文件1整体缺失本申请的核心功能结构——包括第k+1时刻体感温差驱动的色温与亮度联立输出、原权利要求4并入后的E主分段及照度嵌套映射、以及“初步期望值→目标期望值”两阶段时序与智控循环维持。对比文件2亦不能弥补上述缺失。因此，即便将对比文件2的色温调节“叠加”到对比文件1上，所能得到的至多是“照度调亮度 + 温湿度调色温”的松散功能并列，而不是本申请所保护的耦合联立调控方案，更不能自然得到修改后权利要求1的完整技术特征组合。至于“响应控制请求触发”即便属于惯用手段，也不能弥补上述实质性区别，更不能使结合后的方案整体变得显而易见。
